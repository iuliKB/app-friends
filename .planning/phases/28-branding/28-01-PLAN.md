| Boundary | Description |
|----------|-------------|
| Developer machine → app binary | app.config.ts values are compiled into native binary at build time; no runtime attack surface |
| OS splash → JS bundle | Splash screen is dismissed before JS loads; no user input possible during splash |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-28-01 | Tampering | assets/images/icon.png | accept | Asset is a static PNG baked into the binary at build time; no runtime write path exists. Supply-chain risk is managed by the user providing the file directly. |
| T-28-02 | Information Disclosure | app.config.ts (EAS_PROJECT_ID) | accept | Already present in codebase; not introduced by this phase. EAS project ID is not a secret (it is public in EAS submissions). |
| T-28-03 | Denial of Service | SplashScreen.setOptions (module scope) | accept | setOptions is a synchronous SDK call with no network I/O and no user input. Failure mode is an unchanged splash (no crash). |

**Security assessment:** This phase introduces zero new attack surface. All changes are build-time config values and a single synchronous JS call. ASVS L1 categories V2-V6 (auth, session, access control, input validation, crypto, data) do not apply.
