## Spacing Scale

Project tokens from `src/theme/spacing.ts`:

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xs | 4px | Avatar overlap offset in AvatarStack |
| sm | 8px | Gap between avatar stack and member badge; card internal row gaps |
| md | 12px | Gap between card content rows (date, avatar row) |
| lg | 16px | Card internal padding; section horizontal padding |
| xl | 24px | SectionHeader `paddingTop` (existing component) |
| xxl | 32px | SectionHeader `paddingTop` at top of section (existing behavior) |

**Exception: `SPACING.md = 12px`** — Pre-existing project token from `src/theme/spacing.ts`, not a value introduced by this phase. 12 = 4×3, grid-aligned. The project's 8-point-adjacent scale includes this intermediate step for tighter spacing needs and is used throughout the existing codebase.

Exceptions:
- Event card width: 200px (not a spacing token — card dimension). Source: CONTEXT.md D-01.
- Event card height: 140px (not a spacing token — card dimension). Source: CONTEXT.md D-01.
- AvatarStack size on event cards: 24px avatar diameter (smaller than default 28px to fit the compact card layout). Claude's discretion.
- Horizontal FlatList `contentContainerStyle` left padding: 16px (`SPACING.lg`) to align with screen edge. Right padding: 8px (`SPACING.sm`) to show card bleed.
- Card gap in FlatList: 12px (`SPACING.md`) between cards.

### Event Card Background Colors

Cards use a pastel background palette — NOT theme COLORS tokens. These are deterministic per plan (assigned via plan ID hash):

| Slot | Hex | Description |
|------|-----|-------------|
| Pastel pink | `#F9A8C9` | Warm pink |
| Pastel yellow | `#FDE68A` | Soft amber-yellow |
| Pastel blue | `#93C5FD` | Light sky blue |
| Pastel green | `#86EFAC` | Soft mint green |
| Pastel lavender | `#C4B5FD` | Light violet |

Assignment: `PASTEL_COLORS[plan.id.charCodeAt(0) % PASTEL_COLORS.length]`

These are hardcoded constants in the EventCard component with an `// eslint-disable-next-line campfire/no-hardcoded-styles` comment per project convention.

### Card Image Overlay

When a cover image is present: dark overlay at **40% opacity** (`rgba(0,0,0,0.40)`) over the image, applied as an absolutely-positioned View on top of the image. This ensures card title (white, `COLORS.text.primary`) meets 4.5:1 contrast against the darkened image background. Source: Claude's discretion.

### Card Text on Colored Background

When no cover image (pastel background): card title and date text use `#1a1a1a` (dark, hardcoded) for contrast against light pastels. Member badge text uses `#1a1a1a` as well. These are the only color values that deviate from `COLORS.text.primary` (`#f5f5f5`) — required for legibility on light surfaces.

## Interaction Contracts

### EventCard tap
- Trigger: `TouchableOpacity` with `activeOpacity={0.8}` (matches PlanCard pattern)
- Action: `router.push` to plan detail screen, same pattern as PlanCard in Explore tab
- Touch target: full card surface (200×140px minimum — well above 44×44px minimum)
- Source: CONTEXT.md D-17

### Horizontal scroll
- Behavior: `FlatList` with `horizontal={true}`, `showsHorizontalScrollIndicator={false}`
- Snap: `snapToInterval={200 + 12}` (card width + gap) with `decelerationRate="fast"` — snaps to card boundaries. Source: Claude's discretion.
- Paging: `pagingEnabled={false}` — snap per card, not per screen
- Bleed: 1.5 cards visible at a time confirms scroll affordance. Source: CONTEXT.md D-01.

### "See all" link
- Trigger: `TouchableOpacity` in `SectionHeader` `rightAction` prop
- Action: `router.push('/(tabs)/explore')` — navigate to Explore tab
- Source: CONTEXT.md D-10

### Placeholder card tap (empty state)
- Trigger: `TouchableOpacity` covering the placeholder card
- Action: Navigate to plan creation screen (same route as FAB on HomeScreen)
- Source: CONTEXT.md D-12

### Image picker (plan creation / edit)
- Trigger: Tappable area (button or camera icon) within plan creation form
- Action: Opens `expo-image-picker` (camera roll only, no camera capture)
- Feedback: Selected image shown as preview thumbnail in the form
- Source: CONTEXT.md D-15

## Card Layout Anatomy

```
┌──────────────────────────────┐
│  [cover image OR pastel bg]  │  ← 200px × 140px, RADII.xl (20px)
│  [dark overlay if image]     │
│                              │
│  Title text                  │  ← FONT_SIZE.lg, semibold, white or dark
│  Mon 15, Aug · in 2 days     │  ← FONT_SIZE.sm, regular, white or dark
│                              │
│  [●●●] +12   ⏱ 1h           │  ← AvatarStack size=24 + badge + duration
└──────────────────────────────┘
```

Card radius: 20px (use hardcoded `20` with `// eslint-disable-next-line campfire/no-hardcoded-styles` since `RADII.xl` = 16px and D-05 calls for "extra rounded corners / RADII.xl or 20px"). Source: CONTEXT.md D-05. Prefer `RADII.xl` (16px) first; if visually insufficient, use hardcoded 20 with lint disable comment.

Card shadow: use `SHADOWS.card` (`shadowOpacity: 0.15, elevation: 2`) — subtle depth without competing with the swipe card shadow. Source: Claude's discretion.

Member badge: `+N` in accent color (`COLORS.interactive.accent` when on dark/image card; dark text when on light pastel card) rendered inline after `AvatarStack`. Badge is a `Text` element, not a separate rounded container.

Duration/time icon: Ionicons `"time-outline"` at 12px, same color as date text, displayed inline after member badge with `SPACING.sm` left margin. Source: Claude's discretion (reference image shows a clock indicator).

## Empty / Edge States

### Empty state (no upcoming events)
Display: section header stays visible. Below it, a single placeholder card (same 200×140px dimensions) with `COLORS.surface.card` background and dashed `COLORS.border` border (1px, `borderStyle: 'dashed'`).

Placeholder card contents:
- Icon: Ionicons `"calendar-outline"` at 32px, `COLORS.text.secondary`
- Text: centered, two lines (see Copywriting Contract below)
- Tapping the card navigates to plan creation

Do NOT use the full-screen `EmptyState` component — section should remain visible in the HomeScreen ScrollView. Source: CONTEXT.md D-12.

### Single event
No special case — renders normally in horizontal FlatList. Source: CONTEXT.md D-13.

### Loading state
While `usePlans` is fetching: render section header + a single placeholder card with `opacity: 0.5` and no text (skeleton-style). No spinner overlay.

## Date Format Contract

Format per CONTEXT.md D-18 and D-19:

```
"Mon 15, Aug · in 2 days"
```

Implementation: extend `formatPlanTime` from `PlanCard.tsx` to return a composite string:

- Short date: `date.toLocaleDateString([], { weekday: 'short', day: 'numeric', month: 'short' })` → "Mon 15, Aug"
- Separator: " · "
- Relative: reuse existing `formatPlanTime` relative logic → "in 2 days", "Tomorrow", "Today", "In 3h"

Export a new `formatEventCardDate(scheduledFor: string): string` utility from `PlanCard.tsx` or a new `src/lib/formatDate.ts` file.

## Accessibility Notes

- All `TouchableOpacity` elements on EventCard must have `accessibilityRole="button"` and `accessibilityLabel` containing plan title + date (e.g. "On your bike, Monday August 15th").
- AvatarStack `overflow` badge already has `accessibilityLabel={`${overflow} more members`}` — no change needed.
- Dark overlay on image cards must achieve 4.5:1 contrast for white title text — 40% opacity overlay on typical photos achieves this (confirmed by reference image visual).
- Reduced motion: no animations in this phase beyond standard React Native scroll. No additional guard needed.
