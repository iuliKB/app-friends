<objective>
Migrate all 30 shared, auth, status, friends, and notifications components from static `COLORS` imports to the `useTheme()` hook.

Purpose: Shared components are consumed by every screen. Migrating them first means that every screen that renders these components immediately gets correct light/dark colors once screens are migrated in Plans B and C. Starting from the bottom of the dependency tree prevents any "partially themed" intermediate state.

Output: 30 component files migrated in-place. Each file imports `useTheme` instead of `COLORS`, calls `const { colors } = useTheme()` inside the component body, and wraps `StyleSheet.create` in `useMemo([colors])`. The compat shim in `src/theme/index.ts` is NOT removed yet — that is Plan C's job. `tsc --noEmit` passes with zero errors after this plan.
</objective>

Apply the three-step migration recipe to each of the 13 files, using the patterns below. After each file, grep the file for `COLORS\.` and `import.*COLORS` to confirm zero matches before moving to the next.

**Standard recipe (applies to every file except special cases):**
1. Change `import { COLORS, ... } from '@/theme'` → `import { useTheme, ... } from '@/theme'` (keep all other named imports like SPACING, FONT_SIZE, RADII, etc.)
2. Add `useMemo` to the React import: `import React, { useMemo } from 'react'` (or add `useMemo` to existing destructure)
3. Inside the component function body, add `const { colors } = useTheme();` as the first line
4. Move any module-level `StyleSheet.create({ ...COLORS... })` inside the component body and wrap it: `const styles = useMemo(() => StyleSheet.create({ ... }), [colors]);`
5. Replace every `COLORS.x` with `colors.x` (both inside StyleSheet and inline JSX props like `color={COLORS.x}`, `trackColor={}`, `thumbColor={}`, `tintColor={}`, `shadowColor={}`)

**SPECIAL CASE — LoadingIndicator.tsx (Pattern 4):**
The current signature has `color = COLORS.text.secondary` as a default parameter. Hooks cannot be called in default parameter values. Instead:
```typescript
// BEFORE signature:
export function LoadingIndicator({
  color = COLORS.text.secondary,
  size = 'large',
  style,
}: LoadingIndicatorProps) { ... }

// AFTER — remove default from signature, resolve inside body:
export function LoadingIndicator({
  color,
  size = 'large',
  style,
}: LoadingIndicatorProps) {
  const { colors } = useTheme();
  const resolvedColor = color ?? colors.text.secondary;
  // use resolvedColor wherever color was used
}
```
If `LoadingIndicator.tsx` has a static `StyleSheet.create` with only non-color entries (layout), it may remain at module scope — no useMemo needed for non-color-only StyleSheets.
