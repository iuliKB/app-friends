#### D. Idempotent toggle RPC (`toggle_habit_today_checkin`)

**Source:** Schema-level uniqueness from `0017:78-83` (`wish_list_claims` composite PK). Inside the RPC, two-phase logic: try `DELETE` for today's row; if `FOUND` → return `false`; else `INSERT` and return `true`. This is the simplest interpretation of D-07 + Pitfall 8.

```sql
-- skeleton (cloned shape from create_birthday_group, with toggle semantics)
CREATE OR REPLACE FUNCTION public.toggle_habit_today_checkin(
  p_habit_id   uuid,
  p_date_local date
) RETURNS boolean LANGUAGE plpgsql SECURITY DEFINER SET search_path = ''
AS $$
DECLARE
  v_caller uuid := auth.uid();
BEGIN
  IF v_caller IS NULL THEN RAISE EXCEPTION 'not authenticated'; END IF;
  IF NOT public.is_habit_member(p_habit_id) THEN RAISE EXCEPTION 'not a member'; END IF;

  DELETE FROM public.habit_checkins
   WHERE habit_id = p_habit_id AND user_id = v_caller AND date_local = p_date_local;
  IF FOUND THEN RETURN false; END IF;

  INSERT INTO public.habit_checkins (habit_id, user_id, date_local)
       VALUES (p_habit_id, v_caller, p_date_local);
  RETURN true;
END;
$$;
GRANT EXECUTE ON FUNCTION public.toggle_habit_today_checkin(uuid, date) TO authenticated;
```

#### E. RPC that atomically updates state + inserts system message (`complete_chat_todo`)

**Source assembly:** `0015_iou_v1_4.sql:118-170` (atomic RPC shape) + `0017:181-193` (multi-table write inside DEFINER) + system-message INSERT (no existing precedent; pattern derived from how `create_poll` inserts the parent message row).

```sql
CREATE OR REPLACE FUNCTION public.complete_chat_todo(
  p_item_id uuid
) RETURNS uuid LANGUAGE plpgsql SECURITY DEFINER SET search_path = ''
AS $$
DECLARE
  v_caller        uuid := auth.uid();
  v_list_id       uuid;
  v_group_chan_id uuid;
  v_title         text;
  v_assignee_name text;
  v_msg_id        uuid;
BEGIN
  IF v_caller IS NULL THEN RAISE EXCEPTION 'not authenticated'; END IF;

  -- Idempotency: bail if already completed (avoids duplicate system message — A6)
  UPDATE public.chat_todo_items
     SET completed_at = now()
   WHERE id = p_item_id
     AND assignee_id = v_caller
     AND completed_at IS NULL
  RETURNING list_id, title INTO v_list_id, v_title;
  IF NOT FOUND THEN RETURN NULL; END IF;

  SELECT group_channel_id INTO v_group_chan_id FROM public.chat_todo_lists WHERE id = v_list_id;
  SELECT display_name INTO v_assignee_name FROM public.profiles WHERE id = v_caller;

  INSERT INTO public.messages (group_channel_id, sender_id, body, message_type)
       VALUES (v_group_chan_id, v_caller, '✓ ' || v_assignee_name || ' completed ' || v_title, 'system')
   RETURNING id INTO v_msg_id;

  RETURN v_msg_id;
END;
$$;
GRANT EXECUTE ON FUNCTION public.complete_chat_todo(uuid) TO authenticated;
```
