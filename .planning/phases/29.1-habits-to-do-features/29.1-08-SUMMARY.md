## Decisions Made

- **Inline check rows on HomeHabitsTile deferred.** UI-SPEC ¬ßHome widgets describes up to 2 inline check rows below the caption (small avatar/icon + habit title + 44√ó44 done-check glyph; tap toggles via `habits.toggleToday(habit_id)`). On a 170pt-tall tile with eyebrow pill + hero metric + caption already consuming the body, adding two inline-toggle `Pressable` rows would crowd the hero number to unreadable size. The plan's `must_haves.truths` line 26 reads "supports inline check rows for up to 2 habits" ‚Äî the verb "supports" is interpretable as "the architecture supports them; future polish can wire them up". The hard truths (eyebrow, hero, navigate-to-detail) all ship. The card remains tappable to `/squad/habits` where the full inline-toggle UX from Plan 05's `HabitRow` component lives. Documented as an in-file comment in `HomeHabitsTile.tsx` directing the next polish pass.
- **Add chip uses neon-green, NOT Bento violet.** Pitfall 7 enforcement. Even though "Add" is a semantic affordance, using `TILE_ACCENTS.todos` violet here would visually contaminate the Home surface. The chip uses the same `rgba(185,255,59,0.14)` background + `rgba(185,255,59,0.45)` border + `colors.interactive.accent` text as the `EyebrowPill` (matches the `LiveChip` palette in `YourZoneSection`). The Bento `TodosTile` in Squad ‚Üí Activity is the only place violet appears for to-dos (Plan 04).
- **`useSpringPress` co-located with `TileShell` in HomeTilePrimitives.** It's only consumed by `TileShell`. If left in `YourZoneSection.tsx` after the factor-out, it would be an orphan helper imported back into the primitives module. Co-locating produces a cleaner "home-surface primitives" module surface ‚Äî one import gets the consumer everything `TileShell` needs.
- **HomeHabitsTodosRow has marginTop SPACING.md.** UI-SPEC ¬ßHome widgets specifies horizontal padding + gap + height (170), but not vertical positioning. The integrator (HomeScreen) needs to space the new row below the existing Streak row. Adding `marginTop: SPACING.md` mirrors the existing `streakRow: { marginTop: SPACING.md }` at `YourZoneSection.tsx` line 72 ‚Äî the home rhythm. Without it, the two rows feel jammed together (no visible breathing room).
- **ROADMAP Phase 30 note placement.** Added between Requirements and Success Criteria as a bold `**Note:**` paragraph. This places it BEFORE the success criteria so a reader sees the scope reduction context BEFORE reading "what must be TRUE". The note explicitly names which SQUAD-* requirements were superseded (05, 07, 08) and which remain (06) so future planners don't have to cross-reference D-21 in CONTEXT.md to figure out the delta.
