**File 2: `src/components/todos/ChatTodoListRow.tsx`**

**Pattern: controlled-prop expansion (W9 from checker pass 1).** Parent owns `expanded` state and decides when items are fetched. ChatTodoListRow renders based on `items.length` — non-empty + same `row.list_id` → expanded view; empty → collapsed view. Animation driven by `LayoutAnimation.configureNext` inside the `onPress` of the header BEFORE calling `onExpand` (parent's state update triggers the re-render that animates).

Props:
```typescript
interface ChatTodoListRowProps {
  row: ChatTodoRow;
  items: ChatTodoItem[];                  // empty = collapsed; populated = expanded (controlled-prop)
  loadingItems: boolean;                  // true while child items load
  onExpand: (listId: string) => void;     // header tapped — parent fetches items + re-renders with non-empty items
  onToggleItem: (itemId: string) => Promise<unknown>;
}
```

Layout:
- Collapsed state (`items.length === 0`): `📋 {row.title} · {row.done_count}/{row.total_count} done` + chevron-down (rotates 0deg)
- Expanded state (`items.length > 0`): same header + chevron rotates 180deg + below each child item rendered as row (`{item.title}` + 44×44 checkbox)
- Tap header → call `LayoutAnimation.configureNext(LayoutAnimation.Presets.easeInEaseOut)` THEN call `onExpand(row.list_id)` (per UI-SPEC §Animation > Chat to-do list expand)
- Component does NOT maintain internal `expanded` state — `items.length > 0` is the sole source of truth
- Chevron rotation: `Animated.timing` with `useNativeDriver: true`, `duration: ANIMATION.duration.fast`.
- When expanded but `items` is empty AND `loadingItems`: render 3 SkeletonPulse rows (height=40).
- When `items` populated: render each, sorted incomplete-first.

a11y: header `accessibilityLabel={row.is_list ? 'To-do list ' + row.title : ...}` + `accessibilityHint="Tap to expand"`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User input → create_todo RPC | Title, due_date, notes, priority all user-controlled |
| User input → todos.completed_at UPDATE | Direct table UPDATE — RLS enforces ownership |
| Expanded chat_todo_items SELECT | Caller queries items for a list they may or may not own/be assigned |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-29.1-28 | Tampering | Attacker updates someone else's Mine to-do via direct PATCH | mitigate | `todos` RLS UPDATE policy: `user_id = auth.uid()` (Plan 01). |
| T-29.1-29 | Info Disclosure | Expanded chat_todo_items query returns items from lists caller doesn't own/get-assigned | mitigate | `chat_todo_items` SELECT RLS: `is_chat_todo_list_owner(list_id) OR list.created_by = auth.uid()` (Plan 01). |
| T-29.1-30 | DoS | Attacker submits to-do with 1MB title/notes | mitigate | `todos.title` CHECK constraint `length BETWEEN 1 AND 120`; notes uncapped but PostgREST request size limit applies (Supabase default 1MB). Acceptable for v1. |
| T-29.1-31 | XSS | Notes field accepts user-typed HTML, rendered as text | mitigate | React Native's `<Text>` does not interpret HTML. Plain-text rendering throughout. No XSS surface in RN environment. |
