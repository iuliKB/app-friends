# Phase 29.1: Habits & To-Dos - Context

**Gathered:** 2026-05-12
**Status:** Ready for planning

<domain>
## Phase Boundary

Add **two new social features** to Campfire, each modeled after the IOU/Birthday feature pattern (schema migration → Bento tile in Squad → Activity tab → full-screen list/detail → home widget):

1. **Habits** — Users create habits they want to keep (solo) or commit to with friends (group). Group habits let small friend groups commit to a shared cadence (e.g., "gym 3× per week together"). Each member checks in independently.
2. **To-Dos** — Personal task list, with an "From chats" section showing to-dos assigned to the user via group chat (wires up the existing `SendBar` "To-Do List" attachment stub that today shows "Coming Soon").

**In scope:** Schema (migrations + RLS + RPCs), Squad → Activity tab Bento tiles, Spotlight tile eligibility, full-screen list/detail routes under `/squad/habits` and `/squad/todos`, home widget for each feature, group chat to-do creation + assignment + completion roundtrip (system message + bubble update).

**Out of scope (deferred):** push notification action buttons (EAS dev build dependency), tagging, recurring/cron-style schedules, OS calendar sync, account-level analytics.

</domain>

<decisions>
## Implementation Decisions

### Names & labels
- **D-01:** Habit feature is called **"Habits"** (Squad Activity tile title + screen header). Avoids collision with existing "Streak" and "Goals" terminology.
- **D-02:** Within Habits, solo vs group flavors are labeled **"Solo / Group"** (chip-style discriminator on each habit row).
- **D-03:** To-Do feature is called **"To-Dos"** (Squad Activity tile title + screen header). Matches the existing `SendBar` attachment label "To-Do List".
- **D-04:** Personal To-Dos screen has **two sections: "Mine" and "From chats"** (collapsible, each scrollable).

### Habit mechanics
- **D-05:** Cadence options at habit creation: **Daily / Weekly / N× per week** (`weekly_target` integer for N×). Free-form schedules deferred.
- **D-06:** Group-habit completion rule: **each member checks in independently**. The card/tile shows e.g. "2/3 friends done today". No threshold or shared-checkbox modes.
- **D-07:** Check-in surface: **inline on the Squad Activity tile** (today's habits are tappable rows; one tap toggles done) **plus** the detail screen (history + edit). Home widget also supports one-tap check (see D-19).
- **D-08:** Squad Activity Habits tile hero metric: **"X/Y done today"** (refreshes daily, midnight reset).

### To-Do ↔ chat link
- **D-09:** `SendBar` "To-Do List" attachment opens a **two-step picker**: user chooses **single item** vs **list of items**, then enters details. Replaces the current "Coming Soon" alert at `ChatRoomScreen.tsx:149`.
- **D-10:** Chat-to-do **assignment**: creator picks **one assignee** from the group chat members at creation time. Chat-to-do shape: `title + due date + assignee + origin chat`. No notes/priority in chat to-dos.
- **D-11:** Completion roundtrip: when the assignee marks a chat to-do done, the chat bubble visually updates to checked state **and** a system message is posted in the thread: `"✓ [name] completed [task]"` (mirrors how "plan created" system messages work).
- **D-12:** Personal "Mine" to-dos shape: **title + optional due date + notes + priority**. Quick-add FAB on the To-Dos screen. Richer than chat-origin shape — the two sections render slightly different row layouts.
- **D-13:** Chat-to-do **lists** (multiple items created in one chat attachment): the entire list is **assigned to one person** at creation. The list surfaces in that assignee's "From chats" section as **one expandable row with progress** (e.g., "📋 Trip prep · 2/5 done"); tap-to-expand reveals items.

### UI surface placement
- **D-14:** Squad screen IA stays as 3 tabs (Friends / Memories / Activity). Habits and To-Dos live inside the **Activity tab Bento Hub** as **two new tiles** alongside existing IOU, Birthdays, Streak, and the Squad Challenges placeholder. Grid grows from 2×2 to **3×2** (or equivalent — final tile arrangement is implementation-flexible).
- **D-15:** The **Squad Challenges placeholder tile stays** (still a future feature); we are *adding* two tiles, not replacing the placeholder.
- **D-16:** **Both Habits and To-Dos are eligible Spotlight candidates.** `selectSpotlight()` (in `src/hooks/useSpotlight.ts`) extends its urgency signals: "habit about to break streak" and "to-do overdue or due today" join existing urgent-birthday / large-IOU / active-streak signals.
- **D-17:** Per-tile hero metric: **Habits "X/Y done today"** + **To-Dos "N due"** (count of overdue + due-today). Matches existing tile metric rhythm (one big number per tile).
- **D-18:** Routes: nested under squad — **`/squad/habits`** (list), **`/squad/habits/[id]`** (detail), **`/squad/habits/create`**; **`/squad/todos`** (list with Mine/From-chats sections), **`/squad/todos/[id]`** (detail), **`/squad/todos/create`**. Matches existing nested squad routes (`/squad/expenses/create`, `/squad/wish-list`).
- **D-19:** **Home screen** gets **both** a compact Habits widget (today's habits with inline check) and a To-Dos widget (count of items due today + CTA). Even though Phase 29 just overhauled Home, these are explicitly requested. Widgets must fit the existing home-widget pattern (compact, themed, navigates to detail).
- **D-20:** Profile tab gets **no** Habits/To-Dos surface — Profile stays account-focused per the v1.2 IA decision ("people in Squad, account in Profile").
- **D-21:** **Phase 30 scope reduction:** 29.1 supersedes the Bento tile redesign portion of Phase 30. Phase 30 retains only the Friends list (CompactFriendRow) polish (SQUAD-06). Phase 30's discussion will need to acknowledge this.

### Claude's Discretion
- Habit row icon choice per habit (single global icon vs user-pickable from a small set)
- Exact two-step chat picker UX (single bottom sheet vs two sequential sheets)
- Empty-state copy + illustrations for each new screen
- Done-toggle animation (haptic + check transition) — but must use existing animation tokens from `src/theme/animation.ts`
- Bento tile gradient / accent color choice for Habits and To-Dos (extend `tileAccents.ts`)
- Tile arrangement in the 3×2 / 2×3 grid (which tile sits where)
- Overdue visual treatment for to-dos in "Mine" and "From chats" rows
- Skeleton loaders for new screens (use existing `SkeletonPulse` shimmer pattern)
- Whether and how to surface group-habit invitation flow inline vs a separate push (constraints below in code_context)

</decisions>

## Existing Code Insights

### Reusable Assets
- **Bento system** (`src/components/squad/bento/`): BentoCard wrapper, BentoGrid layout, tileSurface, tileAccents — directly reused; we add `HabitsTile.tsx` and `TodosTile.tsx`.
- **Spotlight picker** (`src/hooks/useSpotlight.ts`): extend `selectSpotlight()` to consider habit-streak-about-to-break and overdue-to-do signals.
- **Squad nested routes** (`src/app/squad/*`): existing pattern for `/squad/expenses/create`, `/squad/wish-list` etc. — same pattern for `/squad/habits/*` and `/squad/todos/*`.
- **Atomic RPC pattern**: `create_expense` (Phase 8), `add_plan_photo` (Phase 21) — SECURITY DEFINER plpgsql functions for race-free creation. Use the same shape for `create_habit`, `complete_habit`, `create_todo`, `assign_chat_todo`, etc.
- **Optimistic UX**: `useChatRoom` send pattern, `useIOUSummary` refetch pattern — apply to habit check-in and to-do completion.
- **Native DateTimePicker** (v1.4 pattern): used for due-date input on to-dos.
- **SkeletonPulse** (v1.7): use on new list screens during load.
- **SystemMessage rendering**: chat thread already renders system messages for "plan created" / similar events — Habits/To-Dos completion uses the same pattern.

### Established Patterns
- **`useTheme()` + `useMemo([colors])`** for every themed component (v1.6 mandatory).
- **Design tokens only** — no raw colors/fontSize/padding. ESLint enforces.
- **TypeScript strict** with `noUncheckedIndexedAccess`. No `any`.
- **One Realtime channel per user** with `postgres_changes` on row-level tables — used for group habit check-in broadcast and chat-to-do completion broadcast. **No JSONB broadcast** (free-tier budget).
- **RLS-first security**: every new table needs RLS policies before any client query.
- **Integer storage for monetary-like things** — habit `weekly_target` is a smallint, not a float.

### Integration Points
- **Bento grid layout**: `BentoGrid.tsx` must accept two new tiles. Grid math (2 columns × 3 rows, or 3 × 2) needs to handle 6 tiles + spotlight gracefully on small screens.
- **`useSpotlight`** is the single source of "what's urgent" — must be extended atomically with both signals so Spotlight selection stays deterministic.
- **`SendBar` attachment menu** → `ChatRoomScreen.handleAttachmentAction('todo')` — switch from `Alert.alert('Coming Soon')` to a two-step picker (single-item vs list) flow.
- **`MessageBubble` chat rendering** — needs a new "to-do bubble" variant (similar to poll/expense bubbles already in the codebase).
- **Home screen**: Phase 29 just shipped overhauled Home. New Habits + To-Dos widgets must respect the new layout (insert in the home widget row alongside IOU + Birthday + Memories widgets) without breaking Phase 29's design.
- **Friend-membership check** for group habits: reuse the existing friendship/group membership pattern from `create_expense` (membership check inside SECURITY DEFINER RPC).
- **Tab telemetry**: `src/app/(tabs)/squad.tsx` already routes `?tab=memories` for Memory home-widget tap; add `?tab=activity` deep link from the new home widgets.

</code_context>

<specifics>
## Specific Ideas

- **User example for group habit**: *"go 3 times to the gym together"* — drives the **N× per week** cadence option in D-05.
- **Activity tab restructure context**: the user explicitly placed Habits and To-Dos *in the Activity tab*, not as separate stacked cards. This came after the recent (`5caf16a`) Bento Hub redesign.
- **Two-section personal To-Dos** ("Mine" / "From chats") preferred over a single mixed list — discoverability of chat-origin items matters to the user.
- **System message on completion** preferred over silent bubble update or push — keeps the chat thread visible-aware without spamming pushes.
- The user wants both features to **feel native to the Squad/Activity Bento aesthetic** — accent palette, gradient surface, spring press scale, light haptic — not a new visual language.

</specifics>

<deferred>
## Deferred Ideas

- **Push notification action buttons** for habit reminders / to-do due alerts — requires EAS dev build (no Apple Dev account yet, per project_hardware_gate_deferral memory). Revisit in a v1.9 hardware gate phase.
- **Recurring / cron-style habit schedules** (Mon/Wed/Fri etc.) — out of scope; only Daily / Weekly / N×/week supported in v1.
- **Per-item assignment inside chat to-do lists** — out of scope; whole list owned by one assignee.
- **To-Do notes & priority surfaced into chat to-dos** — chat shape stays minimal (title + due + assignee).
- **Profile tab integration** for Habits or To-Dos — explicitly rejected (Profile stays account-focused).
- **Phase 30 re-scoping**: the Bento tile redesign portion of Phase 30 is **superseded** by 29.1's tile work. Phase 30 should be re-scoped to just SQUAD-06 (Friends list / CompactFriendRow polish) during its own discuss-phase.
- **Squad Challenges placeholder tile fate**: stays in the grid as-is. Its real implementation is still a future phase.
- **OS calendar sync** for to-do due dates — out of project scope (V2, per PROJECT.md).
- **Group habit invitation push** — for v1, invited member sees pending group habit in their Habits list and accepts inline (no separate push). Push integration deferred to the hardware gate phase.
- **Habit streak surfacing** — for v1, habits track completion count, not a separate visible streak number on each habit (avoid double-streak confusion with the Squad Goals streak). Reconsider after v1 user feedback.
- **Reviewed Todos (not folded):** none — todo cross-reference returned 0 matches for phase 29.1.

</deferred>
