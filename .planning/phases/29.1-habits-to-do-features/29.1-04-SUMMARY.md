## Decisions Made

- **HabitsTile hero metric rendered as two adjacent Text nodes**, not a single template string. The UI-SPEC §Color rule says "numerator only; `/Y` denominator stays in `text.secondary`" — a single string would force one color across the whole hero. Two Text nodes inside a `heroRow` (flexDirection: 'row', alignItems: 'baseline') give independent color control without breaking the visual alignment.
- **TodosTile accent flips on `overdueCount > 0`**, not on `total > 0`. UI-SPEC §Color explicitly says "Destructive: overdue to-do count when > 0 (To-Dos tile shows N due in destructive when overdue exists, in text.primary when only 'due today')". The flip is binary on `overdueCount`, not on the total count — a tile with only due-today items stays violet.
- **BentoGrid row arrangement: Row 3 = Streak + Goals (Squad Challenges).** Plan 04 PATTERNS.md showed the same arrangement; UI-SPEC §Component Inventory confirms it. Squad Challenges (GoalsTile) placeholder explicitly preserved per D-15.
- **Reanimated jest mock extended in this plan (Rule 3 deviation, see below).** The existing stub lacked `useReducedMotion` and `default.View` — without them, any test rendering a tile through `BentoCard` errors immediately. Fix is scoped to the mock file, doesn't touch production code, and unlocks every Bento-component test going forward.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 — Blocker] Reanimated jest mock missing `useReducedMotion` and `default.View`/`default.Text`**
- **Found during:** Task 2 (running `npx jest --testPathPatterns="HabitsTile|TodosTile"` after creating the two tile components).
- **Issue:** `BentoCard.tsx:42` calls `useReducedMotion()` from `react-native-reanimated`, and `BentoCard.tsx:103` renders `<Animated.View>` from the default export. The existing mock at `src/__mocks__/reanimated.js` only exposed `useSharedValue` / `useAnimatedStyle` / `withSpring` / `withTiming` — `useReducedMotion` was `undefined`, causing a `TypeError: (0, _reactNativeReanimated.useReducedMotion) is not a function`. The default export was `{}`, so `Animated.View` resolved to `undefined`, causing React renderer to fail with "Element type is invalid". Without the fix, every test that renders a tile through `BentoCard` (i.e., all 9 new tests in this plan and any future Bento-component tests) would fail.
- **Fix:** Added `useReducedMotion: jest.fn(() => false)` as a named export. Added `default` export with `View: 'Animated.View'`, `Text: 'Animated.Text'`, `Image: 'Animated.Image'`, `ScrollView: 'Animated.ScrollView'`, `createAnimatedComponent: (C) => C` — mirrors the host-string approach already used in `src/__mocks__/react-native.js` for `Animated.View`. Also re-exposed `View` / `Text` as named exports for completeness.
- **Files modified:** `src/__mocks__/reanimated.js`
- **Verification:** All 8 HabitsTile + TodosTile tests pass GREEN after the fix; pre-existing 43 tests across the rest of the project remain GREEN (`npx jest --no-coverage` → 14 suites, 52 tests, all passed).
- **Committed in:** `1a13544` (Task 2 commit — bundled with the tile components because the mock fix unblocks the same test run).
