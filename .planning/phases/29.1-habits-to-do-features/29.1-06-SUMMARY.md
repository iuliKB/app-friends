## Decisions Made

- **ChatTodoListRow uses controlled-prop expansion exclusively** — `items.length > 0 || loadingItems` is the sole truth for `expanded`. No internal `useState` for expansion. The plan-text §<action> calls this pattern out explicitly (`Component does NOT maintain internal expanded state — items.length > 0 is the sole source of truth`) per W9 from the checker pass. Benefits: (a) parent owns the items cache, so screen-focus refetches and toggle-toggle-toggle don't trigger stale state; (b) the test re-render pattern in the existing test file just works without an `act()`-wrapped imperative expand call.
- **Detail route disambiguation: try Mine first, fall back to chat-origin** — alternative was a `?source=mine|chat` URL param at the section-row tap level. Rejected because (a) `id` is a UUID either way (no collision risk on a single-shot fetch), (b) RLS silently filters non-owned rows so the first lookup is safe and bounded, (c) the deep-link surface is one route per to-do id which is cleaner for future shareable URLs.
- **Chat-origin Mark done is one-way in v1** — un-completing a chat-origin item is not supported. The RPC `complete_chat_todo` is idempotent (server returns NULL if already complete) and there's no `uncomplete_chat_todo` counterpart. Plan §<action> spec at the detail screen says "Mark done becomes ✓ Done after tap" without defining the reverse. Mine items can flip freely via direct UPDATE on `completed_at`. Future v1.9+ may add an un-complete path (D-10 surface) but it's out of scope here.
- **TodoQuickAdd lives inside the Mine section header**, not floating at screen bottom. Matches UI-SPEC empty copy `Nothing on your list. Quick-add with the + below.` which implies a visible `+` affordance directly under the Mine section header. Trade-off: when Mine is collapsed (user has tapped to hide), TodoQuickAdd is also hidden — acceptable because the FAB `+ New to-do` covers the "always-available creation" path.
- **Section data uses `data: []` for collapsed sections, not omitting the section** — keeps the section header row visible (so the user can re-expand) while hiding the rows underneath. Alternative (omit collapsed sections from the sections array) was rejected because it would also drop the header and make re-expansion impossible without a separate toggle bar.
- **`(supabase as any)` cast applied at every new RPC + table-query call site** — five new sites in this plan: `create_todo` (quick-add + create.tsx), `complete_chat_todo` ([id].tsx), `todos` UPDATE/DELETE ([id].tsx), `chat_todo_items` SELECT (index.tsx lazy-fetch + chat-item refresh), `chat_todo_lists+group_channels` join ([id].tsx). Same anticipated pattern from Plan 03 + Plan 05; `database.ts` regeneration explicitly deferred per Plan 01 SUMMARY (migration 0024 still local-only).
