# Phase 29.1 — UI Design Contract: Habits & To-Dos

> Visual and interaction contract for adding two new social features (Habits + To-Dos) across Squad → Activity Bento, full-screen routes, chat attachment, chat bubble variant, system message, and home widgets. Generated by gsd-ui-researcher; verified by gsd-ui-checker.

**Stack reality check:** This is a React Native + Expo project with a mature in-house token-driven design system (`src/theme/`). There is no shadcn/Tailwind/Radix. shadcn gate not applicable. All decisions below extend existing tokens and clone existing component shapes — no new visual language is introduced.

## Design System

| Property | Value |

## Typography

Project tokens (`src/theme/typography.ts`). This phase declares **4 size roles + 2 weight roles** as required by the contract — concretely sourced from existing tokens.

| Role | Token | Size (px) | Family / Weight | Line height | Usage in this phase |
|------|-------|-----------|-----------------|-------------|---------------------|
| Eyebrow / micro | `FONT_SIZE.xs` | 12 | `body.bold` (700), uppercase, `letterSpacing: 1.2` | 1.2 | Bento eyebrow pill labels ("HABITS", "TO-DOS"), system message text, chat to-do bubble due timestamp |
| Body / row label | `FONT_SIZE.md` | 14 | `body.medium` (500) for label, `body.regular` (400) for caption | 1.5 | Habit row title, to-do row title, chat to-do bubble title, two-step picker step labels |
| Title / hero secondary | `FONT_SIZE.lg` | 16 | `display.semibold` (600) | 1.4 | Bento tile titles ("Habits" / "To-Dos"), full-screen route headers (delegated to `<ScreenHeader>`), section headers ("Mine" / "From chats") |
| Hero metric | `FONT_SIZE.display` | 32 | `display.semibold` (600), `letterSpacing: -0.5` | 1.0 | Tile hero metric ("3/5", "7"), spotlight amount when habit/to-do is the spotlight item |

**Allowed weights for this phase (exactly 2):**
- `regular` (400) — captions, secondary text, system message body
- `semibold` (600) — titles, hero metrics, tile labels, eyebrow text, CTA labels

**Forbidden in this phase:** `display.extrabold` (reserved for `YourZoneSection` Streak hero), `FONT_SIZE.hero` 44px (reserved for Spotlight IOU amount), bold (700) for body text outside eyebrow pills.

**Line-height rules:**
- Body and row text: `1.5` (multiply size by 1.5, round to nearest int)
- Hero metrics: `1.0` (tight, character-height equals lineHeight)
- Titles: `1.4`

## Color

Two surfaces coexist in this phase and **must remain visually distinct** (Pitfall 7 in RESEARCH.md):

### Surface A — Squad → Activity Bento tiles (charcoal gradient + per-tool accent)
Source: `src/components/squad/bento/tileSurface.ts` + `tileAccents.ts`. Lit-from-above gradient on hairline-bordered cards.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `colors.surface.base` (`#0E0F11` dark / `#F5F6F8` light) | Page background behind the grid |
| Secondary (30%) | `TILE_SURFACE.tileGradient` (`['#23262E', '#1A1D24']`) on hairline border `rgba(255,255,255,0.06)` | All 6 Bento tile surfaces, including Habits + To-Dos |
| Accent (10%) — **per-tile, NOT global** | `TILE_ACCENTS.habits` = `#22D3EE` (cyan-400) · `TILE_ACCENTS.todos` = `#A78BFA` (violet-400) — **NEW additions to `tileAccents.ts`** | Icon bubble bg (`accent + '1F'` = 12% alpha) · icon glyph stroke · hero metric color when `> 0` · spotlight halo when habit/to-do is selected |
| Destructive | `colors.interactive.destructive` (`#F87171` dark / `#DC2626` light) | Overdue to-do count when `> 0` (To-Dos tile shows `N due` in destructive when overdue exists, in `text.primary` when only "due today") · "Delete habit" / "Delete to-do" confirmation copy |

**Why cyan + violet:**
- Existing accents: green (IOU positive), red (IOU negative), pink (Birthday), orange (Streak), blue (Goals/Spotlight neutral), neon green (brand). Cyan-400 (`#22D3EE`) and violet-400 (`#A78BFA`) are the next two perceptually-distinct hues that don't collide on a 6-tile grid and stay on-trend with the existing 400-weight Tailwind palette source. Cyan = "checked / habitual / clear" semantic. Violet = "task / agenda / list" semantic. Both pass WCAG AA contrast at 12% alpha against `#1A1D24`.

**Accent reserved for (explicit list — never "all interactive elements"):**
- Habits icon bubble background + icon glyph (`TILE_ACCENTS.habits`)
- Habits hero metric numerator color when `doneToday > 0` (numerator only; `/Y` denominator stays in `text.secondary`)
- Habits row "done today" check glyph (filled circle + check)
- Habits widget on home screen — accent color used ONLY for the check glyph; eyebrow pill stays neon-green (Phase 29 lock)
- To-Dos icon bubble background + icon glyph (`TILE_ACCENTS.todos`)
- To-Dos hero metric color when `total > 0` AND no overdue
- Chat to-do bubble checked-state border + check glyph (uses `TILE_ACCENTS.todos`)
- Spotlight halo color when `selectSpotlight()` returns a habit-urgent or to-do-urgent item (uses the matching accent)

### Surface B — Home widgets (neon-green Phase 29 aesthetic)
Source: `src/components/home/YourZoneSection.tsx`. Flat charcoal cards with neon-green eyebrow pills.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `colors.surface.base` | Home page background |
| Secondary (30%) | `colors.surface.card` (`#1D2027` dark / `#FFFFFF` light) on `colors.border` | Home Habits widget tile + Home To-Dos widget tile |
| Accent (10%) | `colors.interactive.accent` — neon green (`#B9FF3B` dark / `#4D7C00` light) | EyebrowPill icon + label, hero numerals when "hot", check glyphs on widget rows |
| Destructive | `colors.interactive.destructive` | Overdue to-do count badge in widget |

**Critical rule (Pitfall 7):** Home Habits widget and Home To-Dos widget MUST use the `TileShell + EyebrowPill` pattern (cloned/factored from `YourZoneSection.tsx:135-203`). They MUST NOT import `BentoCard`, `tileSurface`, or `tileAccents`. The home palette stays single-accent neon-green; the per-tool cyan/violet accents do NOT cross over to the home surface.

### Other UI elements

| Element | Color |
|---------|-------|
| Two-step picker sheet background | `colors.surface.card` |
| Two-step picker backdrop | `colors.overlay` (`rgba(0,0,0,0.5)`) |
| Solo/Group chip on habit row | Solo = `colors.text.secondary` border + bg `colors.surface.overlay`; Group = `colors.interactive.accent` border + bg `accent + '1F'` |
| Priority chip on to-do row | low = `colors.text.secondary`; medium = `colors.feedback.info`; high = `colors.interactive.destructive` |
| Overdue treatment (Mine + From chats rows) | Title color stays `text.primary`; left border-left `3px` `colors.interactive.destructive`; due-date label flips to `colors.interactive.destructive` with prefix "OVERDUE · " |
| System message ("✓ Sam completed Buy bread") | Centered, `colors.text.secondary`, `FONT_SIZE.xs`, `body.regular`, italic; check glyph `colors.interactive.accent`; no avatar, no bubble background, no long-press menu |
| Skeleton bars | `colors.border` (matches existing tile skeletons in `IOUTile`/`StreakTile`) |

### Spotlight extension (D-16)
| Spotlight kind | Title format | Subtitle | CTA | Accent |
|----------------|--------------|----------|-----|--------|
| `habit-urgent` (about to break streak) | `{habit title}` | `One more check-in to keep it going` | `Mark done` | `TILE_ACCENTS.habits` |

## Accessibility Contract

| Element | Requirement |
|---------|------------|
| All Bento tiles | `accessibilityRole="button"` + descriptive `accessibilityLabel` (e.g. `"Habits. 3 of 5 done today. Tap to open."`) — clones `StreakTile`/`IOUTile` precedent |
| Done-toggle / checkbox | `accessibilityRole="checkbox"` + `accessibilityState={{ checked }}` + label per copywriting contract |

| System message | `accessibilityRole="text"` + `accessibilityLabel="System: {body}"` — explicitly announce as system, not user-said |
| Two-step picker | Sheet has `accessibilityViewIsModal={true}` + focuses Step 1 heading on open; back-arrow has label `"Back to step 1"` |
| Tap targets | Min 44×44pt — apply `hitSlop` where visual is smaller (e.g. checkboxes inside dense rows) |
| Color is never the sole signal | Overdue treatment uses both color (destructive) AND text prefix (`"OVERDUE · "`); group-done state uses both color (accent green) AND copy (`"All done!"`) |
| Reduce-motion | All animations have a no-motion fallback; spring snaps to final value, LayoutAnimation skipped |
| Dynamic Type / large fonts | All `FONT_SIZE` tokens scale via system; tile heights use `minHeight` not `height` to allow growth (Bento tiles already do this) |
