## Spacing Scale

Declared values from `src/theme/spacing.ts` — no raw numbers permitted in any Phase 29 component:

| Token | Value | Phase 29 Usage |
|-------|-------|----------------|
| `SPACING.xs` | 4px | Avatar overlap offsets, date pill internal padding, inline gaps |
| `SPACING.sm` | 8px | Card gap between events, separator heights |
| `SPACING.md` | 12px | Gap between event cards in FlatList (`CARD_GAP`) |
| `SPACING.lg` | 16px | Card internal padding, screen horizontal padding, list left offset |
| `SPACING.xl` | 24px | Section spacing, EmptyState CTA margin-top |
| `SPACING.xxl` | 32px | EmptyState horizontal padding |

Exceptions:
- Touch targets for DEAD bubbles: non-interactive (no `Pressable` wrapper). Existing `hitSlop` on small ALIVE/FADING bubbles (`targetSize < 44 ? { top: 4, bottom: 4, left: 4, right: 4 }`) is preserved.
- Date pill height: not a SPACING token — pill is sized by its text (`FONT_SIZE.xs` = 12px) + 4px vertical padding each side = ~20px tall. Acceptable exception for inline pill.

**Accent (`colors.interactive.accent`) is reserved for:**
- "See all" link in the Upcoming Events section header
- Date pill background on EventCard (Claude's discretion: use `colors.interactive.accent` at 15% opacity as pill background, accent text at full opacity — provides theme-aware highlight without burning the full neon in light mode)
- `PrimaryButton` background in the EmptyState CTA

Accent is NOT used for: radar bubble colors, event card backgrounds, name labels, or any general interactive highlight.

**DEAD bubble color treatment (D-01, Claude's discretion):**
- Outer `Animated.View` opacity: `0.38` (vs. current `0.5` — refined for clearer DEAD vs. FADING distinction; FADING stays at `0.6`)
- Greyscale simulation: wrap `AvatarCircle` in a `View` with `position: 'absolute'`, render a `View` overlay with `backgroundColor: colors.surface.base` at `opacity: 0.55` and `borderRadius: targetSize / 2` — desaturates appearance without CSS filter. Remove `LinearGradient` and `PulseRing` for DEAD.
- Name label: `colors.text.secondary` (same as FADING) — consistent dimming signal

## Interaction Contracts

### DEAD Bubble (HOME-05)

- Non-interactive: no `Pressable` wrapper, no `onPress`, no `onLongPress`
- No accessibility role or label (inaccessible by intent — DEAD friend is visually suppressed)
- Visual state: opacity 0.38, greyscale overlay, no gradient, no pulse ring
- Same bubble size as current DEAD: `BubbleSizeMap.dead` = 48px
- Name label: `colors.text.secondary`, `numberOfLines={1}`

### ALIVE Bubble (HOME-05, unchanged)

- Green `PulseRing` (variant `'alive'`), 1200ms cycle, scale 1.7
- Status gradient overlay via `LinearGradient`
- `Pressable` with `onPress` (DM) and `onLongPress` (action sheet)

### FADING Bubble (HOME-05, unchanged)

- Amber `PulseRing` (variant `'fading'`, color `FADING_PULSE_COLOR = #F59E0B`), 2000ms cycle, scale 1.5
- Outer opacity: 0.6
- No gradient
- `Pressable` active (DM + action sheet available)
- Name label: `colors.text.secondary`

### EventCard (HOME-08)

- Dimensions: **240×160px** (`CARD_WIDTH = 240`, FlatList `style.height = 160`)
- `CARD_GAP = SPACING.md` (12px) — unchanged
- Snap interval: `CARD_WIDTH + CARD_GAP = 252px`
- Date pill: positioned `top: SPACING.sm` (8px), `left: SPACING.sm` (8px), absolute within card
  - Background: `colors.interactive.accent` at 15% opacity (`rgba(185,255,59,0.15)` dark / `rgba(77,124,0,0.12)` light)
  - Text: `colors.interactive.accent` at full opacity
  - Font: `FONT_SIZE.xs` (12px), `FONT_FAMILY.body.medium` (500), `lineHeight: 12`
  - Padding: `SPACING.xs` (4px) vertical, `SPACING.sm` (8px) horizontal
  - Border radius: `RADII.sm` (6px)
  - Content: formatted date string via existing `formatEventCardDate()`, e.g. "Thu May 8"
- AvatarStack: `size={28}`, `maxVisible={5}`, add `+N` overflow chip when member count > 5
- Press feedback: spring scale 1.0→0.96 (existing, unchanged)
- Image overlay: `rgba(0,0,0,0.40)` (existing, unchanged)
- Text color on pastel card: `#1a1a1a` (existing, unchanged)

### Events Loading Skeleton (HOME-08, D-09)

- Render 2 `SkeletonPulse` placeholders at 240×160px, `borderRadius: RADII.xl` (16)
- Layout: horizontal row with `CARD_GAP` (12px) between, left-aligned with `paddingLeft: SPACING.lg`
- Shown while `useUpcomingEvents` is in loading state (expose `isLoading` from hook)
- Transition: skeleton fades out when data arrives (Animated.timing, duration `ANIMATION.duration.normal` = 300ms)

### Zero-Friends EmptyState (HOME-07)

- Icon: emoji "👥" (people group)
- Heading: "Invite your crew"
- Body: "Add friends to see who's free and make plans"
- CTA label: "Invite friends"
- CTA action: navigate to Add Friend screen (`/friends/add`)
- Component: existing `EmptyState` — pass updated props only
- Remove: `OnboardingHintSheet` render and its associated `useState`/`useEffect` in `HomeScreen.tsx`

### View Mode Persistence (HOME-06)

- No visual change — `useViewPreference.ts` already persists via `campfire:home_view` AsyncStorage key
- Plan task: verify behavior + add unit test only
