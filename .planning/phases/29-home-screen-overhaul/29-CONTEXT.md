# Phase 29: Home Screen Overhaul - Context

**Gathered:** 2026-05-06
**Status:** Ready for UI design contract (/gsd-ui-phase 29)

<domain>
## Phase Boundary

Ground-up visual redesign of the Home screen: radar bubble freshness clarity (ALIVE/FADING/DEAD distinction), view-mode persistence verification, zero-friends CTA redesign, and upcoming events section polish.

Requirements: HOME-05, HOME-06, HOME-07, HOME-08

**NOT in scope:** New sections or widgets, changes to CardStackView gesture physics, Squad/Explore/Auth screens (those are Phases 30–32).

</domain>

<decisions>
## Implementation Decisions

### Radar Freshness (HOME-05)

- **D-01:** DEAD friends render greyscale + ~40% opacity in-place within the radar scatter — no position change, same bubble size. No pulse ring. Desaturate using a greyscale filter or by removing the status gradient entirely.
- **D-02:** DEAD bubbles are non-interactive — no tap/long-press response. No action sheet, no DM entry point.
- **D-03:** FADING treatment stays as-is — amber PulseRing variant, same size as ALIVE. No additional dimming needed.
- **D-04:** ALIVE treatment stays as-is — green PulseRing, status gradient overlay.

### View Mode Persistence (HOME-06)

- **D-05:** `useViewPreference.ts` already persists to AsyncStorage (`campfire:home_view`) — this requirement is functionally complete. Plan should verify the behavior and add a unit test, not implement it from scratch.

### Zero-Friends CTA (HOME-07)

- **D-06:** Update the existing `EmptyState` component render — change copy to "Invite your crew" (heading) / "Add friends to see who's free and make plans" (body). CTA label: "Invite friends".
- **D-07:** CTA routes directly to the Add Friend screen (not the Squad tab root).
- **D-08:** Retire the `OnboardingHintSheet` — it duplicates the message. Remove its render and the `@campfire/onboarding_hint_shown` AsyncStorage flag check from `HomeScreen.tsx`.

### Events Section Polish (HOME-08)

- **D-09:** Add a loading skeleton: show 2–3 shimmer placeholder cards (using existing `SkeletonPulse` component) while plans are loading. No more silent wait.
- **D-10:** Increase `EventCard` size from 200×140px to **240×160px**. Update both the card dimensions and the `FlatList` explicit height in `UpcomingEventsSection`.
- **D-11:** Add a **date pill** anchored top-left on the card — small colored pill showing formatted date (e.g. "Thu May 8"). Date leads the card, visually more prominent than the title.
- **D-12:** Larger participant avatars — increase from 24px to 28–30px, show up to 4–5 avatars, add a count chip for overflow (e.g. "+2").

### Design Approach

- **D-13:** Run `/gsd-ui-phase 29` using the `/ui-ux-pro-max` plugin to generate `UI-SPEC.md` before planning. The planner must consume the design contract for all visual decisions.

### Claude's Discretion

- Date pill color token: use `colors.interactive.accent` or a muted surface — Claude picks what fits the light/dark theme.
- Exact greyscale/opacity values for DEAD bubbles: Claude picks within the spirit of ~40% opacity and full desaturation.
- Skeleton card shape: Claude reuses `SkeletonPulse` in a layout that mirrors the new 240×160px card proportions.

</decisions>

- `SkeletonPulse` (`src/components/common/SkeletonPulse.tsx`): existing shimmer component — use for events loading skeleton (D-09)
- `EmptyState` (`src/components/common/EmptyState.tsx`): has `icon`, `heading`, `body`, `ctaLabel`, `onCta` props — update props only, no new component needed (D-06/D-07)

- `PulseRing` (internal to `RadarBubble.tsx`): already has `variant: 'alive' | 'fading'` — no new variant needed for DEAD (D-01 is purely visual CSS-level change)

### Established Patterns
- `useTheme()` + `useMemo([colors])` wrapping `StyleSheet.create`: mandatory for all themed styles
- `computeHeartbeatState()` from `src/lib/heartbeat`: returns `ALIVE | FADING | DEAD` — use this to gate DEAD styling in `RadarBubble`
- Design token imports: `SPACING`, `FONT_SIZE`, `FONT_FAMILY`, `RADII`, `SHADOWS`, `ANIMATION` — no raw values

### Integration Points
- `RadarBubble.tsx`: add DEAD branch in the render — when `computeHeartbeatState()` returns `DEAD`, apply greyscale filter + reduced opacity + skip `Pressable` wrapper (or disable `onPress`)
- `HomeScreen.tsx`: remove `OnboardingHintSheet` render + its `useState`/`useEffect` (D-08)
- `UpcomingEventsSection.tsx`: update `CARD_WIDTH`, `flatList` height, and add skeleton branch when loading (D-09/D-10)
- `EventCard.tsx`: add date pill, increase dimensions, update `AvatarStack` props (D-10/D-11/D-12)
