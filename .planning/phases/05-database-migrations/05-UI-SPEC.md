# Phase 5 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Scope Note

Phase 5 is a backend-only migration phase. No new screens, flows, or interactive components are introduced. The sole client-visible change is renaming the `iou_notes` field to `general_notes` in the plans UI — existing label copy, component layout, and behavior are preserved exactly. All design values below document the existing project token system so downstream phases (6-10) consume it correctly.

## Spacing Scale

Declared values from `src/theme/spacing.ts`:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Avatar overlap offset, tight gaps |
| sm | 8px | Separator height, inline gaps, icon-label gap |
| md | 12px | Card section gaps, separator marginLeft |
| lg | 16px | Standard screen padding, card padding |
| xl | 24px | Large section spacing, button-from-form gap, FAB bottom offset |
| xxl | 32px | Screen header top, auth header padding |

Exceptions: none for this phase. Touch targets must be minimum 44px height (iOS HIG) — achieved via `minHeight: 44` on pressable rows, not via a spacing token.

## Typography

From `src/theme/typography.ts`:

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px (lg) | 400 (regular) | 1.5 |
| Label / secondary info | 14px (md) | 400 (regular) | 1.4 |
| Caption / timestamp | 13px (sm) | 400 (regular) | 1.3 |
| Section heading | 20px (xl) | 600 (semibold) | 1.2 |

Notes:
- `FONT_WEIGHT.bold` (700) is not in the declared token set; use `FONT_WEIGHT.semibold` (600) for all emphasis in this phase. If bold is required, use hardcoded `'700'` with `// eslint-disable-next-line` per established project convention (STATE.md Phase 01.1 decision).
- No new display-size (24px) headings in this phase — no new screens.

## Color

From `src/theme/colors.ts`:

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#1a1a1a` (`surface.base`) | App background |
| Secondary (30%) | `#2a2a2a` (`surface.card`) | Cards, inputs, tab bar |
| Accent (10%) | `#f97316` (`interactive.accent`) | See reserved list below |
| Destructive | `#ef4444` (`interactive.destructive`) | Destructive actions only |

Accent (`#f97316`) reserved for:
- Primary CTA buttons (Save, Create)
- Active toggle/segment indicator
- Highlighted stat values (streak count)

Accent is NOT used for: labels, body copy, informational icons, borders, or passive state indicators.

For the `iou_notes → general_notes` rename: the `IOUNotesField` component label changes from "IOU Notes" to "Notes" (see Copywriting Contract). No color changes required.

## Copywriting Contract

This phase has one client-visible copy change: the `plans.iou_notes` field is renamed to `plans.general_notes`. The label in `IOUNotesField.tsx` must be updated to remove the "IOU" qualifier.

| Element | Copy |
|---------|------|
| Field label (IOUNotesField) | "Notes" (was: "IOU Notes") |
| Field placeholder | "Add notes for this plan..." |
| Primary CTA | Not applicable — no new CTAs in this phase |
| Empty state heading | Not applicable — no new screens |
| Empty state body | Not applicable — no new screens |
| Error state | Not applicable — no new screens |
| Destructive confirmation | Not applicable — no destructive actions in this phase |

Rationale: "Notes" is unambiguous once `iou_notes` is gone. "General Notes" would be redundant — the field lives inside a plan context where "notes" is self-describing. Source: CONTEXT.md D-02.

## Downstream Phase Baseline

Phases 6-10 must inherit these token values when building new screens. This section provides the reference anchors:

- Background surfaces: `COLORS.surface.base` (#1a1a1a), `COLORS.surface.card` (#2a2a2a)
- Primary text: `COLORS.text.primary` (#f5f5f5)
- Secondary text: `COLORS.text.secondary` (#9ca3af)
- Borders: `COLORS.border` (#3f3f46)
- Accent CTA: `COLORS.interactive.accent` (#f97316)
- Destructive: `COLORS.interactive.destructive` (#ef4444)
- Status indicators: free (#22c55e), busy (#ef4444), maybe (#eab308)
- Standard screen padding: `SPACING.lg` (16px)
- Card internal padding: `SPACING.lg` (16px)
- Section gaps: `SPACING.xl` (24px)
- Body copy: `FONT_SIZE.lg` (16px) at `FONT_WEIGHT.regular` (400)
- Section headings: `FONT_SIZE.xl` (20px) at `FONT_WEIGHT.semibold` (600)
