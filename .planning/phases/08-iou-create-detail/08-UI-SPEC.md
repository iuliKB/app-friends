## Screen Layouts

### Screen 1: Expense Create (`/squad/expenses/create`)

Structure (single ScrollView, no nested FlatList):

```
ScrollView (background: COLORS.surface.base, padding: SPACING.lg)
  ├── Section: "Expense title"
  │   └── FormField (placeholder: "What's this for?")
  ├── Section: "Total amount"
  │   └── TextInput currency-formatted (keyboardType: numeric, placeholder: "$0.00")
  ├── Section: "Split with" (label at 14px/semibold/secondary)
  │   └── FlatList of friends — each row: AvatarCircle + name + checkbox (Ionicons)
  │       (useFriends() checkbox pattern from PlanCreateModal)
  ├── Section: "Split mode"
  │   └── SplitModeControl (even / custom)
  ├── [Conditional: custom mode only]
  │   ├── Per-person TextInput rows (name + amount field)
  │   └── RemainingIndicator
  └── PrimaryButton "Create Expense" (disabled when: no title, amount = 0, no friends selected, custom mode remaining ≠ 0)
```

Keyboard behavior: ScrollView `keyboardShouldPersistTaps="handled"` to keep friend list tappable while keyboard is open.

### Screen 2: Expense Detail (`/squad/expenses/[id]`)

Structure (single ScrollView with ListHeaderComponent pattern to avoid FlatList-in-ScrollView):

```
ScrollView (background: COLORS.surface.base)
  ├── [Conditional: all settled] AllSettledBanner
  ├── ExpenseHeroCard (title, total, payer, date, split type)
  └── Participants section label ("Split between N people" at 14px/semibold/secondary)
      └── [for each participant] ParticipantRow
```

Loading state: Render `ExpenseHeroCardSkeleton` (grey boxes at opacity 0.5, `COLORS.border` background) in place of hero card + 3 `ParticipantRowSkeleton` placeholders. Matches BirthdayCard skeleton pattern.

## Interaction Contracts

### Currency input (amount field on create screen)

- State: `rawDigits: string` (only digit characters, no decimal point)
- Display: `formatCentsDisplay(parseInt(rawDigits || '0', 10))` — derived, never stored separately
- On change: `setRawDigits(text.replace(/\D/g, ''))`
- On submit: `totalCents = parseInt(rawDigits || '0', 10)`
- Placeholder: `"$0.00"` (shown when rawDigits is empty)
- Keyboard type: `numeric`

### Custom split amount fields (per-person)

- Each field follows same `rawDigits` pattern as the total amount field
- `RemainingIndicator` updates live on every keystroke
- Submit button `disabled` when remaining !== 0 (any non-zero remaining, positive or negative)

### Friend checkbox toggle

- `Set<string>` for selected IDs (PlanCreateModal pattern)
- `Ionicons name={selected ? 'checkbox' : 'square-outline'}` at size 20, color `COLORS.interactive.accent`
- Row `Pressable`, entire row is tappable (44px min-height)

### Settle action (detail screen)

1. Creator taps "Settle" button on a participant row
2. `settleLoading` state set to `true` on that row's button (spinner, button disabled)
3. `supabase.from('iou_members').update(...)` called
4. On success: `Haptics.impactAsync(ImpactFeedbackStyle.Medium).catch(() => {})` fires, row badge updates to "Settled" (green), settle button disappears
5. On error: `settleLoading` resets, row unchanged, inline error shown below row at 13px/destructive

### Submit (create expense)

1. PrimaryButton shows spinner, `disabled={true}`, label stays "Create Expense"
2. `create_expense()` RPC called (payer userId auto-included in participant IDs)
3. On success: `Haptics.impactAsync(ImpactFeedbackStyle.Medium).catch(() => {})` fires, navigate to `/squad/expenses/[newId]`
4. On error: spinner stops, button re-enables, form state preserved (do not clear), inline error shown below button
