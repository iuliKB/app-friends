From src/hooks/useUpcomingBirthdays.ts (hook pattern to replicate):
```typescript
import { useCallback, useEffect, useState } from 'react';
import { supabase } from '@/lib/supabase';
import { useAuthStore } from '@/stores/useAuthStore';

export function useUpcomingBirthdays(): UpcomingBirthdaysData {
  const session = useAuthStore((s) => s.session);
  const userId = session?.user?.id ?? null;
  const [entries, setEntries] = useState<BirthdayEntry[]>([]);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState<string | null>(null);

  const refetch = useCallback(async () => {
    if (!userId) { setLoading(false); setEntries([]); return; }
    setLoading(true);
    setError(null);
    const { data, error: rpcErr } = await supabase.rpc('get_upcoming_birthdays');
    if (rpcErr) {
      console.warn('...', rpcErr);
      setError(rpcErr.message);
    } else {
      setEntries((data ?? []) as BirthdayEntry[]);
    }
    setLoading(false);
  }, [userId]);

  useEffect(() => { refetch(); }, [refetch]);
  return { entries, loading, error, refetch };
}
```

      // Form state
      title: string;

      setRawDigits: (d: string) => void;

      // Submit state
      submitting: boolean;
      submitError: string | null;
      canSubmit: boolean;                     // title non-empty && totalCents > 0 && selectedFriendIds.size > 0 && (even OR remaining === 0)

      // Actions
      submit: () => Promise<void>;            // calls create_expense RPC, fires haptic, navigates to /squad/expenses/[id]

    Implementation notes:
    1. Use useAuthStore to get session.user.id (userId). Guard all Supabase calls with userId.
    2. Friends list: call supabase.from('profiles') ... actually use `supabase.rpc('get_friends')`
       to fetch accepted friends. Map result to FriendEntry[]. Load in a useEffect on mount.
       If get_friends fails, set friends=[] and friendsLoading=false (silent fail, log warn).

       a. Guard canSubmit (no-op if false)
       b. Build participantIds: Array.from(selectedFriendIds) — then auto-include payer:

          `splitMode === 'custom' ? ids.map(id => parseInt(customAmounts[id] || '0', 10)) : null`
          Note: ids order must match p_participant_ids order; customAmounts keyed by userId.
          For the payer's own slot (if auto-added): their customAmounts entry may be '0' — that is
          correct since the payer's share is tracked but may be $0 if they only paid for others.
          In even mode payer share is computed server-side via largest-remainder.

    Create src/hooks/useExpenseDetail.ts. This hook fetches an expense by ID (three
    sequential Supabase queries) and provides a settle() action.

    Export interfaces:
    ```typescript
    export interface ParticipantEntry {
      userId: string;
      displayName: string;
      avatarUrl: string | null;
      shareCents: number;
      isSettled: boolean;
      settleLoading: boolean;   // per-row loading state for settle button
    }

    export interface ExpenseDetail {
      id: string;
      title: string;
      totalCents: number;
      splitMode: 'even' | 'custom';
      createdBy: string;        // userId of expense creator
      payerName: string;        // display_name of creator
      createdAt: string;
      participants: ParticipantEntry[];
      allSettled: boolean;      // true when every participant.isSettled === true
    }

    export interface ExpenseDetailData {
      detail: ExpenseDetail | null;
      loading: boolean;
      error: string | null;
      refetch: () => Promise<void>;
      settle: (participantUserId: string) => Promise<void>;
      isCreator: boolean;       // current userId === detail.createdBy
    }
    ```

    Implementation notes:
    1. Hook signature: `export function useExpenseDetail(expenseId: string): ExpenseDetailData`
    2. Guard: `if (!userId || !expenseId) { setLoading(false); return; }`
    3. Fetch sequence (three queries — all in refetch callback):
       a. `const { data: group } = await supabase.from('iou_groups').select('*').eq('id', expenseId).single()`
       b. `const { data: members } = await supabase.from('iou_members').select('user_id, share_amount_cents, settled_at, settled_by').eq('iou_group_id', expenseId)`
       c. `const memberIds = (members ?? []).map(m => m.user_id); const { data: profiles } = await supabase.from('profiles').select('id, display_name, avatar_url').in('id', memberIds)`
       d. Build participants: map members, join profile by user_id. isSettled = settled_at !== null.
       e. payerName: profiles.find(p => p.id === group.created_by)?.display_name ?? 'Unknown'
       f. allSettled: participants.every(p => p.isSettled)
       g. Set all fields on detail state.
    4. Error handling: if any query errors, set error="Couldn't load expense. Pull down to refresh." and detail=null.
    5. settle(participantUserId):
       a. Guard: if (!userId || !detail) return
       b. Set per-row settleLoading=true (update participants array immutably)
       c. Call: `const { error } = await supabase.from('iou_members').update({ settled_at: new Date().toISOString(), settled_by: userId }).eq('iou_group_id', expenseId).eq('user_id', participantUserId)`
       d. On success: fire `Haptics.impactAsync(Haptics.ImpactFeedbackStyle.Medium).catch(() => {})`, call refetch() to reload fresh state.
       e. On error: set per-row settleLoading=false, set error="Couldn't mark as settled. Try again."
    6. isCreator: `userId === detail?.createdBy`

    CRITICAL — composite PK (Pitfall 3): settle() MUST use `.eq('iou_group_id', ...).eq('user_id', ...)` together.
    Never use `.eq('id', ...)` — iou_members has no `id` column.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Hook → Supabase RPC | create_expense() is the only path to insert expense data; two-step inserts are prohibited |
| Hook → iou_members UPDATE | settle() sends userId as settled_by; RLS UPDATE policy enforces creator-only on server |
| Client userId → participant array | payer auto-include is client convenience; RPC enforces nothing about the caller being included |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-P03-01 | Elevation of Privilege | useExpenseDetail.settle() | mitigate | Client hides settle button via isCreator check (UX); authoritative enforcement is Supabase RLS UPDATE policy `iou_members_update_creator_settles` (is_iou_group_creator) — server rejects non-creator silently (no rows updated) |
| T-08-P03-02 | Tampering | useExpenseCreate.submit() — p_custom_cents | mitigate | Submit is disabled client-side when allocatedCents !== totalCents (canSubmit guard); server-side: create_expense() RPC validates custom cents sum against total and raises exception on mismatch (verified in migration 0015) |
| T-08-P03-03 | Repudiation | settle() settled_by column | accept | settled_by = current userId is stored on the iou_members row — audit trail is implicit; no separate audit log needed for v1.4 |
| T-08-P03-04 | Denial of Service | useExpenseCreate.submit() — double tap | mitigate | submitting state set to true before RPC; canSubmit evaluated with submitting guard — button disabled during in-flight call |
</threat_model>
