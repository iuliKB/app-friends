## Spacing Scale

All values from `src/theme/spacing.ts` (confirmed in codebase):

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| SPACING.xs | 4px | Inline gaps, tight offsets (e.g., clear button margin) |
| SPACING.sm | 8px | Char count margin-top, dropdown option row vertical padding |
| SPACING.md | 12px | Gap between Month and Day dropdowns |
| SPACING.lg | 16px | Screen horizontal padding, dropdown trigger paddingHorizontal |
| SPACING.xl | 24px | Section gap between birthday field and Save button |
| SPACING.xxl | 32px | Screen paddingTop / paddingBottom (matches edit.tsx) |

Exception — SPACING.md (12px): This is a pre-existing token in `src/theme/spacing.ts`, not introduced by this spec. It is used as the gap between the Month and Day dropdown triggers to create a tighter paired-input visual grouping than SPACING.lg (16px) would allow. The two dropdowns read as one compound field at 12px; at 16px they visually separate into independent fields.

Exception — Dropdown trigger height (52px): Matches existing TextInput height in edit.tsx to preserve visual alignment with the display name field above. Not a SPACING token.

Touch targets: All dropdown triggers are 52px tall, exceeding the 44px minimum touch target requirement.

## Layout Contract

### Profile Edit Screen — Field Order

```
[ ScreenHeader: "Edit Profile" ]

[ Avatar section — centered, 80px AvatarCircle ]
[ "Change Photo" link — accent color ]

[ Display Name TextInput — height 52, COLORS.surface.card, RADII.lg ]
[ Char count — right-aligned, 12px, COLORS.text.secondary ]

[ "Birthday" field label — FONT_SIZE.md, COLORS.text.secondary ]
[ BirthdayPicker — two dropdowns side by side ]        <-- FOCAL POINT
  [ Month dropdown trigger — flex 1 ]   [ gap: SPACING.md ]   [ Day dropdown trigger — flex 1 ]
[ "Clear Birthday" link — shown only if both month and day are set ]

[ Save Changes button — marginTop: SPACING.xl ]
```

Focal point: `BirthdayPicker` is the primary new UI element on this screen. It occupies the full content width as a two-column compound input and is the only new interactive element introduced in this phase. All supporting elements (label, clear link) serve to frame it.

> Source: CONTEXT.md §D-03 — "Birthday field placed below the display name TextInput, above the Save button. Natural flow: avatar → name → birthday → save. No section dividers needed."

### BirthdayPicker Trigger Dimensions

- Height: 52px (matches existing TextInput height — visual alignment)
- Background: COLORS.surface.card
- Border radius: RADII.lg (12px)
- Padding horizontal: SPACING.lg (16px)
- Layout: `flexDirection: 'row'` with two triggers, `gap: SPACING.md`

### Dropdown Modal Sheet

- Follows FriendActionSheet pattern exactly: Modal + TouchableWithoutFeedback backdrop + Animated.View sliding up from translateY=300 to 0
- Sheet background: COLORS.surface.card
- Sheet border radius top: RADII.lg (12px)
- Max height: 300px (covers ~8 visible options before scroll)
- Option row height: 48px (exceeds 44px touch target minimum)
- Option text: FONT_SIZE.lg (16px), FONT_WEIGHT.regular (400), COLORS.text.primary
- Selected option: FONT_WEIGHT.semibold (600) weight to indicate selection

> Source: RESEARCH.md §Animated Bottom Sheet Modal (FriendActionSheet pattern).

## Interaction Contract

### Dropdown Trigger States

| State | Visual |
|-------|--------|
| Empty (no selection) | Placeholder text "Month" / "Day" at COLORS.text.secondary |
| Value selected | Selected text at COLORS.text.primary, FONT_WEIGHT.regular |
| Disabled (saving=true) | opacity 0.5 (matches inputDisabled pattern in edit.tsx) |

### Dropdown Open/Close

- Tap trigger: open corresponding modal (month OR day, not both)
- Tap backdrop: dismiss modal, keep current selection
- Tap option row: select value, dismiss modal immediately
- Animation: translateY 300→0 open (250ms), setValue(300) on close (no exit animation — matches FriendActionSheet)

### Month Change Side Effect

When month changes AND current day > max days for new month: reset day to null (forces re-selection). Do NOT clamp silently.

> Source: RESEARCH.md §Pitfall 2 — reset to null rather than clamp to prevent invalid save.

### Month Display Format

Use abbreviated month names (3 letters): Jan, Feb, Mar, Apr, May, Jun, Jul, Aug, Sep, Oct, Nov, Dec.

`accessibilityLabel` uses full names: January, February, etc.

> Source: RESEARCH.md §Birthday Dropdown Display — "Abbreviated month names fit in a side-by-side layout without truncation on a 375px screen". CONTEXT.md §Claude's Discretion — pick what fits layout best.

### Clear Birthday

Show a small "Clear Birthday" text link below the dropdown row ONLY when both month and day are non-null. Tapping sends `onChange(null, null)`. Styled as: FONT_SIZE.sm (13px), COLORS.interactive.accent, textAlign right, marginTop SPACING.xs. accessibilityLabel: "Clear birthday".

> Source: RESEARCH.md §Open Questions — "Add a small 'Clear' link/button below the dropdowns if both fields are set."

### Keyboard Handling

Call `Keyboard.dismiss()` before opening any dropdown modal.

> Source: RESEARCH.md §Pitfall 6 — Android keyboard z-order issue.

### isDirty Extension

Save button is enabled when:
- `displayName.trim().length > 0` (existing gate — no change)
- AND at least one field differs from its original loaded value
- Birthday counts as dirty if `birthdayMonth !== originalBirthdayMonth || birthdayDay !== originalBirthdayDay`

### Partial Birthday on Save

If exactly one of (month, day) is set when Save is tapped: treat as "no birthday" — save both fields as null. Do NOT save a half-filled birthday.

> Source: RESEARCH.md §Pitfall 5.

### Feb 29 Normalization

The dropdown shows Feb with days 1–29. If user selects Feb 29, the dropdown shows "Feb 29" as selected. At save time (handleSave), normalize: if month=2 and day=29, save day=28 instead. No user-visible indication of normalization.

> Source: CONTEXT.md §D-02, RESEARCH.md §Pattern 2.

## Accessibility Contract

| Requirement | Implementation |
|-------------|---------------|
| Touch targets ≥ 44px | Dropdown triggers: 52px height. Option rows: 48px height. |
| accessibilityLabel on dropdowns | Month trigger: "Select birth month, currently {name or 'not set'}". Day trigger: "Select birth day, currently {number or 'not set'}". |
| accessibilityLabel on Clear link | "Clear birthday" |
| Placeholder contrast | COLORS.text.secondary (#9ca3af) on COLORS.surface.card (#2a2a2a) — verify passes 3:1 minimum for placeholder text (WCAG AA large text threshold) |
| Disabled state | `disabled={saving}` prop on BirthdayPicker. Opacity 0.5 visual indicator. |
| No color-only indicators | Selected state communicated via font weight change (semibold) in addition to position |

> Source: ui-ux-pro-max skill §Accessibility (CRITICAL) — touch-target-size minimum 44x44px.
