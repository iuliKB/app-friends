   Add a small inline component above the `SquadScreen` function (after the `SCREEN_WIDTH` constant):

   ```tsx
   function MemoriesRedirect({ onNavigate }: { onNavigate: () => void }) {
     const { colors } = useTheme();
     // Navigate immediately when this page becomes visible.
     // We use a Pressable full-screen tap target so the user has a clear affordance
     // while the navigation fires. Using useEffect would fire before the pager
     // animation settles; a tap-to-open is more predictable.
     return (
       <View style={{ flex: 1, backgroundColor: colors.surface.base, justifyContent: 'center', alignItems: 'center' }}>
         <TouchableOpacity onPress={onNavigate} activeOpacity={0.7} accessibilityRole="button" accessibilityLabel="Open Memories">
           <Text style={{ fontSize: FONT_SIZE.lg, fontFamily: FONT_FAMILY.body.regular, color: colors.text.secondary }}>
             Open Memories
           </Text>
         </TouchableOpacity>
       </View>
     );
   }
   ```

   **Implementation note:** The "Open Memories" tap-target approach is intentional. The Squad pager uses a horizontal Animated.ScrollView; firing router.push inside a useEffect on the pager page causes a navigation during the swipe animation which can produce a flash. A visible tap target is instant, predictable, and avoids the animation race. The label uses `colors.text.secondary` to be visible in both light and dark themes without introducing new styles.

   No other changes to squad.tsx — the TABS array, tab header, pager, and all other pages stay as-is.
