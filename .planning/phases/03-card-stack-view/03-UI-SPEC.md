## Typography

Sourced from `src/theme/typography.ts`. This phase uses 4 of the 6 declared sizes and both declared weights.

| Role | Token | Size | Weight | Line Height | Usage |
|------|-------|------|--------|-------------|-------|
| Counter label | `FONT_SIZE.md` | 14px | `FONT_WEIGHT.regular` (400) | 1.4 | "2 more free" counter above deck |
| Card name | `FONT_SIZE.lg` | 16px | `FONT_WEIGHT.semibold` (600) | 1.3 | Friend display name on card |
| Card mood/context | `FONT_SIZE.md` | 14px | `FONT_WEIGHT.regular` (400) | 1.4 | Mood ┬Ę context tag line |
| Card last-active | `FONT_SIZE.sm` | 13px | `FONT_WEIGHT.regular` (400) | 1.4 | "2h ago", "just now" secondary text |
| Action button label | `FONT_SIZE.sm` | 13px | `FONT_WEIGHT.regular` (400) | 1.0 | "Skip" and "Nudge" labels under icons |

Notes:
- No heading-level text in this component ŌĆö it slots inside the HomeScreen scroll context
- Counter uses `COLORS.text.secondary` to remain subordinate to the deck
- **13px vs 14px adjacency (checker flag):** Both sizes are established tokens from `src/theme/typography.ts` (`sm` and `md`). The 1px difference is acceptable here because the two roles that use them (last-active/button labels at 13px, mood/context at 14px) are always visually separated by layout position and color (`text.secondary` vs `text.primary`), not solely by size. No hierarchy collapse risk in this specific context.

## Component Inventory

Components to build for this phase:

### CardStackView (container)
- Renders the deck of ALIVE+FADING friends
- Manages deck state: current index, history for undo, loop behavior
- Provides counter label above deck: `"{N} more free"` or `"Just you for now"`
- Width: 80% of screen width (D-01 ŌĆö breathing room around edges)
- Source: slots into `HomeScreen.tsx` line 127-130, replacing `cardsPlaceholder`

### FriendSwipeCard (single card)
- Avatar (64px `AvatarCircle`) left-aligned ŌĆö D-02
- Name (`FONT_SIZE.lg`, semibold) right of avatar
- Mood ┬Ę context tag (`FONT_SIZE.md`) right of avatar, below name
- Last-active time (`FONT_SIZE.sm`, secondary color) below mood line ŌĆö D-03
- Status gradient covering full card background ŌĆö D-04
- Border radius: `RADII.xl` (16px) ŌĆö D-05 ("more playful than RADII.lg")
- Card elevation: `shadowColor: COLORS.shadow` with shadowRadius 12, shadowOpacity 0.4, elevation 8

### StackDepthEffect (depth cards behind front)
- 2 depth cards visible behind front card ŌĆö D-06
- Depth card 1 (index+1): scale 0.95, translateY +8px, opacity 0.6
- Depth card 2 (index+2): scale 0.90, translateY +16px, opacity 0.35
- Background: `COLORS.surface.card` (no gradient on depth cards)

### ActionButtonRow (below deck)
- Two buttons: Skip (left) and Nudge (right)
- Layout: icon centered, small text label underneath ŌĆö D-15
- Skip button: 56px circle, `COLORS.surface.overlay` background, `Ō£Ģ` icon (`Ionicons: close`), `COLORS.text.secondary` color
- Nudge button: 56px circle, `COLORS.interactive.accent` background, chat icon (`Ionicons: chatbubble`), `COLORS.text.primary` color
- Minimum touch target: 44├Ś44px (both buttons exceed at 56px)
- Haptic feedback: `ImpactFeedbackStyle.Light` on both button presses ŌĆö existing pattern

## Interaction Contract

### Swipe Gesture
- Horizontal swipe only ŌĆö left or right both = Skip (D-07)
- Gesture handler: `react-native-gesture-handler` `Gesture.Pan` (D-08)
- Swipe threshold: 100px horizontal translation before commit
- Card rotation during drag: `rotation = translateX / 12` degrees (max ┬▒15┬░)
- Exit animation: fly off screen in swipe direction with final rotation of ┬▒20┬░ ŌĆö D-09
  - Duration: 250ms, `Easing.out(Easing.ease)`
  - useNativeDriver: true (transform + opacity only)
- isInteraction: false on all `Animated.timing` loops (STATE.md locked decision)

### Swipe Down to Undo
- Pan gesture downward (threshold: 80px vertical, velocity > 500) ŌĆö D-10
- Undo animation: card returns from off-screen above, sliding down with slight deceleration
  - Duration: 300ms, `Easing.out(Easing.back(1.5))`

### Deck Loop
- Auto-loops: when index reaches end, reset to index 0 ŌĆö D-11
- No "end of deck" empty state needed ŌĆö the deck always has cards (ALIVE+FADING friends)

### Nudge Button
- Tap ŌåÆ async DM navigation via `supabase.rpc('get_or_create_dm_channel')` ŌĆö D-14
- Loading state: disable button + reduce opacity to 0.5 during async call
- Error state: `Alert.alert('Error', "Couldn't open chat. Try again.")` ŌĆö mirrors `HomeFriendCard` pattern
