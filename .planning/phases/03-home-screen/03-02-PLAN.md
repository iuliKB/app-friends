1. Add import: `useRef` from react (if not already imported)

2. Add `channelRef` inside the hook:
   ```
   const channelRef = useRef<ReturnType<typeof supabase.channel> | null>(null);
   ```

3. Create `subscribeRealtime` function inside the hook:
   ```
   function subscribeRealtime(friendIds: string[]) {
     // Tear down existing channel before re-subscribing
     if (channelRef.current) {
       supabase.removeChannel(channelRef.current);
       channelRef.current = null;
     }
     if (friendIds.length === 0) return;

     const filter = `user_id=in.(${friendIds.join(',')})`;
     channelRef.current = supabase
       .channel('home-statuses')
       .on(
         'postgres_changes',
         {
           event: '*',
           schema: 'public',
           table: 'statuses',
           filter,
         },
         (_payload) => {
           fetchAllFriends();
         }
       )
       .subscribe();
   }
   ```

4. Modify `fetchAllFriends` to call `subscribeRealtime` after data loads:
   - After building the `allFriends` array and calling `setFriends(allFriends)`, extract the friend IDs: `const ids = allFriends.map((f) => f.friend_id);`
   - Call `subscribeRealtime(ids);`
   - This ensures the subscription filter is always up-to-date with the current friend list

5. Update the `useEffect` cleanup to tear down the channel:
   ```
   useEffect(() => {
     fetchAllFriends();
     return () => {
       if (channelRef.current) {
         supabase.removeChannel(channelRef.current);
         channelRef.current = null;
       }
     };
   }, [session?.user?.id]);
   ```
   The cleanup function ensures the WebSocket connection is released when the user navigates away from the home tab or logs out.

Key implementation notes:
- Single channel named `'home-statuses'` — NOT one channel per friend (budget-conscious for Supabase free tier 200 concurrent connections)
- `event: '*'` catches INSERT, UPDATE, and DELETE on the statuses table
- Filter `user_id=in.(uuid1,uuid2,...)` — works because `statuses` table has `REPLICA IDENTITY FULL` set in migration
- On any event, call `fetchAllFriends()` (re-fetch all, not surgical patch) — simpler and avoids complex state management
- `subscribeRealtime` is called inside `fetchAllFriends` so it re-subscribes with the latest friend IDs on every fetch cycle
- Previous channel is torn down before creating a new one (prevents connection leak on re-fetch)

Do NOT:
- Use `supabase.from('statuses').on()` — that is the deprecated v1 Realtime API
- Create multiple channels (one per friend) — wastes connections
- Set up subscription before friend IDs are known — empty filter is invalid
- Forget to call `supabase.removeChannel()` in cleanup — leaks WebSocket connections
