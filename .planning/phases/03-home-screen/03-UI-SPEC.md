## Layout Contract

### Screen Structure

```
<ScrollView refreshControl={RefreshControl}>
  <SegmentedControl />                          ← already rendered, Phase 2

  <Animated.Text style={displayHeading}>        ← "X friends free now" or "No friends free right now"
    count with scale pulse animation
  </Animated.Text>

  {freeFriends.length > 0 && (
    <FlatList numColumns={3} scrollEnabled={false}
      data={freeFriends}
      renderItem={HomeFriendCard (no StatusPill)}
    />
  )}

  {friends.length === 0 && <EmptyState />}       ← zero friends total

  {otherFriends.length > 0 && (
    <Text style={sectionLabel}>Everyone Else</Text>
    <FlatList numColumns={3} scrollEnabled={false}
      data={otherFriends}
      renderItem={HomeFriendCard (with StatusPill)}
    />
  )}
</ScrollView>

<TouchableOpacity style={fab}>                  ← absolute positioned, bottom-right
  <Ionicons name="add" /> <Text>Start Plan</Text>
</TouchableOpacity>
```

## Interaction Contract

### Count Heading Animation

- Trigger: `freeFriends.length` changes (detected via `useEffect`)
- Animation: `Animated.sequence` — scale to 1.15 over 100ms, back to 1 over 150ms
- Driver: `useNativeDriver: true` — only `transform: [{ scale }]` animated, no layout properties
- Initial render: no animation — only fires when count changes after first load

### Pull-to-Refresh

- Component: `RefreshControl` on outer `ScrollView`
- `tintColor`: `COLORS.textSecondary` (#9ca3af)
- On refresh: calls `fetchAllFriends()`, sets `refreshing=true`, clears on promise resolution

### Realtime Updates

- Subscription: single `supabase.channel('home-statuses')` with `postgres_changes` filter `user_id=in.(...)`
- On any event (`INSERT`, `UPDATE`, `DELETE`): call `fetchAllFriends()` — re-fetch, not surgical patch
- Cleanup: `supabase.removeChannel(channel)` in `useEffect` return
- Subscription setup: after friend IDs known (inside or after `fetchAllFriends()` resolves)

### FAB tap

- Navigates to Plans tab: `router.push('/(tabs)/plans')`
- No modal, no confirmation — direct navigation
- `activeOpacity: 0.8`

### HomeFriendCard tap

- No action in Phase 3 — card is non-interactive
- Do NOT wrap in `TouchableOpacity` — plain `View`

### Empty State CTA

- Button: `PrimaryButton title="Add Friends"`
- On press: `router.push('/friends/add')` (or equivalent Squad/Friends tab navigation)

## States & Loading

| State | What Renders |
|-------|--------------|
| Initial load (no cache) | SegmentedControl + count heading "..." placeholder — no spinner, no blank flash |
| Loading from cache | Immediate render of cached friend grid — no loading indicator |
| Background revalidation | Grid renders; no visible loading state during revalidation |
| Pull-to-refresh active | `RefreshControl` spinner visible at top; grid remains rendered |
| Realtime event received | Re-fetch triggered; grid re-renders when data returns; count animates if changed |
| Zero friends | EmptyState component with fire emoji, heading, body, Add Friends button |
| Zero free friends (but has friends) | Count heading = "No friends free right now"; Everyone Else grid still renders |
| Fetch error | Inline error text below SegmentedControl: "Couldn't load friends. Pull down to try again." (textSecondary color) |

**No full-screen loading spinner.** Zustand cache ensures content renders immediately. Per RESEARCH.md, stale-while-revalidate pattern.
