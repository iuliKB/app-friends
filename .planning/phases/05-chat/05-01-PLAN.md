  plan_id        uuid REFERENCES public.plans(id) ON DELETE CASCADE,

  CONSTRAINT messages_exactly_one_channel CHECK (

**3. Create `src/hooks/useChatRoom.ts`** — The core chat hook. Takes `{ planId?: string; dmChannelId?: string }` as argument. Returns `{ messages: MessageWithProfile[], loading, error, sendMessage }`.

Implementation details:
- **Profiles map:** On initial fetch, query `plan_members` joined with `profiles` (for plan chats) or use the `friend_name`/`friend_avatar_url` route params (for DMs) to build a `Map<string, { display_name: string; avatar_url: string | null }>`. Store in a `useRef`. For the current user, get profile from `useAuthStore`.
- **Initial messages fetch:** Query `supabase.from('messages').select('*').eq(planId ? 'plan_id' : 'dm_channel_id', planId ?? dmChannelId).order('created_at', { ascending: false }).limit(50)`. Reverse the result array so oldest is first (FlatList inverted will flip visually). Enrich each message with sender profile from the profiles map to produce `MessageWithProfile[]`.
- **Realtime subscription:** Follow `useHomeScreen.ts` pattern exactly — `channelRef = useRef(null)`. Subscribe to `postgres_changes` with `event: 'INSERT'`, `schema: 'public'`, `table: 'messages'`, `filter: plan_id=eq.${planId}` or `dm_channel_id=eq.${dmChannelId}`. On INSERT payload: cast `payload.new as Message`, look up sender profile from profiles map ref, check if `payload.new.id` already exists in state (dedup against optimistic), if not found append to messages array. If found (optimistic entry matched by `sender_id` + body within 5 seconds), replace the optimistic entry (remove by tempId, keep the canonical one from realtime).
- **Cleanup:** `useEffect` cleanup removes channel via `supabase.removeChannel(channelRef.current)`.
- **`sendMessage(body: string)`:** Generate `tempId = crypto.randomUUID()` (or `Date.now().toString()` if crypto not available). Create optimistic `MessageWithProfile` with `{ id: tempId, plan_id: planId ?? null, dm_channel_id: dmChannelId ?? null, sender_id: currentUserId, body, created_at: new Date().toISOString(), pending: true, tempId, sender_display_name: currentUserDisplayName, sender_avatar_url: currentUserAvatarUrl }`. Append to state immediately. Then `await supabase.from('messages').insert({ plan_id: planId ?? null, dm_channel_id: dmChannelId ?? null, sender_id: currentUserId, body })`. If error: remove the optimistic message by filtering out tempId, return `{ error }`. If success: the realtime subscription will handle the canonical message arrival and dedup.
- **Mark as read:** On mount, call `AsyncStorage.setItem('chat:last_read:' + (planId ?? dmChannelId), new Date().toISOString())`.
