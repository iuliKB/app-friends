## Spacing Scale

Declared values (must be multiples of 4). Derived from existing StyleSheet measurements across HomeFriendCard, PlanCard, PrimaryButton, and AvatarCircle.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Emoji badge padding, avatar margin offsets, inline gaps |
| sm | 8px | Avatar-to-name gap, bubble bottom margin, chat list row internal gaps |
| md | 16px | Card padding, screen horizontal padding, FlatList contentContainerStyle paddingTop |
| lg | 24px | Screen top padding, section gaps |
| xl | 32px | Empty state vertical centering offsets |
| 2xl | 48px | Empty state illustration-to-text gap |
| 3xl | 64px | Not used in this phase |

**Exceptions:**
- Send bar height: 52px (matches PrimaryButton height — established pattern)
- Touch targets minimum: 44px tall (iOS HIG compliance for send button and chat list rows)
- Chat list row height: 72px (avatar 40px + 16px vertical padding top+bottom + breathing room for two-line content)
- Message bubble max width: 75% of screen width (avoids full-width bubbles, maintains visual rhythm)
- Pinned plan banner height: 48px (thin — leaves maximum room for messages)

**Source:** Derived from PlanCard (padding: 16), HomeFriendCard (paddingVertical: 16, paddingHorizontal: 16, margin: 4, marginBottom: 8), PrimaryButton (height: 52).

## Interaction Contracts

### Chat Room Screen

| Interaction | Behavior |
|-------------|----------|
| Screen mount | Fetch messages, subscribe to Realtime channel, mark chat as read (AsyncStorage) |
| Screen unmount | Unsubscribe Realtime channel (`supabase.removeChannel`) |
| Keyboard opens (iOS) | KeyboardAvoidingView `behavior="padding"`, `keyboardVerticalOffset=useHeaderHeight()`, send bar lifts above keyboard |
| Keyboard opens (Android) | KeyboardAvoidingView `behavior="height"` |
| Send tap (valid input) | Append optimistic message (opacity 0.7, pending flag), clear TextInput immediately, submit insert to Supabase |
| Send success | Realtime INSERT fires, deduplicate by checking `payload.new.id` not in state, replace pending entry |
| Send failure | Remove optimistic entry from state, show Alert "Message failed to send. Try again." with Retry button |
| Realtime INSERT received | Append `payload.new` to message state (no re-fetch) |
| Scroll behavior | Inverted FlatList — newest at bottom; no manual scroll-to-bottom needed |
| Plan chat | Shows PinnedPlanBanner at top, above the FlatList |
| DM chat | No PinnedPlanBanner |
| Pinned banner tap | `router.push('/plans/' + planId)` |

### Chat List Screen

| Interaction | Behavior |
|-------------|----------|
| Screen focus | `useFocusEffect` — refresh chat list from Supabase, check AsyncStorage for unread state per chat |
| Chat row tap | Navigate to `/chat/room` with correct params (plan_id or dm_channel_id + friend_name) |
| Empty state | Shows empty state copy (see Copywriting) |

### DM Navigation (Friend Card)

| Interaction | Behavior |
|-------------|----------|
| "Start DM" tap | Button shows ActivityIndicator (server confirmation pattern), calls `get_or_create_dm_channel` RPC |
| RPC success | `router.push('/chat/room?dm_channel_id=X&friend_name=Y')` |
| RPC failure | `Alert.alert('Error', "Couldn't open chat. Try again.")` |

## Phase-Specific Notes

### Unread Tracking Storage
- AsyncStorage (not Supabase table) — per RESEARCH.md recommendation
- Key format: `chat:last_read:${planId || dmChannelId}`
- Mark as read on ChatRoomScreen mount
- Check in ChatListScreen on focus to render/clear unread dot

### Sender Profile in Realtime Payload
- `payload.new` contains only raw `messages` row — no joined profile
- Pre-load all member profiles into `Map<sender_id, {display_name, avatar_url}>` on initial fetch
- For DM chats: pass `friend_name` and `friend_avatar_url` as route params (known from friend card)

### Plans with No Messages
- Do not show in chat list — only active chats (at least one message) appear
- Same policy for DM channels with no messages

### Consecutive Grouping Logic
- With inverted FlatList, `index 0` = newest message
- `index + 1` = previous message in conversation
- `isFirstInGroup = messages[index + 1]?.sender_id !== messages[index].sender_id || index === messages.length - 1`
