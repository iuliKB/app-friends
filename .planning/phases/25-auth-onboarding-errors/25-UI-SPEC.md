## Spacing Scale

All values drawn directly from `src/theme/spacing.ts` — do not introduce new ad-hoc values.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Avatar overlap offset, tight gaps |
| sm | 8px | Separator height, inline gaps, icon-label gap |
| md | 12px | Card section gaps, separator marginLeft |
| lg | 16px | Standard screen padding, card padding |
| xl | 24px | Large section spacing, button-from-form gap |
| xxl | 32px | Screen header top, auth header padding |

Exceptions:
- Auth screen header `paddingTop` uses 64px (hardcoded in existing `AuthScreen.tsx` — no token exists; do NOT change it in this phase)

## Layout Specifications

### Forgot Password Inline Toggle (AUTH-01)

Position in `AuthScreen.tsx`: placed between the password `FormField` and the `PrimaryButton`, below the password field on the login tab only.

```
[Password field]
[Forgot password?]        ← right-aligned Text link, FONT_SIZE.md, accent color, paddingVertical: SPACING.xl
[Sign In button]
```

Reset mode replaces the form content (email + password fields + sign-in button) with:
```
[Reset your password]     ← FONT_SIZE.xl heading, display.semibold, text.primary
[Email address field]     ← FormField, full width
[Send reset link button]  ← PrimaryButton, full width, loading state
[Back to sign in]         ← centered Text link, FONT_SIZE.md, accent color, paddingTop: SPACING.lg
```

Success state (after email sent) replaces the reset form with:
```
[check-circle Ionicon]    ← size 48, colors.interactive.accent
[Check your email]        ← FONT_SIZE.xl, display.semibold, text.primary, marginTop: SPACING.lg
[We sent a password reset link to {email}]  ← FONT_SIZE.lg, body.regular, text.secondary, textAlign: 'center'
[Back to sign in]         ← centered link, FONT_SIZE.md, accent color, marginTop: SPACING.xl
```

### ToS / Privacy Links (AUTH-02)

Position in `AuthScreen.tsx`: below the `PrimaryButton` ("Create Account") on the signup tab only.

```
[Create Account button]
[gap: SPACING.lg]
[By creating an account you agree to our [Terms of Service] and [Privacy Policy].]
```

Full copy: "By creating an account you agree to our [Terms of Service] and [Privacy Policy]."
- Wrapper: `Text` centered, `FONT_SIZE.md`, `body.regular`, `colors.text.secondary`
- "Terms of Service": inline `Text` with `colors.interactive.accent`, `textDecorationLine: 'underline'`, `onPress` opens `https://campfire.app/tos` via `WebBrowser.openBrowserAsync()`
- "Privacy Policy": inline `Text` with `colors.interactive.accent`, `textDecorationLine: 'underline'`, `onPress` opens `https://campfire.app/privacy` via `WebBrowser.openBrowserAsync()`

### Onboarding Bottom Sheet (AUTH-04)

Sheet structure (matches `StatusPickerSheet.tsx` layout pattern):
```
[Modal + semi-transparent backdrop: colors.overlay]
  [Animated.View, slide from bottom]
    [drag handle area: paddingVertical SPACING.sm, alignItems center]
      [drag handle: width 40, height 4, RADII.xs, colors.border]  ← visual only, no gesture
    [sheet body: paddingHorizontal SPACING.lg, paddingBottom SPACING.xxl]
      [🔥 emoji, fontSize 32, textAlign center]
      [gap: SPACING.sm]
      ["Welcome to Campfire!" heading]
      [gap: SPACING.lg]
      ["Tap your status above to let friends know if you're free." body text]
      [gap: SPACING.sm]
      ["Head to Squad to add friends." body text]
      [gap: SPACING.xl]
      [PrimaryButton title="Get Started" onPress={handleDismiss}]
```

Sheet height: content-driven (no fixed height). Let `paddingBottom: SPACING.xxl` absorb safe area. The sheet background is `colors.surface.card`.

No swipe-to-dismiss (D-11). No close button. "Get Started" is the only exit.

## State Inventory

Every interactive surface must handle all states below:

### Auth Forgot Password Toggle

| State | What the user sees |
|-------|--------------------|
| `login` (default) | Email + password fields, "Forgot password?" link below password field |
| `reset` | Reset heading, email field, "Send reset link" button (loading state supported), "Back to sign in" link |
| `reset-sent` | Success icon, "Check your email" heading, confirmation copy, "Back to sign in" link |

### "Send reset link" Button

| State | Visual |
|-------|--------|
| Idle | Full-width accent button, label "Send reset link" |
| Loading | `PrimaryButton loading={true}` — inline spinner, label hidden |
| Success | Form transitions to `reset-sent` state |
| Error | `ErrorDisplay` inline below the button, `mode='inline'`, message from `mapAuthError()` |

### Onboarding Sheet

| State | Visual |
|-------|--------|
| Not shown | Nothing rendered (flag already set or user has friends) |
| Shown | Modal + animated sheet, "Get Started" button |
| Dismissed | Sheet closes, flag written to AsyncStorage, never shown again |

### Data-Fetching Screens (AUTH-03)

Each of the ~12 screens follows this state model:

| State | What the user sees |
|-------|--------------------|
| Loading | Existing loading indicator (unchanged — skeleton shimmer from Phase 24 lands in Phase 26) |
| Success | Normal screen content |
| Error | `ErrorDisplay mode='screen'` centered in a `flex: 1` container, message per screen (see Copywriting Contract above), retry button calls `refetch()` |

## Accessibility

- All tappable text links (Forgot password, Back to sign in, ToS, Privacy Policy): minimum 44px touch target via `paddingVertical: SPACING.xl` (24px) applied to the wrapping `TouchableOpacity` or via `hitSlop={{ top: 12, bottom: 12 }}`
- ErrorDisplay icon (`alert-circle-outline`, size 48): decorative — no `accessibilityLabel` needed on icon; the `Text` message below it carries meaning
- Onboarding sheet "Get Started" button: `accessibilityRole="button"` (provided by `PrimaryButton`)
- `Modal` in `OnboardingHintSheet`: set `accessibilityViewIsModal={true}` on the sheet container
- Color contrast: all text/background pairs use theme tokens which have been pre-verified (see `light-colors.ts` inline comments)
