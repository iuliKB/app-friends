```typescript
async function handleSendResetLink() {
  const emailErr = validateEmail(resetEmail);
  if (emailErr) { setResetError(emailErr); return; }
  setResetLoading(true);
  setResetError(undefined);
  try {
    const { error } = await supabase.auth.resetPasswordForEmail(resetEmail);
    if (error) {
      setResetError(mapAuthError(error.message));
    } else {
      setAuthMode('reset-sent');
    }
  } catch {
    setResetError('Something went wrong. Check your connection and try again.');
  } finally {
    setResetLoading(false);
  }
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User → Supabase Auth | Email submitted by user is passed to `resetPasswordForEmail()` — Supabase handles rate limiting and token security |
| App → External Browser | URLs opened via `WebBrowser.openBrowserAsync()` — system browser, no in-app session data exposed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-25-01-01 | Information Disclosure | `resetPasswordForEmail()` | accept | Supabase returns success even when email not found — prevents account enumeration; no additional mitigation needed in client |
| T-25-01-02 | Spoofing | Reset email input | mitigate | Reuse existing `validateEmail()` in AuthScreen.tsx before calling `resetPasswordForEmail()` — invalid email format rejected client-side |
| T-25-01-03 | Tampering | ToS/Privacy URLs | accept | URLs are compile-time constants (`https://campfire.app/tos`); system browser handles navigation; no user-supplied URL input |
| T-25-01-04 | Denial of Service | Reset form button | mitigate | `resetLoading` state disables button during in-flight request; `PrimaryButton loading={true}` renders disabled state |
</threat_model>
