## Screen Layouts

### Screen 1: IOU Balance Index (`/squad/expenses`)

Entry point: `IOUCard` tap from Goals tab (D-13), plus existing squad header '+' button context is unchanged from Phase 8 (still opens create screen).

Structure:

```
SafeAreaView (background: COLORS.surface.base)
  ScreenHeader title="Balances"
  [Loading] LoadingIndicator (full-screen centered)
  [Error] ErrorDisplay (full-screen, pull-to-refresh)
  [Empty] EmptyState
      icon="💸" heading="All settled up!" body="No unsettled balances with friends."
  [Data] FlatList (data: balance rows, no nested scroll)
      ItemSeparatorComponent: 1px COLORS.border line
      renderItem: BalanceRow (tappable → /squad/expenses/friend/[id])
      contentContainerStyle: paddingHorizontal: SPACING.lg, paddingTop: SPACING.lg
      refreshControl: pull-to-refresh wired to refetch()
```

Data source: `useIOUSummary` hook — `get_iou_summary()` RPC returns per-friend rows with `friend_id`, `display_name`, `net_amount_cents`, `unsettled_count`. Show only rows with `unsettled_count > 0` (D-06: hide zero-balance friends).

### Screen 2: Per-Friend Expense History (`/squad/expenses/friend/[id]`)

Structure:

```
SafeAreaView (background: COLORS.surface.base)
  ScreenHeader title="Expenses with {friendName}"
  NetBalanceSummaryStrip (48px height, surface.card background)
    ├── Left: "Net balance" label at 14px/secondary
    └── Right: signed amount at 20px/semibold in green or red
  [Loading] 3× ExpenseHistoryRowSkeleton (grey boxes, opacity 0.5)
  [Error] ErrorDisplay
  [Empty] EmptyState
      icon="🧾" heading="No shared expenses" body="Expenses you create together will appear here."
  [Data] FlatList (data: expense rows, ordered newest first)
      ItemSeparatorComponent: 1px COLORS.border line
      renderItem: ExpenseHistoryRow (tappable → /squad/expenses/[id] detail screen — Phase 8)
      contentContainerStyle: paddingHorizontal: SPACING.lg
      refreshControl: pull-to-refresh wired to refetch()
```

Data source: `useExpensesWithFriend(friendId)` hook. `friendId` extracted from route params `[id]`.

Net balance strip: derived from the `netAmountCents` passed via navigation params from the parent balance row. No extra fetch needed for this value.

## Interaction Contracts

### Balance row tap (index → history)

1. User taps `BalanceRow`
2. `router.push('/squad/expenses/friend/' + friendId)` fires — passes `friendName` and `netAmountCents` as route params for immediate header and strip render without waiting for history fetch
3. No haptic on navigation (follow Phase 8 pattern: haptics reserved for write actions only)

### History row tap (history → detail)

1. User taps `ExpenseHistoryRow`
2. `router.push('/squad/expenses/' + expenseId)` fires — navigates to existing Phase 8 detail screen
3. No haptic on navigation

### Pull-to-refresh (both screens)

1. User pulls down on FlatList
2. `RefreshControl` triggers `refetch()` from hook
3. `loading` state re-enters while refetch in flight; spinner shows in `RefreshControl` (do not show full-screen `LoadingIndicator` on refresh, only on initial load)

### IOUCard tap (Goals tab → index)

1. User taps `IOUCard` anywhere on the card surface
2. `router.push('/squad/expenses')` fires
3. `pressed && { opacity: 0.85 }` on card `Pressable`
4. No haptic

### Settled row dimming (history screen)

- `isFullySettled` prop: `true` when all `iou_members` rows for that expense have `settled_at IS NOT NULL`
- Render: wrap entire `ExpenseHistoryRow` in `<View style={{ opacity: 0.45 }}>` — do NOT use `useNativeDriver` opacity animation; static opacity is sufficient and avoids Animated/native driver complexity
- Dimmed rows are still tappable (full audit trail — user can still navigate to detail)

**Currency formatting:** All amounts rendered via `formatCentsDisplay()` from `src/utils/currencyFormat.ts` — no raw float arithmetic ever reaches the UI layer.

**Direction notation:** `→ you` means the friend owes you; `← you` means you owe the friend. Arrow direction is FROM the debtor TO the creditor. This is consistent with D-03 from CONTEXT.md.
