## Spacing Scale

Campfire uses a named spacing scale (not a uniform 8-point scale). All spacing values
are taken directly from `src/theme/spacing.ts`.

| Token | Value | Usage in Phase 27 |
|-------|-------|-------------------|
| `SPACING.xs` | 4px | Icon-to-label gaps, skeleton row gaps |
| `SPACING.sm` | 8px | Gap between RSVP buttons, skeleton element gaps |
| `SPACING.md` | 12px | WishListItem internal padding (vertical), skeleton metadata gap |
| `SPACING.lg` | 16px | PlanCard padding (mirror in skeleton), map overlay card padding |
| `SPACING.xl` | 24px | Map overlay card top/bottom margin from screen edges |
| `SPACING.xxl` | 32px | — |

Exceptions:
- RSVP buttons: `height: 44px` (hardcoded touch target minimum — existing, kept as-is)
- Exception: `SPACING.md = 12px` — pre-existing project token in `src/theme/spacing.ts`, established before Phase 24; not modifiable in this phase without a dedicated token-refactor phase. Used for internal card padding only.
- Map empty state overlay: centered absolutely over map, no margin exception needed

## Component Contracts

### PLANS-01 — PlanCard Skeleton

Shape mirrors `PlanCard.tsx` exactly to prevent layout shift on data reveal.

```
┌──────────────────────────────────────┐  ← borderRadius: RADII.lg (12px)
│ [IMAGE PLACEHOLDER — 140px tall]     │  ← SkeletonPulse width="100%" height=140
├──────────────────────────────────────┤
│ [TITLE BAR — 20px tall]              │  ← SkeletonPulse width="60%" height=20, mt=SPACING.md
│ [META LINE 1 — 14px tall]            │  ← SkeletonPulse width="40%" height=14, mt=SPACING.sm
│ [META LINE 2 — 14px tall]            │  ← SkeletonPulse width="30%" height=14, mt=SPACING.xs
└──────────────────────────────────────┘
```

- **Count:** 3 skeleton cards rendered when `loading && plans.length === 0`
- **Condition:** My Plans tab only — Invitations tab has no skeleton
- **Card outer style:** `backgroundColor: colors.surface.card`, `borderRadius: RADII.lg`,
  `borderWidth: 1`, `borderColor: colors.border`, `padding: SPACING.lg` — mirrors PlanCard exactly
- **Shimmer:** `SkeletonPulse` component (Phase 24) — shimmer sweeps left-to-right at
  `ANIMATION.duration.verySlow` (1200ms), `ANIMATION.easing.decelerate()`, `useNativeDriver: true`
- **Image placeholder:** `borderRadius: RADII.sm` (matches SkeletonPulse default)
- **Pull-to-refresh:** RefreshControl spinner shown over existing content — skeleton only on initial load

### PLANS-02 — RSVP Spring Bounce Animation

Applies to `RSVPButtons.tsx`. Only the tapped button animates.

```
Animation sequence on tap:
  1.0 (rest) → 0.92 (compress, 80ms) → 1.05 (overshoot, 100ms) → 1.0 (settle, 120ms)

Haptic: Haptics.selectionAsync() — fire before animation, fire-and-forget (.catch(() => {}))
```

- **Animated.Value:** one `scaleAnim` per button (3 total: going, maybe, out)
- **spring config:** `Animated.spring` with `ANIMATION.easing.spring` (damping: 15, stiffness: 120)
- **useNativeDriver:** `true` — scale is a transform, eligible for native thread
- **Untapped buttons:** no animation — scale stays at 1.0
- **Disabled state:** haptic and animation do not fire when `disabled || savingRsvp !== null`
- **Wrapper:** replace `TouchableOpacity` with `Animated.createAnimatedComponent(TouchableOpacity)`,
  apply `transform: [{ scale: scaleAnim }]` to the Animated wrapper
- **Touch target:** 44px height preserved (existing constraint, marked `eslint-disable`)

### PLANS-03 — Plan Creation Success Haptic

Fires in `PlanCreateModal.tsx` immediately after the plan is written to Supabase,
before the modal dismisses.

```typescript
// Fire after successful plan creation, before router.back() / dismissal
void Haptics.notificationAsync(Haptics.NotificationFeedbackType.Success).catch(() => {});
```

- **Timing:** after async plan creation resolves successfully, before navigation away
- **Pattern:** fire-and-forget — never `await` in sync/event context
- **No UI change:** haptic only, no visual indicator added

### SQUAD-01 — Friend Request Haptics

Fires in `FriendRequests.tsx` after the async call resolves.

| Action | Haptic | Semantic reasoning |
|--------|--------|--------------------|
| Accept | `Haptics.notificationAsync(NotificationFeedbackType.Success)` | Positive confirmation — new friend |
| Reject | `Haptics.impactAsync(ImpactFeedbackStyle.Medium)` | Physical dismissal — no success signal |

- **Fire location:** `handleAccept` and `handleReject` after `await` completes, before any state update
- **Pattern:** fire-and-forget — `void Haptics.X().catch(() => {})`
- **No visual change** — haptic only

### SQUAD-03 — Squad Dashboard Stagger Animation

Two changes:

**1. Token addition in `src/theme/animation.ts`:**
```typescript
duration: {
  fast: 200,
  normal: 300,
  slow: 700,
  verySlow: 1200,
  staggerDelay: 80,   // ← ADD THIS
}
```

**2. Token usage in `src/app/(tabs)/squad.tsx`:**
```typescript
// BEFORE:
Animated.stagger(80, ...)

// AFTER:
Animated.stagger(ANIMATION.duration.staggerDelay, ...)
```

- **Animation style:** opacity-only (existing `AnimatedCard` wrapper, `Animated.Value(0) → 1`)
- **Duration per card:** `ANIMATION.duration.normal` (300ms)
- **Easing:** `useNativeDriver: true`, `isInteraction: false`
- **Guard:** `hasAnimated.current` remains — stagger fires once on mount, never on refresh or tab re-focus
- **No translateY** — opacity fade is sufficient, avoids layout complexity

**Visual result:**
- StreakCard fades in at t=0ms
- IOUCard fades in at t=80ms
- BirthdayCard fades in at t=160ms

### SQUAD-04 — WishListItem Spring Press Feedback

Applies to the Claim/Unclaim `Pressable` in `WishListItem.tsx`.

```
Animation: 1.0 (rest) → 0.96 (compress on press-in) → 1.0 (release on press-out)
```

- **Animated.Value:** `scaleAnim = new Animated.Value(1)` per WishListItem instance
- **Press-in:** `Animated.spring(scaleAnim, { toValue: 0.96, ...ANIMATION.easing.spring, useNativeDriver: true })`
- **Press-out:** `Animated.spring(scaleAnim, { toValue: 1.0, ...ANIMATION.easing.spring, useNativeDriver: true })`
- **Spring config:** `damping: 15, stiffness: 120` (from `ANIMATION.easing.spring`)
- **Wrapper:** replace `Pressable` with `Animated.createAnimatedComponent(Pressable)` or
  wrap `Pressable` in `<Animated.View style={{ transform: [{ scale: scaleAnim }] }}>`
- **useNativeDriver:** `true` — scale transform is native-eligible
- **Consistency:** matches HOME-04 pattern (1.0→0.96 compress-and-release across all tappable cards)
- **Existing opacity press style** `pressed && { opacity: 0.7 }` is removed — replaced by scale spring

## Accessibility

Applied from ui-ux-pro-max skill guidelines:

| Requirement | Implementation |
|-------------|----------------|
| Touch targets ≥ 44px | RSVP buttons: 44px height (preserved). WishListItem Pressable: vertical padding `SPACING.xs` + `SPACING.md` = 16px — acceptable for secondary action. |
| Color not sole indicator | RSVP active state: background color + label weight change (semibold). Map overlay: icon + text (not color alone). |
| `accessibilityRole` | RSVP buttons: `accessibilityRole="button"` + `accessibilityState={{ selected: isActive }}` (existing, preserved). |
| `accessibilityLabel` | WishListItem: `accessibilityLabel={claimLabel}` (existing, preserved). Map overlay: no interactive elements — overlay is informational. |
| Reduced motion | SkeletonPulse shimmer uses `isInteraction: false`. Squad stagger and press springs use `useNativeDriver: true` (GPU-composited, no JS thread jank). No `prefers-reduced-motion` API in RN — accepted limitation for this phase. |
| Loading feedback | 3 skeleton cards shown for Plans list — no blank state on first load. |
