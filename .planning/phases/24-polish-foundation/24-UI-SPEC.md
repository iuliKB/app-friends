### SkeletonPulse Color Contract

| Layer | Token | Notes |
|-------|-------|-------|
| Base rectangle | colors.surface.card | Dark: `#1D2027` / Light: `#FFFFFF` — the muted surface the shimmer sweeps over |
| Shimmer band — leading edge | transparent | Gradient stop 0 |
| Shimmer band — center highlight | colors.surface.overlay | Dark: `rgba(255,255,255,0.08)` / Light: `rgba(0,0,0,0.06)` — subtle shine readable in both themes |
| Shimmer band — trailing edge | transparent | Gradient stop 2 |

The three-stop gradient (`[transparent, colors.surface.overlay, transparent]`) travels left-to-right. `colors.surface.overlay` is the existing token for overlay tinting in both themes — it reads as a visible "shine" against `colors.surface.card` in dark mode and against `#FFFFFF` in light mode without requiring a new token.

## Component Contracts

### SkeletonPulse (POLISH-01)

**File:** `src/components/common/SkeletonPulse.tsx`

**Visual:**
- Rectangle with RADII.sm (6px) corner radius — always. No `borderRadius` prop.
- Background: `colors.surface.card`
- Shimmer: expo-linear-gradient, horizontal sweep left-to-right
  - Gradient colors: `['transparent', colors.surface.overlay, 'transparent']`
  - Gradient start: `{ x: 0, y: 0 }` → end: `{ x: 1, y: 0 }` (horizontal)
- Shimmer band width: 30% of the component width (achieved via the translateX range)

**Animation:**
- Driver: `Animated.loop` + `useNativeDriver: true` on a `translateX` transform — stays on native thread, zero JS-thread cost during scroll
- Pattern follows RadarBubble PulseRing: `Animated.loop(Animated.timing(...))` with `isInteraction: false`
- Duration: use `ANIMATION.verySlow` (1200ms) once animation tokens are defined (POLISH-02 lands in same phase). Implementation order: define ANIMATION const first, then reference it in SkeletonPulse.
- Easing: `ANIMATION.easing.decelerate()` — `Easing.out(Easing.ease)` — gives a smooth left-to-right glide
- translateX range: from `-componentWidth * 1` to `componentWidth * 1` — shimmer enters from the left edge and exits past the right edge. The component must use `onLayout` or accept `width` as a number to compute this range. When `width` is `'100%'`, use `onLayout` to capture the pixel value.
- `isInteraction: false` — must not block JS gesture handling

**API:**

```ts
interface SkeletonPulseProps {
  width: number | '100%';
  height: number;
}
```

No children. No `style` prop. No `borderRadius` prop. Caller composes multiple `<SkeletonPulse>` rectangles inside a `View` to build skeleton layouts.

**Accessibility:**
- `accessibilityLabel="Loading"` — readable by screen readers while data is fetching
- `accessibilityRole` not set (View default)

**Pattern reference:** RadarBubble.tsx PulseRing — `Animated.loop` + `useNativeDriver: true` on transform.

### Animation Tokens (POLISH-02)

**File:** `src/theme/animation.ts`

**Export:** `ANIMATION` const — barrel-exported from `src/theme/index.ts` as `export { ANIMATION } from './animation'`

```ts
export const ANIMATION = {
  duration: {
    fast: 200,      // quick UI responses, haptic confirms
    normal: 300,    // state transitions, reveals
    slow: 700,      // emphasis animations, status pulses
    verySlow: 1200, // looping ambient animations (radar pulse, skeleton shimmer)
  },
  easing: {
    standard: () => Easing.inOut(Easing.ease),   // balanced in+out — state transitions
    decelerate: () => Easing.out(Easing.ease),   // fast-in, slow-out — content arriving
    accelerate: () => Easing.in(Easing.ease),    // slow-in, fast-out — content leaving
    spring: { damping: 15, stiffness: 120 },     // Reanimated withSpring config — RSVP bounce
  },
} as const;
```

Easing presets are lazy functions (not pre-evaluated values) to avoid import-time side effects — same pattern used in RadarBubble.tsx where `Easing.out(Easing.ease)` is called inline.

**Token usage rules:**
- All new animation components in Phase 24 (SkeletonPulse) must reference `ANIMATION.duration.*` and `ANIMATION.easing.*` — no raw numbers.
- Existing components with hardcoded durations are NOT migrated in Phase 24 (D-08 in CONTEXT.md). Later phases migrate as they touch those files.

## Interaction Contract

| Interaction | Specification |
|-------------|--------------|
| Skeleton visible during | From mount until data resolves — callers wrap in conditional render |
| Skeleton loop | Continuous `Animated.loop` — no pause between cycles |
| Skeleton unmount | Loop stopped via `useEffect` cleanup return `() => loop.stop()` |
| Button loading state | ActivityIndicator replaces label text; button opacity drops to 0.5; `onPress` disabled |
| EmptyState CTA press | Calls caller-supplied `onCta` callback; button uses default PrimaryButton press feedback (activeOpacity 0.8) |
