## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, reaction badge padding, accent bar width |
| sm | 8px | Bubble inner padding, inline gaps, separator heights |
| md | 12px | Card section gaps |
| lg | 16px | Standard screen/card padding, poll card horizontal padding |
| xl | 24px | Sheet section spacing |
| xxl | 32px | Sheet bottom safe-area padding |

Exceptions:
- Touch targets enforce 44px minimum height (pill action rows, option rows, emoji buttons) — hardcoded per existing pattern
- Drag handle: 40px wide × 4px tall, fixed — follows SendBar attachment sheet pattern

**Source:** `src/theme/spacing.ts` + existing component patterns in SendBar.tsx and MessageBubble.tsx.

## Component Inventory

### PollCard (new component)

**Location:** `src/components/chat/PollCard.tsx`

**Layout:**
- Full-width card, NOT a left/right-aligned bubble. Spans the full usable chat width with `SPACING.lg` (16px) horizontal padding on both sides matching the chat container.
- Background: `COLORS.surface.card` (#2a2a2a), border radius: `RADII.lg` (12px).
- Vertical padding: `SPACING.lg` (16px) top and bottom.

**Card header:**
- Row: `📊` emoji (fontSize 20) + question text (FONT_SIZE.lg, FONT_WEIGHT.semibold, COLORS.text.primary).
- Gap between emoji and text: `SPACING.sm` (8px).
- Below header: thin horizontal divider (height 1, color COLORS.border), marginVertical `SPACING.sm` (8px).

**Unvoted state (D-07):**
- Each option row: height 44px minimum (touch target), horizontal padding `SPACING.lg`.
- Row content: unfilled radio circle (○, 20px diameter, border 2px COLORS.border) + option label text (FONT_SIZE.lg, FONT_WEIGHT.regular, COLORS.text.primary).
- No progress bars. No vote counts. No total footer count visible to this user.
- Active opacity: 0.7 on tap.

**Voted state (D-08):**
- Each option row: radio circle (20px) + label text + progress bar + vote count integer, all in one row.
- Selected option: radio ● filled with `COLORS.interactive.accent`; progress bar color `COLORS.interactive.accent`.
- Unselected options: radio ○ (border `COLORS.border`); progress bar color `COLORS.surface.overlay` (rgba(255,255,255,0.14)).
- Progress bar: height 4px, border-radius `RADII.xs` (4px), positioned below or beside the label. Bar fills from 0 to `(optionVotes / totalVotes) * 100%`.
- Vote count: FONT_SIZE.sm, FONT_WEIGHT.regular, COLORS.text.secondary — rendered after the bar, right-aligned or trailing.
- Animation from unvoted → voted: progress bar width animates from 0 to target value over 300ms (`Animated.timing`, `useNativeDriver: false`).

**Card footer (D-09):**
- "N votes" total. FONT_SIZE.sm, FONT_WEIGHT.regular, COLORS.text.secondary.
- Separator above footer: thin divider (height 1, COLORS.border), marginTop `SPACING.sm`.
- Footer padding: `SPACING.sm` (8px) vertical.

**Context menu (D-10):**
- Reactions: shown (same emoji strip as other message types).
- Reply action: HIDDEN (no `reply_to_message_id` for poll messages).
- Copy action: HIDDEN (no text body).
- Delete action: shown only to poll creator — uses `COLORS.interactive.destructive`, soft-delete to "Message deleted" placeholder.

### PollCreationSheet (new component)

**Location:** `src/components/chat/PollCreationSheet.tsx`

**Presentation:** Bottom sheet modal over chat — same slide-up animation as the attachment action sheet in SendBar.tsx (Animated.Value translateY 300→0, duration 250ms; close 200ms). Uses React Native Modal with transparent backdrop.

**Layout:**
- Background: `COLORS.surface.card`, border-top-radius: `RADII.lg` (12px).
- Drag handle: 40px wide × 4px tall, `COLORS.border`, centered, marginTop `SPACING.sm`, marginBottom `SPACING.md`.
- Bottom safe-area padding: `SPACING.xxl` (32px).

**Sheet header:**
- Text "New Poll" — FONT_SIZE.xl (20px), FONT_WEIGHT.semibold, COLORS.text.primary. Centered or left-aligned at `SPACING.lg` horizontal padding.

**Question input (D-02):**
- TextInput, full-width, `SPACING.lg` horizontal margin.
- Placeholder: "Ask the group…" — COLORS.text.secondary.
- Font: FONT_SIZE.lg, COLORS.text.primary.
- Background: COLORS.surface.base, border-radius: RADII.md (8px), padding `SPACING.sm`.
- marginBottom `SPACING.md`.

**Option rows (D-02, D-03):**
- Each row: TextInput with label e.g. "Option 1", "Option 2" etc.
- Placeholder text is the label string — COLORS.text.secondary.
- Font: FONT_SIZE.lg, COLORS.text.primary.
- Background: COLORS.surface.base, border-radius: RADII.md, padding `SPACING.sm`.
- Options 1 and 2: no × button. Options 3 and 4: × button (Ionicons "close-circle-outline", size 20, COLORS.text.secondary) on the right end.
- Row gap: `SPACING.sm` (8px) between option rows.
- "+ Add option" row: visible when fewer than 4 options; hidden when 4th is added.
  - Style: FONT_SIZE.lg, FONT_WEIGHT.semibold, COLORS.interactive.accent. Ionicons "add" icon (size 20, accent) inline before label.
  - Touch target: minHeight 44px.

**Buttons (D-02, D-04):**
- Row at bottom: Cancel (left) + Send Poll (right).
- Cancel: FONT_SIZE.lg, FONT_WEIGHT.semibold, COLORS.text.secondary. No background. minHeight 44px.
- Send Poll (active): background COLORS.interactive.accent, border-radius RADII.md (8px), label FONT_SIZE.lg FONT_WEIGHT.semibold COLORS.surface.base. minHeight 44px.
- Send Poll (disabled): background COLORS.surface.overlay (muted), label COLORS.text.secondary. Disabled until question non-empty AND all visible option fields non-empty.
- Button row: `SPACING.lg` horizontal padding, `SPACING.xl` (24px) top margin from last option.

**Keyboard behavior:** KeyboardAvoidingView wrapping the sheet content, behavior "padding" on iOS. Sheet scrolls up to remain above the keyboard.

## Interaction States

### Poll option tap — vote / change vote / un-vote

| State | Trigger | Visual |
|-------|---------|--------|
| Unvoted | Initial load | All options show ○ radio, no bars |
| Voting (optimistic) | Tap unvoted option | Immediately: selected ● fills accent, bars appear with current counts + 1 on selected |
| Changing vote (optimistic) | Tap different option while voted | Previous ● → ○, new ○ → ●, bar values update instantly |
| Un-voting (optimistic) | Tap currently selected option | All ● → ○, bars and counts disappear, card returns to unvoted display |
| Revert on failure | DB write fails | Silent revert to previous state (no toast — follows Phase 15 pattern) |

### Send Poll button

| State | Condition | Visual |
|-------|-----------|--------|
| Disabled | Question empty OR any visible option empty | Background muted overlay, label text.secondary |
| Enabled | All fields filled | Background COLORS.interactive.accent, label surface.base |

## Accessibility

- All tappable option rows: `accessibilityLabel` = "{option label}, {vote count} votes, {selected/unselected}".
- Radio circle: not a separate accessible element — parent row carries the label.
- Send Poll button: `accessibilityLabel` = "Send Poll" when enabled, "Send Poll, disabled — fill in question and all options" when disabled.
- + Add option: `accessibilityLabel` = "Add another option".
- × remove button: `accessibilityLabel` = "Remove option {N}".
- Poll card: `accessibilityRole` = "none" on card container; each option row `accessibilityRole` = "button".
- Minimum touch targets: 44px height enforced on all interactive rows (follows existing MessageBubble/SendBar pattern).
