<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user A friend list → user B post sign-out | Inherited Wave 1 mitigation (authBridge.removeQueries + per-user keys). |
| wishlist vote write → server | Server RLS gates: caller can vote at most once per wish_item. Optimistic flip relies on the same constraint. |
| IOU settle write → server | Server enforces caller is one of the expense members. Optimistic flip only modifies caller's own member row. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-31-16 | Information Disclosure | Friend list cache surviving sign-out | mitigate | Wave 1 authBridge. Per-user `friends.list(userId)` key. |
| T-31-17 | Tampering | Wishlist vote optimistic write affects another user's vote count | mitigate | Optimistic write only adjusts `voted_by_me` for the caller's row and increments/decrements the aggregate `vote_count` by 1. Server RLS is the authoritative check; on RPC error the rollback restores the pre-vote snapshot. |
| T-31-18 | Tampering | IOU settle optimistic write flips foreign member's settled_at | mitigate | Settle setQueryData callback filters by `m.user_id === userId`. RLS enforces server-side. |
</threat_model>
