# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Foundation + Auth. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — React Native StyleSheet only |
| Preset | not applicable |
| Component library | none (PROJECT.md: "No UI libraries — React Native StyleSheet only") |
| Icon library | @expo/vector-icons (Ionicons subset) — included in Expo SDK, Expo Go compatible |
| Font | System default (San Francisco on iOS, Roboto on Android) — no custom font loading in Phase 1 |

**Rationale:** PROJECT.md explicitly prohibits NativeBase, Tamagui, Gluestack, and all UI libraries. shadcn is web-only and incompatible with React Native. Registry safety gate: not applicable.

---

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding |
| sm | 8px | Input internal padding (vertical), label-to-input gap |
| md | 16px | Screen horizontal padding, form field spacing, button vertical padding |
| lg | 24px | Section gaps, card padding |
| xl | 32px | Header bottom margin, section breaks |
| 2xl | 48px | Vertical rhythm between major form sections |
| 3xl | 64px | Top safe-area offset on auth screen |

Exceptions:
- Touch targets minimum 44px height (Apple HIG) — applies to all tappable rows, tab bar items, and OAuth buttons
- Bottom tab bar height: 56px (content area) + safe area inset
- Offline banner height: 32px fixed

---

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (regular) | 1.5 |
| Label | 14px | 400 (regular) | 1.4 |
| Heading | 20px | 600 (semibold) | 1.2 |
| Display | 28px | 700 (bold) | 1.1 |

**Notes (source: default — standard mobile type scale):**
- Display (28px/700) used only for: Campfire app name in branded header on auth screen and splash screen
- Heading (20px/600) used for: screen titles, auth tab labels (Login / Sign Up) when active, profile creation section header
- Body (16px/400) used for: form field input text, placeholder text, paragraph copy, tagline
- Label (14px/400) used for: form field labels above inputs, helper text below fields, tab bar labels, "or continue with" divider text, inline error messages (color change to destructive, weight stays 400)

---

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #1a1a1a | App background, screen backgrounds — dark base |
| Secondary (30%) | #2a2a2a | Cards, input backgrounds, tab bar background, modal sheets |
| Accent (10%) | #f97316 | Campfire orange — primary interactive elements only |
| Destructive | #ef4444 | Inline auth error text, destructive confirmation actions |

**Accent (#f97316 campfire orange) reserved for:**
- Primary CTA button fill (Sign In, Create Account, Save Profile)
- Active tab indicator in bottom nav
- "Add photo" button text on profile creation screen
- Tappable "Create Account" / "Sign In" switch link on auth screen

**Warm background note:** Splash screen uses #ff6b35 (warm orange-red gradient base) to establish the cozy brand feel before the app loads. Auth screen uses #1a1a1a dominant with the campfire emoji header providing warmth.

**Status colours (defined in Phase 1, used app-wide — source: PROJECT.md):**
- Free: #22c55e
- Busy: #ef4444
- Maybe: #eab308

These are not used in Phase 1 UI directly (no status UI until Phase 2) but must be defined as constants in `src/constants/colors.ts` in Phase 1 for downstream phases.

**Additional semantic colours:**
- Text primary: #f5f5f5 (on dark background)
- Text secondary: #9ca3af (placeholder, helper text, muted labels)
- Border: #3f3f46 (input borders, dividers)
- Surface overlay: #ffffff14 (semi-transparent card borders, input focus ring at 20% white)
- Offline banner background: #92400e (warm amber-brown warning tone)
- Offline banner text: #fef3c7

## Screen Contracts

### 1. Splash Screen

**Trigger:** App cold start, session check in progress
**Duration:** Until `INITIAL_SESSION` fires from Supabase (use `SplashScreen.preventAutoHideAsync()`)

| Element | Specification |
|---------|---------------|
| Background | Full-screen gradient: #ff6b35 → #dc2626 (top to bottom) |
| Logo | Campfire emoji (🔥) at 64px, centered vertically at 40% screen height |
| App name | "Campfire" in Display (28px/700), color #fef3c7 (warm white), 8px below emoji |
| Loading indicator | ActivityIndicator, color #fef3c7, 48px below app name |
| Safe area | Respect top + bottom safe area insets |

**Animation (Claude's discretion — source: CONTEXT.md):** Fade-in the logo + name over 300ms on mount. No splash animation beyond that — keep cold start fast.
