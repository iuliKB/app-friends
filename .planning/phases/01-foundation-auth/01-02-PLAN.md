**SECTION 4: SECURITY DEFINER helper function (prevents RLS recursion on friendships)**
```sql
CREATE OR REPLACE FUNCTION public.is_friend_of(target_user uuid)
RETURNS boolean
LANGUAGE sql STABLE SECURITY DEFINER
SET search_path = ''
AS $$
  SELECT EXISTS (
    SELECT 1 FROM public.friendships
    WHERE status = 'accepted'
      AND (
        (requester_id = (SELECT auth.uid()) AND addressee_id = target_user)
        OR (addressee_id = (SELECT auth.uid()) AND requester_id = target_user)
      )
  );
$$;
```

**SECTION 5: RLS Policies**

For `profiles`:
- SELECT: authenticated users can read own profile AND friends' profiles (using is_friend_of)
- Also allow SELECT by username for friend search: any authenticated user can read id, username, display_name, avatar_url (needed for search)
- INSERT: own row only (WITH CHECK id = auth.uid())
- UPDATE: own row only (USING + WITH CHECK id = auth.uid())

Simplify profiles SELECT to: authenticated can read all profiles (needed for username search). This is safe because profiles only contain public info (username, display name, avatar). The sensitive data is in other tables.

For `statuses`:
- SELECT: own status OR friend's status (using is_friend_of(user_id))
- INSERT: own row only (WITH CHECK user_id = auth.uid())
- UPDATE: own row only (USING + WITH CHECK user_id = auth.uid())

For `friendships`:
- SELECT: rows where auth.uid() is requester_id OR addressee_id
- INSERT: only as requester (WITH CHECK requester_id = auth.uid())
- UPDATE: only addressee can update status (USING addressee_id = auth.uid(), WITH CHECK addressee_id = auth.uid())
- DELETE: either party can delete (USING requester_id = auth.uid() OR addressee_id = auth.uid())

For `plans`:
- SELECT: plan members can read (user is in plan_members for this plan_id)
- INSERT: any authenticated user (WITH CHECK created_by = auth.uid())
- UPDATE: creator only (USING + WITH CHECK created_by = auth.uid()) — EXCEPT link_dump and iou_notes editable by any member (separate policy or handle via RPC)
- For simplicity: UPDATE by any plan member (USING user_id in plan_members)

For `plan_members`:
- SELECT: plan members can see all members of their plans
- INSERT: plan creator can add members (check via plans.created_by)
- UPDATE: own RSVP only (USING + WITH CHECK user_id = auth.uid())

For `dm_channels`:
- SELECT: participant only (user_a = auth.uid() OR user_b = auth.uid())
- INSERT: participant only (WITH CHECK user_a = auth.uid() OR user_b = auth.uid())

For `messages`:
- SELECT: can read if member of the plan OR participant of the DM channel
- INSERT: sender_id must be auth.uid(), and must be member of the plan/DM (WITH CHECK)

CRITICAL: Every policy must use `(SELECT auth.uid())` wrapping (not bare `auth.uid()`) for query plan caching performance per PITFALLS.md.

**SECTION 6: Triggers**

handle_new_user trigger (from RESEARCH.md Pattern 7):
```sql
CREATE OR REPLACE FUNCTION public.handle_new_user()
RETURNS trigger
LANGUAGE plpgsql
SECURITY DEFINER SET search_path = ''
AS $$
BEGIN
  INSERT INTO public.profiles (id, display_name)
  VALUES (
    new.id,
    COALESCE(
      new.raw_user_meta_data ->> 'full_name',
      new.raw_user_meta_data ->> 'name',
      split_part(new.email, '@', 1)
    )
  );
  -- Auto-create status row with default 'maybe'
  INSERT INTO public.statuses (user_id)
  VALUES (new.id);
  RETURN new;
END;
$$;

CREATE TRIGGER on_auth_user_created
  AFTER INSERT ON auth.users
  FOR EACH ROW EXECUTE PROCEDURE public.handle_new_user();
```

get_friends() — returns friends of current user:
```sql
CREATE OR REPLACE FUNCTION public.get_friends()
RETURNS TABLE (
  friend_id uuid,
  username text,
  display_name text,
  avatar_url text,
  friendship_status friendship_status,
  created_at timestamptz
)
LANGUAGE sql STABLE SECURITY DEFINER
SET search_path = ''
AS $$
  SELECT
    CASE
      WHEN f.requester_id = (SELECT auth.uid()) THEN f.addressee_id
      ELSE f.requester_id
    END AS friend_id,
    p.username,
    p.display_name,
    p.avatar_url,
    f.status AS friendship_status,
    f.created_at
  FROM public.friendships f
  JOIN public.profiles p ON p.id = CASE
    WHEN f.requester_id = (SELECT auth.uid()) THEN f.addressee_id
    ELSE f.requester_id
  END
  WHERE (f.requester_id = (SELECT auth.uid()) OR f.addressee_id = (SELECT auth.uid()))
    AND f.status = 'accepted';
$$;
```

get_free_friends() — returns friends with status 'free':
```sql
CREATE OR REPLACE FUNCTION public.get_free_friends()
RETURNS TABLE (
  friend_id uuid,
  username text,
  display_name text,
  avatar_url text,
  status availability_status,
  context_tag text,
  status_updated_at timestamptz
)
LANGUAGE sql STABLE SECURITY DEFINER
SET search_path = ''
AS $$
  SELECT
    gf.friend_id,
    gf.username,
    gf.display_name,
    gf.avatar_url,
    s.status,
    s.context_tag,
    s.updated_at AS status_updated_at
  FROM public.get_friends() gf
  JOIN public.statuses s ON s.user_id = gf.friend_id
  WHERE s.status = 'free'
  ORDER BY s.updated_at DESC;
$$;
```

get_or_create_dm_channel(other_user_id uuid) — returns or creates DM channel:
```sql
CREATE OR REPLACE FUNCTION public.get_or_create_dm_channel(other_user_id uuid)
RETURNS uuid
LANGUAGE plpgsql SECURITY DEFINER
SET search_path = ''
AS $$
DECLARE
  channel_id uuid;
  current_user_id uuid := (SELECT auth.uid());
BEGIN
  -- Check existing channel
  SELECT id INTO channel_id
  FROM public.dm_channels
  WHERE (user_a = least(current_user_id, other_user_id)
    AND user_b = greatest(current_user_id, other_user_id));

  IF channel_id IS NOT NULL THEN
    RETURN channel_id;
  END IF;

  -- Create new channel
  INSERT INTO public.dm_channels (user_a, user_b)
  VALUES (least(current_user_id, other_user_id), greatest(current_user_id, other_user_id))
  RETURNING id INTO channel_id;

  RETURN channel_id;
END;
$$;
```
