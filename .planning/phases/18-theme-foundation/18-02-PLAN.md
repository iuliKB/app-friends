**Change 2: Wrap GestureHandlerRootView children with ThemeProvider**

In the main authenticated `return` block (NOT in the splash `if (!ready || !fontsLoaded)` early return), find:
```tsx
<GestureHandlerRootView style={{ flex: 1, backgroundColor: COLORS.surface.base }}>
  <OfflineBanner />
  <Stack ...>
```
Change to:
```tsx
<GestureHandlerRootView style={{ flex: 1, backgroundColor: COLORS.surface.base }}>
  <ThemeProvider>
    <OfflineBanner />
    <Stack ...>
      ...
    </Stack>
  </ThemeProvider>
</GestureHandlerRootView>
```

**Important constraints (from RESEARCH.md Anti-Patterns):**
- ThemeProvider must be INSIDE `GestureHandlerRootView`, not outside — `GestureHandlerRootView` remains the outermost element
- The splash early-return (`if (!ready || !fontsLoaded)`) must NOT be wrapped — that renders with static `COLORS` directly. ThemeProvider only wraps the main authenticated navigation tree in the final return.
- `GestureHandlerRootView`'s own `style` prop still uses static `COLORS.surface.base` — do NOT change that style to use `useTheme()` (D-09: compat shim, this file is not migrated in Phase 18)
- `OfflineBanner` does not call `useTheme()` so it is safe inside ThemeProvider even though it uses static COLORS internally
