```typescript
export type ThemePreference = 'light' | 'dark' | 'system';

const STORAGE_KEY = 'campfire:theme';

type ThemeContextValue = {
  colors: typeof DARK;
  isDark: boolean;
  theme: ThemePreference;
  setTheme: (t: ThemePreference) => void;
};
```

**ThemeProvider implementation (D-02, D-03, THEME-03):**
- `useState<ThemePreference>('system')` — initial state is 'system' (D-02)
- `useColorScheme()` — synchronous, called unconditionally at top of component
- `useEffect` with `AsyncStorage.getItem('campfire:theme')`:
  - Accept only `'light' | 'dark' | 'system'` values (validates before setState — security per RESEARCH.md §Security)
  - `.catch(() => {})` — silent fallback to 'system'
- `setTheme` via `useCallback`: calls `setThemeState(t)` then fire-and-forget `AsyncStorage.setItem(STORAGE_KEY, t)` with `.catch(() => console.warn('[ThemeProvider] Failed to persist theme preference'))`
- Palette resolution: `const effectiveScheme = theme === 'system' ? (systemScheme ?? 'dark') : theme;` (null-safe: D-02 + RESEARCH.md Pitfall 5)
- `const isDark = effectiveScheme === 'dark';`
- `const colors = isDark ? DARK : LIGHT;`

**useTheme hook (D-04):**
```typescript
export function useTheme(): ThemeContextValue {
  const ctx = useContext(ThemeContext);
  if (!ctx) throw new Error('useTheme must be used inside ThemeProvider');
  return ctx;
}
```

**Important:** ThemeContext itself is NOT exported from this file (it is an internal implementation detail). Only export: `ThemePreference` type, `ThemeProvider` function, `useTheme` function, and `LIGHT` (re-exported for barrel access).

Do NOT use module-scope `StyleSheet.create` — this file has no JSX styles.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| AsyncStorage → ThemeProvider | Stored value read on mount; untrusted (could be tampered on jailbroken device) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-01 | Tampering | AsyncStorage `campfire:theme` read | mitigate | Validate stored value: `if (stored === 'light' \|\| stored === 'dark' \|\| stored === 'system')` before calling setState — any other value is silently ignored, falling back to 'system' |
| T-18-02 | Information Disclosure | Theme preference value | accept | `'light' \| 'dark' \| 'system'` contains no PII; acceptable low-value data |
| T-18-03 | Denial of Service | AsyncStorage read failure | mitigate | `.catch(() => {})` in hydration useEffect — failure silently defaults to 'system', app continues rendering |
</threat_model>
