## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap in FAB, avatar stack overlap offset |
| sm | 8px | Inline padding on pills, compact row gaps |
| md | 16px | Horizontal screen padding, field vertical padding, card internal spacing |
| lg | 24px | Section padding, FAB bottom offset from insets |
| xl | 32px | Major section breaks, empty state top padding |
| 2xl | 48px | Empty state top padding (matches HomeScreen pattern) |
| 3xl | 64px | Bottom scroll padding to clear FAB |

Exceptions:
- FAB touch area: `paddingHorizontal: 16, paddingVertical: 16` — matches existing HomeScreen FAB
- RSVP buttons: minimum touch target 44px height (accessibility)
- Avatar stack overlap: 10px (intentional, not on scale — matches existing AvatarStack pattern)
- Plan card avatar stack avatar size: 28px (not on scale — small avatar context)

**Source:** Extracted from HomeScreen.tsx, PrimaryButton.tsx, and AvatarStack pattern in RESEARCH.md

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 | 1.5 |
| Label | 14px | 600 | 1.2 |
| Heading | 20px | 600 | 1.2 |
| Display | 24px | 600 | 1.2 |

Usage notes:
- **Body (16px/400):** Field inputs (TextInput), location text, link dump content, IOU notes content, section body copy
- **Label (14px/600):** Status pills, RSVP pills, time sub-labels, "Creator" badge, "In 2 hours" relative time segment, member name in compact rows, section group headers ("Going (3)", "Maybe (1)")
- **Heading (20px/600):** Plan card title, plan dashboard section headers ("Details", "Who's Going", "Links", "IOU Notes"), Quick Plan modal field labels
- **Display (24px/600):** Plans list screen heading, plan dashboard plan title in header

**Source:** Extracted from HomeScreen.tsx styles (`countHeading: 24/600`, `emptyHeading: 20/600`, `emptyBody: 16`, `sectionLabel: 18/600` — normalized to 20 for consistency), PrimaryButton.tsx (`text: 16/600`), StatusPill.tsx (`label: 14/600`)

## Component Inventory

New components to create for Phase 4. All use React Native StyleSheet only.

### PlanCard
- Plan title (Heading/600/20px, textPrimary)
- Smart time label (Label/600/14px, textSecondary) — e.g. "In 2 hours • 7:00 PM"
- Location (Body/400/16px, textSecondary) — omit if null
- RSVP summary (Label/600/14px, textSecondary) — e.g. "3 going, 1 maybe"
- AvatarStack (max 5 visible + "+N" overflow badge, 28px size, 10px overlap)
- Background: secondary (#2a2a2a), borderRadius 12, padding md (16px)

### AvatarStack
- Overlapping AvatarCircle components, absolute-offset with `marginLeft: -10` per subsequent avatar
- "+N" overflow badge: secondary background, accent text (#f97316), Label/600/14px, 28px circle
- Max 5 visible avatars. Source: RESEARCH.md Pattern 11 / code example

### RSVPButtons
- Three buttons in a horizontal row: "Going", "Maybe", "Out"
- Active state: full background fill using semantic colors (green/yellow/red)
- Inactive state: secondary (#2a2a2a) background, secondary text (#9ca3af), border (#3f3f46)
- Loading state: ActivityIndicator replacing button label during server confirmation
- Height: 44px minimum (touch target), borderRadius 8, flex: 1 each, gap sm (8px)
- Label: Label/600/14px

### MemberList
- FlatList with section headers grouped by RSVP: "Going (N)", "Maybe (N)", "Invited (N)", "Not Going (N)"
- Section header: Label/600/14px, textSecondary
- Member row: AvatarCircle (32px) + display name (Body/400/16px, textPrimary) + "Creator" badge if applicable
- "Creator" badge: Label/600/12px, accent (#f97316), no background fill — just colored text label
- "Not Going" members: opacity 0.5 (dimmed)

### LinkDumpField / IOUNotesField
- Expandable section: header row (chevron icon + label, Heading/600/20px) is tappable to toggle
- Collapsed: shows only header
- Expanded: shows multiline TextInput below header
- TextInput: Body/400/16px, textPrimary, secondary background (#2a2a2a), borderRadius 8, padding md (16px), minHeight 80px
- Save-on-blur: brief ActivityIndicator replacing chevron icon during save

### QuickPlanModal
- Full-screen modal, slides up from bottom (Expo Router `presentation: 'modal'`)
- Header: "New Plan" (Display/600/24px) + close button (Ionicons "close", textSecondary)
- Title field: TextInput, pre-filled "Tonight" or "Tomorrow", Body/400/16px
- Time field: native DateTimePicker trigger, secondary background row
- Location field: TextInput placeholder "My place, a park, TBD…", Body/400/16px
- Friend selector: FlatList, each row = AvatarCircle (40px) + display name + StatusPill + checkbox
- Pre-checked: friends with status "free"
- Create button: PrimaryButton ("Create Plan"), disabled when title is empty
- Background: dominant (#1a1a1a)

### PlanDashboardScreen
- Navigation header: plan title (Display/600/24px) as stack header title + back arrow
- 4 sections with section headers (Heading/600/20px): "Details", "Who's Going", "Links", "IOU Notes"
- Edit mode toggle: "Edit" button (Label/600/14px, accent #f97316) top-right of Details section; switches to "Save" + "Cancel" pair when active
- Edit mode fields: title (TextInput), time (DateTimePicker trigger), location (TextInput) — identical styling to QuickPlanModal fields
- "Open Chat" button: PrimaryButton full-width, at bottom of scrollable content, label "Open Chat"
- Delete plan: Ionicons "trash-outline" icon button, textSecondary color, placed in navigation header right side (creator only)

## Interaction States

### Quick Plan Creation
1. User taps FAB (Home screen or Plans list) → modal slides up from bottom
2. Title field pre-filled "Tonight" / "Tomorrow" based on time of day
3. Time field defaults to next round hour
4. Friend list loads with free friends pre-checked; others unchecked
5. "Create Plan" button disabled while title is empty
6. On submit: button shows ActivityIndicator; on success navigate to new plan dashboard; on error show Alert

### RSVP
1. User taps "Going", "Maybe", or "Out"
2. Tapped button shows ActivityIndicator immediately (server confirmation)
3. On success: tapped button shows active state (filled color), others show inactive
4. On error: Alert "Couldn't update your RSVP. Try again."

### Edit Mode (Dashboard Details)
1. User taps "Edit" button → title, time, location fields become editable; "Edit" replaced by "Save" + "Cancel" pair
2. "Save" submits changes to Supabase; shows ActivityIndicator during save
3. "Cancel" reverts all field values to pre-edit state without saving
4. Any member (not only creator) can enter edit mode (per Phase 1 RLS decision)

### Expandable Sections (Links / IOU Notes)
1. Section header row is tappable (chevron rotates 90° open/closed)
2. Collapsed by default on first open
3. Expanded: TextInput multiline visible
4. On blur: saves to Supabase (ActivityIndicator briefly replaces chevron during save)

### Delete Plan
1. Trash icon in navigation header (creator only — hide for non-creators)
2. Alert with "Delete this plan?" / "This can't be undone." / "Delete" (destructive) + "Cancel"
3. On confirm: DELETE request to Supabase; navigate back on success

### Plans Tab Badge
1. Count = plan_members rows where user_id = current user AND rsvp = "invited"
2. Badge updates on tab focus (useFocusEffect pattern, same as pending friend requests)
3. Badge hidden when count = 0
