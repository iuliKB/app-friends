## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline gaps between icon and text, status badge padding |
| sm | 8px | Avatar-to-text gap in compact rows, badge padding |
| md | 16px | Default horizontal padding on all screens, card internal padding |
| lg | 24px | Section header top margin, between major section groups |
| xl | 32px | Bottom padding on scroll views, logout button top margin on profile |
| 2xl | 48px | Large section breaks (logout row top margin existing: 48px) |
| 3xl | 64px | Screen top padding (existing: `paddingTop: 64` on profile, chat screens) |

Exceptions:
- Profile header avatar: 80px diameter (AvatarCircle size prop — from CONTEXT.md decision)
- Touch row minimum height: 52px (existing: `minHeight: 52` on PrimaryButton and profile rows)
- FriendCard minimum height: 64px (existing: `minHeight: 64` on FriendCard)
- EmptyState icon: 48px Ionicons (centred in flex:1 container)
- Notification toggle row: minimum height 52px, matches existing profile rows

Source: `src/app/(tabs)/profile.tsx`, `src/components/common/PrimaryButton.tsx`, `src/components/friends/FriendCard.tsx`.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (regular) | 1.5 |
| Label | 14px | 400 (regular) | 1.4 |
| Heading | 20px | 600 (semibold) | 1.2 |
| Display | 28px | 600 (semibold) | 1.2 |

**Usage map:**
- Body (16/400): Row labels, form input text, message text, edit screen field values
- Label (14/400): Usernames (@handle), section headers (uppercase), timestamps, char count on edit screen
- Heading (20/600): Screen titles ("Profile", "Edit Profile", friend display name on profile header), section titles on other-user profile screen
- Display (28/600): Not used in this phase — reserved for future marketing surfaces

**Specific to Phase 6:**
- Profile tab header: display name at 20px/600 below avatar (replaces email text at 16px/400)
- Edit profile char count: 14px/400, `COLORS.textSecondary`
- EmptyState heading: 18px/600, `COLORS.textPrimary` (established in RESEARCH.md Pattern 6)
- EmptyState body: 14px/400, `COLORS.textSecondary`
- Other-user profile @username: 14px/400, `COLORS.textSecondary`
- Other-user profile status: 16px/400 with status colour dot

Source: `src/app/(tabs)/profile.tsx` — `heading: fontSize 20/600`, `email: fontSize 16/400`, `sectionHeader: fontSize 14/400`. RESEARCH.md Pattern 6.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#1a1a1a` | App background, screen backgrounds (`backgroundColor: COLORS.dominant`) |
| Secondary (30%) | `#2a2a2a` | Cards, input backgrounds, tab bar, AvatarCircle fallback background, toggle track |
| Accent (10%) | `#f97316` | Primary CTA buttons, initials text in AvatarCircle, pencil edit icon on profile header |
| Destructive | `#ef4444` | "Remove Friend" action button, Busy status indicator, inline error messages, friend request alert badge |

**Status colours (semantic — not in the 60/30/10 split, applied only to status indicators):**
| Status | Value | Reserved for |
|--------|-------|-------------|
| Free | `#22c55e` | Status pill/badge, status dot on other-user profile, SegmentedControl Free segment active state |
| Busy | `#ef4444` | Status pill/badge, status dot (shares value with `COLORS.destructive`) |
| Maybe | `#eab308` | Status pill/badge, status dot, SegmentedControl Maybe segment active state |

**Accent reserved for:**
1. PrimaryButton background (`backgroundColor: COLORS.accent`)
2. AvatarCircle initials text when no image is set
3. Pencil/edit icon on profile tab header (new in this phase)
4. Notification toggle thumb when enabled (active state)

**Text colours:**
- Primary text: `#f5f5f5` (`COLORS.textPrimary`) — all body content, headings
- Secondary text: `#9ca3af` (`COLORS.textSecondary`) — subtext, section headers, timestamps, char counts
- Border: `#3f3f46` (`COLORS.border`) — row separators, card borders, toggle track

Source: `src/constants/colors.ts` — all values already defined and in use across Phases 1–5. REQUIREMENTS.md UIPOL-04.

## Component Inventory (Phase 6 New Components)

### EmptyState (new — `src/components/common/EmptyState.tsx`)

**Props:**
```
icon: string          — emoji character OR Ionicons name
iconType?: 'emoji' | 'ionicons'  — default 'emoji'
heading: string       — 18px/600, COLORS.textPrimary
body: string          — 14px/400, COLORS.textSecondary
ctaLabel?: string     — renders PrimaryButton below body if provided
onCta?: () => void
```

**Layout:** flex:1 container, `alignItems: 'center', justifyContent: 'center'`, icon top, heading 16px below icon, body 8px below heading, CTA button 24px below body (full width, 52px height).

**Instances and copy (see Copywriting Contract below):**
- Home screen: no free friends
- Plans list: no active plans
- Chat list: no conversations
- Friends list: no friends yet
- Friend requests: no pending requests

### LoadingIndicator (new — `src/components/common/LoadingIndicator.tsx`)

**Props:**
```
color?: string     — default COLORS.textSecondary
size?: 'small' | 'large'  — default 'large'
style?: ViewStyle
```

**Layout:** `<ActivityIndicator>` centred in flex:1 (`alignItems: 'center', justifyContent: 'center'`). Replaces all inline `<ActivityIndicator style={styles.loader} />` usages.

### Profile Edit Screen (new — `src/app/profile/edit.tsx`)

**Layout:** ScrollView with `paddingHorizontal: 16`, `paddingTop: 32`

**Top section:** AvatarCircle at 80px centred horizontally. Below avatar: "Change Photo" label at 14px/400, `COLORS.accent`. Tapping avatar or label opens Alert.alert action sheet with "Choose from Library" / "Take Photo" / "Cancel".

**Form fields:**
- Display name TextInput: 16px/400 text, `COLORS.textPrimary` on `COLORS.secondary` background, 12px border-radius, 52px height, `paddingHorizontal: 16`. Placeholder: "Display name" in `COLORS.textSecondary`.
- Char count: right-aligned below field, 12px/400, `COLORS.textSecondary`, format: "32/50"
- Save button: PrimaryButton ("Save Changes"), disabled when display name is empty or unchanged, shows ActivityIndicator on save in progress
- Cancel: navigation back arrow in header (no separate cancel button)

**Header:** Stack navigation header, title "Edit Profile", `COLORS.textPrimary` title, `COLORS.dominant` background, back arrow in `COLORS.textPrimary`.

### Other-User Profile Screen (new — `src/app/friends/[id].tsx`)

**Header:** Stack navigation header, friend's display name as title, back arrow.

**Layout:** ScrollView, `paddingHorizontal: 16`, `paddingTop: 32`

**Top section (centred):**
- AvatarCircle at 80px, centred
- Display name: 20px/600, `COLORS.textPrimary`, marginTop 16px
- @username: 14px/400, `COLORS.textSecondary`, marginTop 4px
- Status dot + status text: row centred, 16px/400, status colour, marginTop 8px. Dot: 10px circle in status colour, marginRight 6px.
- Emoji tag (if set): displayed inline after status text, same row.

**Action buttons (below top section, marginTop 24px):**
- "Start DM": PrimaryButton, full width, 52px, `COLORS.accent` background, label "Message [First Name]"
- "Remove Friend": text-only button, 52px height, `COLORS.destructive` text at 16px/400, marginTop 8px. On press: Alert.alert confirmation (see Copywriting Contract).

**No mutual plans section** — keep V1 simple per CONTEXT.md decision.

### Profile Tab Redesign (existing — `src/app/(tabs)/profile.tsx`)

**New header section (replaces plain "Profile" heading + email):**
- AvatarCircle at 80px, centred, tappable (opens Edit Profile)
- Pencil icon (Ionicons "pencil-outline", 16px, `COLORS.accent`) overlaid bottom-right of avatar, in a 24px circle with `COLORS.secondary` background
- Display name: 20px/600, `COLORS.textPrimary`, marginTop 12px, centred
- Tapping display name or avatar navigates to `profile/edit`

**Notification toggle row (new, added below QR Code row in FRIENDS section or as new NOTIFICATIONS section):**
- New section header: "NOTIFICATIONS" (14px/400, `COLORS.textSecondary`, uppercase)
- Row with bell icon (Ionicons "notifications-outline", 20px, `COLORS.textSecondary`)
- Label: "Plan invites" at 16px/400, `COLORS.textPrimary`
- Switch component (`<Switch>` React Native) on right side, thumb `COLORS.accent` when on, track `COLORS.secondary`
- Toggle state persisted via AsyncStorage key `'campfire:notifications_enabled'`

## Interaction States

### Edit Profile Screen

| State | Visual |
|-------|--------|
| Form idle, unchanged | Save Changes button disabled (opacity 0.5) |
| Form dirty, name non-empty | Save Changes button enabled (full opacity) |
| Display name empty | Save Changes button disabled, no inline error needed |
| Saving in progress | Save Changes button shows ActivityIndicator, input disabled |
| Save success | Navigate back to Profile tab (server confirmation pattern) |
| Upload in progress | AvatarCircle shows ActivityIndicator overlay; "Change Photo" label shows "Uploading..." |

### Notification Toggle

| State | Visual |
|-------|--------|
| Enabled | Switch ON, thumb `COLORS.accent` |
| Disabled | Switch OFF, thumb `COLORS.border` |
| Toggling on (permission not yet granted) | System permission dialog appears; on grant: token registers, switch stays ON |
| Toggling on (permission previously denied) | System settings link shown via Alert.alert: "Go to Settings to enable notifications" |
| Toggling off | Token NOT deleted from `push_tokens` table — just stops registration prompt; AsyncStorage key set to false |

### Other-User Profile Screen

| State | Visual |
|-------|--------|
| Loading | LoadingIndicator (large, centred, `COLORS.textSecondary`) |
| Loaded | Full profile layout as described above |
| Friend already removed | Navigate back to previous screen automatically |
