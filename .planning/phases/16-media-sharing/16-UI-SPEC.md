## Spacing Scale

Project uses a 4-base scale defined in `src/theme/spacing.ts`. Map to template slots:

| Token | Project Token | Value | Usage |
|-------|--------------|-------|-------|
| xs | `SPACING.xs` | 4px | Avatar overlap offset, tight gaps, icon-label gap |
| sm | `SPACING.sm` | 8px | Separator height, inline gaps, icon-label gap |
| md | `SPACING.md` | 12px | Card section gaps, separator marginLeft |
| lg | `SPACING.lg` | 16px | Standard screen padding, card padding |
| xl | `SPACING.xl` | 24px | Large section spacing, FAB bottom offset |
| 2xl | `SPACING.xxl` | 32px | Screen header top, auth header padding |

Exceptions:
- SendBar row height: 52px hardcoded (existing container — do not change)
- Reply bar height: 48px hardcoded (existing — do not change)
- Photo icon touch target: minimum 44x44pt (accessibility requirement per skill `touch-target-size`)
- Image viewer close/save buttons: minimum 44x44pt touch targets
- Full-screen modal black background: `#000000` (not a surface token)

Source: `src/theme/spacing.ts` + `src/components/chat/SendBar.tsx` (hardcoded heights are pre-existing).

## Color

| Role | Value | Token | Usage |
|------|-------|-------|-------|
| Dominant (60%) | `#1a1a1a` | `COLORS.surface.base` | App background, chat screen background |
| Secondary (30%) | `#2a2a2a` | `COLORS.surface.card` | SendBar container, image bubble background (others), modal backdrop fill |
| Accent (10%) | `#f97316` | `COLORS.interactive.accent` | Photo icon in SendBar, send button active state, own-side image bubble border (none by default), pending spinner tint |
| Destructive | `#ef4444` | `COLORS.interactive.destructive` | Delete action in context menu (no new destructive actions in this phase) |

Accent reserved for:
1. Photo icon (`image-outline`) in SendBar — active/tappable state
2. Send button when `canSend = true` (existing — unchanged)
3. ActivityIndicator `color` prop on optimistic image bubble spinner
4. Own-bubble image background fallback if `expo-image` renders before CDN URL resolves

Secondary surface color `COLORS.surface.card` is used as:
- Bubble background for others' image messages (consistent with text bubbles)

Own image bubble background: `COLORS.interactive.accent` (consistent with own text bubbles).

Full-screen modal background: `#000000` hardcoded — semantic for full-screen image viewer (approved deviation; existing pattern in codebase).

Spinner overlay behind ActivityIndicator: `rgba(0,0,0,0.3)` semi-transparent `View` centered over bubble — approved deviation from token system (opacity compositing only).

Source: `src/theme/colors.ts`, `src/components/chat/MessageBubble.tsx`, `src/components/chat/SendBar.tsx`.

## Component Inventory

All new components for this phase:

### 1. SendBar photo icon button (extends `SendBar.tsx`)

- **Position:** Inline in SendBar row, between `add-circle` TouchableOpacity and `TextInput`
- **Icon:** `image-outline` from Ionicons — reads clearly as "send a photo" (library-safe, not camera-specific)
- **Size:** 28pt (matches existing `add-circle` at 28pt for visual alignment)
- **Color:** `COLORS.interactive.accent`
- **Touch target:** `minWidth: 44`, `minHeight: 44` — padding via `paddingHorizontal: SPACING.xs`
- **Prop:** `onPhotoPress: () => void` added to `SendBarProps`
- **accessibilityLabel:** `"Attach photo"`

### 2. ActionSheet (Alert.alert pattern)

- **Title:** `"Send Photo"`
- **Message:** `null`
- **Options:**
  - `{ text: 'Photo Library', onPress: handlePhotoLibrary }`
  - `{ text: 'Camera', onPress: handleCamera }`
  - `{ text: 'Cancel', style: 'cancel' }`
- **Platform note:** iOS renders bottom sheet; Android renders dialog. Both acceptable — native is intentional.
- **Camera permission denied alert:**
  - Title: `"Camera Access Needed"`
  - Message: `"Allow Campfire to use your camera in Settings to take photos."`
  - Button: `"Open Settings"` (links to app settings) + `"Cancel"`
- **Media library permission denied alert:**
  - Title: `"Photo Library Access Needed"`
  - Message: `"Allow Campfire to access your photos in Settings."`
  - Button: `"Open Settings"` + `"Cancel"`

### 3. Image bubble variant (extends `MessageBubble.tsx`)

- **Trigger:** `message.message_type === 'image'` AND `message.image_url !== null` (or `message.pending === true` with local URI)
- **Max width:** 240pt (`maxWidth: 240`)
- **Max height:** 320pt (`maxHeight: 320`)
- **Aspect ratio:** `aspectRatio` derived from image natural dimensions via `onLoad` event; fallback to `4/3` (portrait) before load
- **Border radius:** `RADII.pill` top corners (18pt); `RADII.xs` (4pt) on bottom-right (own) or bottom-left (others) — consistent with text bubble tail pattern
- **Own bubble:** `backgroundColor: COLORS.interactive.accent`
- **Others' bubble:** `backgroundColor: COLORS.surface.card`
- **Image component:** `<Image>` from `expo-image` with `contentFit="cover"`, `recyclingKey={message.id}`
- **Pending state overlay:** centered `ActivityIndicator` (`color: COLORS.interactive.accent`, `size: 'large'`) on semi-transparent `rgba(0,0,0,0.3)` `View` covering entire bubble dimensions
- **Context menu:** same long-press modal as text bubbles; "Copy" action omitted for image messages; "Reply" and "Delete" (own only) remain
- **Quoted block preview text:** `"📷 Photo"` (already coded in QuotedBlock fallback)

### 4. ImageViewerModal (`src/components/chat/ImageViewerModal.tsx` — new file)

- **Modal type:** `<Modal animationType="fade" statusBarTranslucent transparent={false}>`
- **Background:** `#000000` (`flex: 1, backgroundColor: '#000'`)
- **Image container:** `<ScrollView minimumZoomScale={1} maximumZoomScale={4} centerContent bouncesZoom>` wrapping `<Image>` from `expo-image` with `contentFit="contain"`, `style={{ width: screenWidth, height: screenHeight }}`
- **Dismiss:** X button top-right OR tap anywhere outside ScrollView content (via `onPress` on surrounding `TouchableWithoutFeedback`)
- **Close button (top-right):**
  - Icon: `close` from Ionicons, size 28pt, color `#f5f5f5` (`COLORS.text.primary`)
  - Position: `absolute, top: SPACING.xl + safeAreaTop, right: SPACING.lg`
  - Touch target: 44x44pt minimum
  - `accessibilityLabel: "Close image viewer"`
- **Save button (top-left):**
  - Icon: `download-outline` from Ionicons, size 24pt, color `#f5f5f5` (`COLORS.text.primary`)
  - Position: `absolute, top: SPACING.xl + safeAreaTop, left: SPACING.lg`
  - Touch target: 44x44pt minimum
  - `accessibilityLabel: "Save to camera roll"`
  - On press: calls `MediaLibrary.saveToLibraryAsync(imageUrl)`; shows permission alert if denied; shows brief success toast on save
- **Props:** `{ visible: boolean; imageUrl: string; onClose: () => void }`
- **Entry point:** `onImagePress` callback prop on `MessageBubble`; wire in `ChatRoomScreen`

## Interaction States

| State | Element | Visual Treatment |
|-------|---------|-----------------|
| Optimistic pending | Image bubble | `opacity: 0.7` on bubble + centered `ActivityIndicator` spinner overlay (`size: large`, `color: COLORS.interactive.accent`) |
| Upload success | Image bubble | Remove spinner, update `image_url` from local URI to CDN URL, clear pending flag |
| Upload failure | Image bubble | Remove placeholder from list; show toast "Photo could not be sent" (FONT_SIZE.sm, `COLORS.interactive.destructive` tint or neutral dark) |
| Pinch-to-zoom active | ImageViewerModal | Native ScrollView zoom handles this; no custom visual treatment needed |
| Save success | Save button | Show brief toast "Saved to camera roll" (FONT_SIZE.sm, `COLORS.text.primary`) |
| Permission denied (camera) | Alert | Native alert with "Camera Access Needed" copy (see above) |
| Permission denied (save) | Alert | Native alert with "Photo Library Access Needed" copy (see above) |
| Long-press on image bubble | Context menu | Same context pill as text bubbles, minus "Copy" action |
