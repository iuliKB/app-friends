# Phase 20: Map Feature - Context

**Gathered:** 2026-04-29
**Status:** Ready for planning

<domain>
## Phase Boundary

Users can attach a map location to a plan (drop a pin on a full-screen map modal); the plan dashboard shows a map tile with a pin; the Explore tab gains a map view with friend plan pins and a list/map toggle; tapping a plan location opens the native maps app for navigation.

Address autocomplete (MAP-F01), Google Maps provider on iOS (MAP-F03), radius filter slider (MAP-F02), and cluster markers are all out of scope for this phase — explicitly deferred to V2 in REQUIREMENTS.md.

</domain>

<decisions>
## Implementation Decisions

### Build Setup
- **D-01:** EAS / custom builds already in use — react-native-maps can be added without additional build setup ceremony.
- **D-02:** Both iOS and Android are in scope.
- **D-03:** Android requires a Google Maps API key. A new key needs to be created (Google Cloud Console, restricted to Android Maps SDK), and wired into `app.config.ts` via an env var (e.g., `EXPO_PUBLIC_GOOGLE_MAPS_KEY`). The plan should include a step for this setup.
- **D-04:** iOS uses the default Apple Maps provider in react-native-maps (no API key needed).

### Database Schema
- **D-05:** Add two new nullable columns to the `plans` table: `latitude FLOAT8 NULL` and `longitude FLOAT8 NULL`. The existing `location TEXT NULL` column stays as-is and becomes the human-readable address label populated by reverse geocoding.
- **D-06:** Plans created before this phase (location text set, no lat/lng) are unaffected — null lat/lng means no map tile, no crash.
- **D-07:** A plan's address label is always generated by reverse geocoding the dropped pin. There is no freetext-without-coordinates path. If a user picks a location, they always get coordinates + an auto-generated address string.

### Location Picker UX (PlanCreateModal / plan edit)
- **D-08:** The existing freetext location input in PlanCreateModal is replaced by a "Add location" button that opens a full-screen LocationPicker modal.
- **D-09:** LocationPicker is a full-screen map modal. The pin stays fixed at screen center; the user moves the map underneath it (map-drag-not-pin-drag pattern). Confirming saves the center coordinates and calls `expo-location reverseGeocodeAsync` to generate the address label.
- **D-10:** The map opens centered on the user's current GPS position (requires foreground location permission from `expo-location`). If permission is denied, fall back to a sensible default region (Claude's discretion on the default coordinate/zoom).
- **D-11:** Reverse geocoding uses `expo-location reverseGeocodeAsync` — OS-native geocoder, free, no API key. Format: street number + street name + city (Claude's discretion on exact string format).

### Plan Dashboard Map Tile
- **D-12:** When a plan has lat/lng, the Plan Dashboard shows a static-looking map tile (a `<MapView>` with `scrollEnabled={false}` and `zoomEnabled={false}`) with a pin at the plan's location.
- **D-13:** The tile is absent when lat/lng is null — no empty placeholder.
- **D-14:** Tapping the address label (or a "Get directions" button near the map tile) opens the native maps app via deep link — `maps://` on iOS, `geo:` on Android. No in-app directions (out of scope per REQUIREMENTS.md).

### Explore Tab Map View
- **D-15:** The Explore tab adds a list/map icon toggle in the ScreenHeader (top-right). Map view is the default when the Explore tab is opened.
- **D-16:** Map view shows pins for all friend plans that have lat/lng within 25km of the user's current location.
- **D-17:** If location permission is denied, fall back to showing all friend plan pins with no radius filter (pan-and-zoom mode). No empty state.
- **D-18:** Tapping a plan pin navigates directly to that plan's dashboard (no callout bubble intermediary).

### Claude's Discretion
- Exact default region/zoom when location permission is denied in LocationPicker
- String format of the reverse-geocoded address label (e.g., "123 Main St, Toronto" vs "Main St, Toronto, ON")
- Visual styling of the map tile on PlanDashboard (height, border radius, pin color — follow design tokens)
- Pin color / marker style for friend plans on Explore map
- Whether to persist the last selected list/map view in Explore tab across sessions

</decisions>
