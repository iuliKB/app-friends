```typescript
useEffect(() => {
  if (!visible) return;
  (async () => {
    const { status } = await Location.requestForegroundPermissionsAsync();
    if (status !== 'granted') return; // map stays on DEFAULT_REGION
    const pos = await Location.getCurrentPositionAsync({ accuracy: Location.Accuracy.Balanced });
    setRegion({
      latitude: pos.coords.latitude,
      longitude: pos.coords.longitude,
      latitudeDelta: 0.01,
      longitudeDelta: 0.01,
    });
  })();
}, [visible]);
```

**MapView setup (CRITICAL — use initialRegion, NOT region prop):**
```tsx
<MapView
  style={StyleSheet.absoluteFillObject}
  initialRegion={DEFAULT_REGION}
  onRegionChangeComplete={(r, { isGesture }) => {
    if (isGesture) setRegion(r);
  }}
  userInterfaceStyle="dark"
/>
```

Do NOT pass `region={region}` — this causes the infinite update loop anti-pattern. The MapView is uncontrolled; only state is updated from gesture events.

**Confirm button handler:**
```typescript
async function handleConfirm() {
  setIsGeocoding(true);
  try {
    const results = await Location.reverseGeocodeAsync({
      latitude: region.latitude,
      longitude: region.longitude,
    });
    const label = results[0]
      ? formatAddress(results[0], region.latitude, region.longitude)
      : `${region.latitude.toFixed(5)}, ${region.longitude.toFixed(5)}`;
    onConfirm({ latitude: region.latitude, longitude: region.longitude, label });
  } finally {
    setIsGeocoding(false);
  }
}
```

**ALWAYS call `requestForegroundPermissionsAsync` before `reverseGeocodeAsync`** — the permission is already requested on mount, but reverseGeocodeAsync must not be called if permission was denied. Add guard:
```typescript
// In handleConfirm, before calling reverseGeocodeAsync:
const { status } = await Location.requestForegroundPermissionsAsync();
if (status !== 'granted') {

  const label = `${region.latitude.toFixed(5)}, ${region.longitude.toFixed(5)}`;
  onConfirm({ latitude: region.latitude, longitude: region.longitude, label });
  return;
}
```

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-20-07 | Information Disclosure | GPS position read on LocationPicker mount | mitigate | Foreground permission requested via `requestForegroundPermissionsAsync` before any GPS read; permission string set in app.config.ts explains usage clearly |
| T-20-08 | Denial of Service | reverseGeocodeAsync call on confirm | accept | OS-native geocoder; no rate limit at this usage level; network failure falls back to raw coords |
| T-20-09 | Tampering | User submits spoofed coordinates via LocationPicker | accept | Coordinates are informational only; no authorization logic depends on them; user can only affect their own plan |
| T-20-10 | Information Disclosure | reverseGeocodeAsync called before permission check | mitigate | handleConfirm explicitly re-checks permission status before calling reverseGeocodeAsync; falls back to raw coords if denied (prevents Android hang per Pitfall 2) |
</threat_model>
