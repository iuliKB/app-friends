<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User input → replyContext state | senderName and previewText are read from the in-memory messages array, which was loaded via RLS-gated SELECT — no new user-controlled string injection |
| FlatList scrollToIndex → UI scroll position | Client-side only; no security boundary |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-04-01 | Tampering | handleSend passing replyToId | mitigate | `reply_to_message_id` FK constraint on the messages table validates the UUID on INSERT; if the ID doesn't exist or the user lacks SELECT access to the referenced message, Postgres FK violation returns an error handled by sendMessage rollback |
| T-14-04-02 | Spoofing | replyContext senderName displayed in reply bar | accept | `senderName` populated from `msg.sender_display_name` which came from a RLS-gated JOIN on the messages query; no direct user input; React Native Text component renders text safely (no HTML injection) |
| T-14-04-03 | Denial of Service | Toast timer accumulation | accept | `showToast` uses `Animated.sequence` + state flag; rapid repeated calls reset the same `toastAnim` value; no timer leak because `setToastVisible(false)` in `.start()` callback and `pointerEvents="none"` on the toast view prevents interaction interference |
| T-14-04-04 | Information Disclosure | scrollToMessage revealing index position | accept | `messages.findIndex()` is client-side state already held by the authenticated user; no new disclosure |
</threat_model>
