Replace the entire content of `squad.tsx`. The new screen has:

**Imports:**
```typescript
import { useState } from 'react';
import { StyleSheet, Text, TouchableOpacity, View } from 'react-native';
import { Ionicons } from '@expo/vector-icons';
import { useRouter } from 'expo-router';
import { useSafeAreaInsets } from 'react-native-safe-area-context';
import { COLORS, SPACING, FONT_SIZE, FONT_WEIGHT } from '@/theme';
import { SquadTabSwitcher } from '@/components/squad/SquadTabSwitcher';
import { FriendsList } from '@/screens/friends/FriendsList';
import { usePendingRequestsCount } from '@/hooks/usePendingRequestsCount';
```

**Component body:**
- `const insets = useSafeAreaInsets();`
- `const router = useRouter();`
- `const { count: pendingCount } = usePendingRequestsCount();`
- `const [activeTab, setActiveTab] = useState<'friends' | 'goals'>('friends');` — Friends is default (SQAD-02)

**JSX structure:**
```
<View style={[styles.container, { paddingTop: insets.top }]}>
  <SquadTabSwitcher activeTab={activeTab} onTabChange={setActiveTab} />

  {activeTab === 'friends' ? (
    <View style={styles.tabContent}>
      {pendingCount > 0 && (
        <TouchableOpacity
          style={styles.requestsRow}
          onPress={() => router.push('/friends/requests')}
          activeOpacity={0.7}
        >
          <Ionicons name="person-add-outline" size={FONT_SIZE.xl} color={COLORS.text.secondary} />
          <Text style={styles.requestsLabel}>Friend Requests ({pendingCount})</Text>
          <Ionicons name="chevron-forward" size={SPACING.lg} color={COLORS.border} />
        </TouchableOpacity>
      )}
      <FriendsList />
    </View>
  ) : (
    <View style={styles.goalsContent}>
      <Ionicons name="lock-closed-outline" size={48} color={COLORS.text.secondary} />
      <Text style={styles.goalsBody}>Group challenges and streaks — coming soon.</Text>
    </View>
  )}
</View>
```

**Styles (StyleSheet.create):**
- `container`: `flex: 1`, `backgroundColor: COLORS.surface.base`
- `tabContent`: `flex: 1` — wraps Friends tab content; no horizontal padding (FriendsList manages its own)
- `requestsRow`: `flexDirection: 'row'`, `alignItems: 'center'`, `backgroundColor: COLORS.surface.card` (#2a2a2a), `paddingHorizontal: SPACING.lg` (16px), `paddingVertical: SPACING.lg` (16px), `marginBottom: SPACING.sm` (8px), `gap: SPACING.md` (12px between icon and text)
- `requestsLabel`: `flex: 1`, `fontSize: FONT_SIZE.lg` (16px), `fontWeight: FONT_WEIGHT.regular` (400), `color: COLORS.text.secondary` (#9ca3af)
- `goalsContent`: `flex: 1`, `alignItems: 'center'`, `justifyContent: 'center'`, `paddingHorizontal: SPACING.xxl` (32px)
- `goalsBody`: `fontSize: FONT_SIZE.lg` (16px), `fontWeight: FONT_WEIGHT.regular` (400), `color: COLORS.text.secondary` (#9ca3af), `marginTop: SPACING.sm` (8px), `textAlign: 'center'`

**Key decisions:**
- No `paddingHorizontal` on container (removed from current stub) — FriendsList has its own horizontal padding, and the requestsRow uses paddingHorizontal internally. This prevents double-padding.
- Friend Requests row is a sibling View ABOVE `<FriendsList />`, not a FlatList ListHeaderComponent — FriendsList owns its FlatList.
- `FriendsList` is only rendered when `activeTab === 'friends'` — this means FAB (inside FriendsList) is automatically hidden on Goals tab.
- `usePendingRequestsCount` is called independently in squad.tsx for the row display. The `_layout.tsx` call is separate for badge display. Both are safe (no Realtime; cheap focus-triggered query).
- No `ScreenHeader` — the SquadTabSwitcher is the primary wayfinding element.
- `paddingTop: insets.top` on container handles safe area (same as current stub).
