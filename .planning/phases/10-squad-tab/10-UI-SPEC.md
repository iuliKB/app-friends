## Spacing Scale

Project spacing tokens from `src/theme/spacing.ts`:

| Token | Value | Usage |
|-------|-------|-------|
| SPACING.xs | 4px | Avatar overlap, tight inline gaps |
| SPACING.sm | 8px | Separator height, icon-label gaps |
| SPACING.md | 12px | Card section gaps, separator marginLeft |
| SPACING.lg | 16px | Standard screen horizontal padding, card padding |
| SPACING.xl | 24px | Large section spacing, FAB bottom offset |
| SPACING.xxl | 32px | Screen header top, auth header padding |

Exceptions:
- `paddingTop: insets.top` — safe area top inset (dynamic, not a fixed token); applied to the outermost container in `squad.tsx` (source: CONTEXT.md decision on no ScreenHeader)
- Underline indicator height: 2px — intentionally sub-grid; fixed visual thickness for tab underline (source: RESEARCH.md Pattern 1)
- Touch targets for tab buttons and the Friend Requests row: minimum 44px tall (SPACING.xl = 24px paddingVertical × 2 = 48px tall) — meets accessibility requirement

**Source:** `src/theme/spacing.ts` (codebase scan)

## Screen Layout Contract

### squad.tsx — outer container

```
[paddingTop: insets.top — dynamic safe area]
[SquadTabSwitcher — full width, 48px tall, border bottom]
[tab content area — flex: 1]
  Friends tab:
    [Friend Requests row — only if pendingCount > 0, SPACING.sm margin-bottom]
    [FriendsList — unchanged, flex: 1, owns its own FAB]
  Goals tab:
    [Goals stub — centered, lock icon + text]
```

- No ScreenHeader title — SquadTabSwitcher is the primary wayfinding element
- `paddingHorizontal: SPACING.lg` on the screen container applies to the Friends tab wrapper and Goals stub; FriendsList manages its own horizontal padding internally

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Tap inactive tab | Haptic (Light), active tab switches, underline moves, content area re-renders |
| Tap active tab | No haptic, no state change (noop) |
| Tap Friend Requests row | `router.push('/friends/requests')` — navigates to existing screen |
| Pull down on FriendsList | Pull-to-refresh — handled internally by FriendsList |
| FAB tap (Friends tab) | Navigates to Add Friend — handled internally by FriendsList |
| FAB on Goals tab | Not rendered — FriendsList is conditionally unmounted when Goals tab is active |
| Tab focus (returning to Squad) | `usePendingRequestsCount` hook inside `squad.tsx` re-runs on focus (uses `useFocusEffect`) — badge and row count refresh |
