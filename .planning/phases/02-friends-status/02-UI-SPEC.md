# Phase 2 — UI Design Contract

> Visual and interaction contract for Phase 2: Friends + Status. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — React Native StyleSheet only |
| Preset | not applicable |
| Component library | none (PROJECT.md: "No UI libraries — React Native StyleSheet only") |
| Icon library | @expo/vector-icons (Ionicons subset) — Expo Go compatible |
| Font | System default (San Francisco on iOS, Roboto on Android) — inherited from Phase 1 |

**Rationale:** Identical to Phase 1. shadcn is web-only and incompatible with React Native. Registry safety gate: not applicable.

---

## Spacing Scale

Declared values (multiples of 4) — inherited from Phase 1 unchanged:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, segmented control inner padding |
| sm | 8px | Between emoji tag buttons, between accept/reject buttons, card internal gap |
| md | 16px | Screen horizontal padding, list item vertical padding, section spacing |
| lg | 24px | Section gaps, card padding, between segmented control and emoji row |
| xl | 32px | Screen top padding for modal sheets |
| 2xl | 48px | Vertical rhythm between major profile sections |
| 3xl | 64px | Not used in Phase 2 |

Exceptions:
- Touch targets minimum 44px height (Apple HIG) — segmented control height 44px, all friend card rows min 44px, emoji tag buttons 44px, accept/reject buttons 44px
- FAB button: 56px diameter circle
- Status pill height: 24px (compact badge — not a tap target itself)
- Bottom sheet action rows: 52px height

---

## Typography

Inherited from Phase 1 — Phase 2 uses all 4 declared sizes:

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (regular) | 1.5 |
| Label | 14px | 400 (regular) | 1.4 |
| Heading | 20px | 600 (semibold) | 1.2 |
| Badge | 12px | 600 (semibold) | 1.2 |

**Phase 2 usage notes:**
- Heading (20px/600): Screen titles ("Friends", "Add Friend", "Friend Requests", "My QR Code")
- Body (16px/400): Display names in friend cards, search input text, bottom sheet action labels, QR hint text
- Label (14px/400): Usernames below display names, status pill text, "Pending" button text, empty state body copy, emoji picker section label, section headers in Profile tab rows; segmented control labels (both inactive 14px/400 and active 14px/600)
- Badge (12px/600): Notification count badges in header and profile rows, count pills

**Weights in use:** 400 (regular) and 600 (semibold) only.

## Color

Inherited from Phase 1 — `src/constants/colors.ts` is the single source of truth:

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #1a1a1a | Screen backgrounds, QR code modal background |
| Secondary (30%) | #2a2a2a | Friend cards, bottom sheet background, segmented control track, search input background, profile section rows |
| Accent (10%) | #f97316 | Campfire orange — primary interactive elements only (see reserved-for list) |
| Destructive | #ef4444 | "Remove friend" action label in bottom sheet, reject request button border/text, error states |

**Accent (#f97316) reserved for Phase 2:**
- FAB button fill (Add Friend)
- Active tab indicator (inherited from Phase 1 navigation shell)
- "Add Friend" button fill on search results and post-QR-scan confirmation card
- Accept request button fill

**Status colours (first active use in Phase 2 — source: PROJECT.md + src/constants/colors.ts):**
- Free: #22c55e — fills status pill + fills active segment when "Free" selected
- Busy: #ef4444 — fills status pill + fills active segment when "Busy" selected
- Maybe: #eab308 — fills status pill + fills active segment when "Maybe" selected

**Status colour usage rules:**
- Status pill background uses the status colour at full opacity; pill text uses #1a1a1a (dominant) for legibility on all three colours
- Active segmented control segment uses the status colour as background fill; active label text uses #1a1a1a (dominant)
- Inactive segmented control segments use #2a2a2a (secondary) background; inactive label text uses #9ca3af (textSecondary)

**Additional semantic colours (inherited):**
- Text primary: #f5f5f5
- Text secondary: #9ca3af (usernames, helper text, hint text, placeholder)
- Border: #3f3f46 (card borders, input borders, bottom sheet top border, segment track inner border)
- Surface overlay: #ffffff14 (bottom sheet backdrop tint behind modal)

**QR code colours:**
- QR foreground (modules): #f5f5f5 (textPrimary)
- QR background: #1a1a1a (dominant)
- Source: RESEARCH.md Pattern 6 verified example adapted to project tokens
