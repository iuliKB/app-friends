**useFriends.ts hook** — Create a custom hook that provides all friend operations using Supabase:

1. `fetchFriends()` — calls `supabase.rpc('get_friends')`, then for each friend fetches their status from `supabase.from('statuses').select('user_id, status, context_tag').in('user_id', friendIds)`. Merges results. Returns array of `{ friend_id, username, display_name, avatar_url, status: StatusValue, context_tag: EmojiTag }`. Sorts client-side: status order (free=0, busy=1, maybe=2), then alphabetical by display_name.

2. `fetchPendingRequests()` — queries `supabase.from('friendships').select('id, requester_id, created_at, profiles!friendships_requester_id_fkey(id, username, display_name, avatar_url)').eq('addressee_id', session.user.id).eq('status', 'pending').order('created_at', { ascending: false })`.

3. `sendRequest(targetUserId: string)` — inserts into friendships with `{ requester_id: session.user.id, addressee_id: targetUserId }`. Handles error.code === '23505' as "Friend request already sent."

4. `acceptRequest(friendshipId: string)` — updates friendships set status='accepted' where id=friendshipId and addressee_id=session.user.id.

5. `rejectRequest(friendshipId: string)` — updates friendships set status='rejected' where id=friendshipId and addressee_id=session.user.id.

6. `removeFriend(friendId: string)` — deletes from friendships using `.or()` filter matching both directions: `and(requester_id.eq.${myId},addressee_id.eq.${friendId}),and(requester_id.eq.${friendId},addressee_id.eq.${myId})`.

7. `searchUsers(query: string)` — queries `supabase.from('profiles').select('id, username, display_name, avatar_url').ilike('username', `%${query.toLowerCase().trim()}%`).limit(10)`. Excludes own user ID from results.

The hook takes no args, reads session from useAuthStore. Each function returns `{ data, error }` pattern. State (friends array, pending requests array) managed with useState, loading flags per operation.

**usePendingRequestsCount.ts** — Lightweight hook that queries `supabase.from('friendships').select('id', { count: 'exact', head: true }).eq('addressee_id', session.user.id).eq('status', 'pending')`. Returns `{ count: number, refetch: () => void }`. Fetches on mount and exposes refetch.
