<objective>
Add a session-ready `useEffect` to `src/app/(tabs)/_layout.tsx` that calls `registerForPushNotifications(userId)` once when the user becomes authenticated AND re-registers on every `AppState` 'active' transition. This is the wiring that closes the silent plan-invite gap on fresh installs (PUSH-05 → satisfied transitively by PUSH-01).

Moving it to a session-ready effect means every authenticated launch registers a token automatically. The `AppState` foreground listener catches APNs/FCM token rotation that happens silently while the app is backgrounded.

```typescript
  // PUSH-01 + PUSH-02 (D-16): register token once per authenticated session,
  // re-register on every AppState 'active' transition to catch OS token rotation.
  // Failures are intentionally swallowed — registration must not crash the tabs layout.
  const userId = useAuthStore((s) => s.session?.user?.id);
  const appState = useRef<AppStateStatus>(AppState.currentState);

  useEffect(() => {
    if (!userId) return;

    // Initial register on session-ready
    registerForPushNotifications(userId).catch(() => {});

    // Foreground re-register
    const sub = AppState.addEventListener('change', (next) => {
      if (appState.current.match(/inactive|background/) && next === 'active') {
        registerForPushNotifications(userId).catch(() => {});
      }
      appState.current = next;
    });

    return () => sub.remove();
  }, [userId]);
```

**3. Do NOT touch** the existing `usePendingRequestsCount`, `useInvitationCount`, or any of the `<Tabs.Screen>` definitions.

**4. Do NOT** call `requestPermissionsAsync` directly here — `registerForPushNotifications` handles permission internally and gates on the `campfire:push_prompt_eligible` AsyncStorage flag (Plan 05).
