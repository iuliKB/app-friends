## Spacing Scale

Declared values from `src/theme/spacing.ts` (project-specific, note: `md` = 12px not 16px):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, tight inline spacing, avatar overlap offsets |
| sm | 8px | Separator height, inline gaps, icon-label gap |
| md | 12px | Card section gaps, internal padding |
| lg | 16px | Standard screen padding, card padding, toggle margin |

- Radar container height: 320px (fixed, within D-22's 300–350px range — Claude's discretion)

- Bubble label bottom margin: 6px (between bubble and name text — no exact token, closest to xs=4)

### RadarBubble (new component)

```
Sizes (D-02):
  Free (ALIVE) = 64px diameter
  Maybe (ALIVE) = 48px diameter
  Busy (ALIVE) = 36px diameter
  DEAD = 36px diameter at 50% opacity
  FADING = same size as ALIVE status but opacity: 0.6

Avatar: AvatarCircle at bubble size, borderRadius = size/2 (RADII.full pattern)
Status gradient: absolute View behind avatar, same dimensions, borderRadius = size/2
  Free: rgba(34,197,94,0.30) center → transparent edge
  Maybe: rgba(234,179,8,0.25) center → transparent
  Busy: rgba(239,68,68,0.20) center → transparent
  FADING: no gradient

Depth effect (D-07): bubbles positioned in upper half of container get scale: 0.92 and opacity multiplier 0.85 applied on top of base size/opacity

Name label: directly below bubble, FONT_SIZE.sm (13px), FONT_WEIGHT.regular (400), color COLORS.text.primary (FADING: COLORS.text.secondary), textAlign: center, maxWidth = bubble size + 16px

Touch target: minimum 44×44px hit slop applied via hitSlop prop when bubble < 44px
```

Source: CONTEXT.md D-02, D-07, D-09, D-11, D-12, D-13. Sizes from CONTEXT.md D-02.

### PulseRing (new sub-component of RadarBubble)

```
Renders only for ALIVE friends (D-09, RADAR-02).
Shape: circular View, border only (no fill), borderRadius = RADII.full
Starting size: bubble diameter
Expanding to: bubble diameter × 1.7
Border color: status color at 70% opacity (start) → 0% opacity (end)
Border width: 2px
Animation: Animated.timing, useNativeDriver: true, isInteraction: false (D-04 pattern)
Duration: 1200ms expand + fade, 600ms pause, loop
Easing: Easing.out(Easing.ease)
```

Source: CONTEXT.md D-09, D-10. Timing from Claude's discretion (within D-09 spirit).

Source: CONTEXT.md D-20, D-22. Size from Claude's discretion (34px, midpoint of 32–36 range).

## Scatter Layout Algorithm Contract

Grid-cells + offset algorithm (D-06):

```
Container dimensions: from onLayout (not Dimensions.get) — RADAR-06
Grid: 3 columns × 2 rows = 6 slots (max friends in radar)
Cell width: containerWidth / 3
Cell height: radarHeight / 2 (= 320px / 2 = 160px)

For each friend slot (up to 6):
  cellCol = slot % 3
  cellRow = Math.floor(slot / 3)
  cellOriginX = cellCol * cellWidth
  cellOriginY = cellRow * cellHeight

  Bubble center X = cellOriginX + random(bubbleRadius + 8, cellWidth - bubbleRadius - 8)
  Bubble center Y = cellOriginY + random(bubbleRadius + 8, cellHeight - bubbleRadius - 8)
  (8px safety margin prevents edge clipping)

Depth effect applied after placement:
  upperHalf = bubbleCenterY < radarHeight / 2
  scaleMultiplier = upperHalf ? 0.92 : 1.0
  opacityMultiplier = upperHalf ? 0.85 : 1.0
  (applied on top of FADING 0.6 opacity where relevant)
```

Positions re-randomized on each HomeScreen mount (D-03). No deterministic seeding.

Source: CONTEXT.md D-03, D-06, D-07, RADAR-06.
