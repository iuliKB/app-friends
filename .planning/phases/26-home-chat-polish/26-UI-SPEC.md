## Spacing Scale

Source: `src/theme/spacing.ts` (locked project tokens — do not introduce new values)

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Avatar overlap offset, tight gaps, icon-label spacing |
| sm | 8px | Separator height, inline gaps, skeleton row gaps |
| md | 12px | Card section gaps, separator marginLeft |

`SPACING.md` (12px) is a locked project token present throughout the codebase since project inception. It is a multiple of 4 (grid-aligned) and deviates from the standard set {4, 8, 16, 24, 32, 48, 64} as an approved project-level decision, not introduced by this phase. All usages are via the token — no raw 12px values appear.

## Color

Source: `src/theme/colors.ts` (dark) + `src/theme/light-colors.ts` (light). Both modes must be respected via `useTheme()`.

| Role | Dark Value | Light Value | Usage |
|------|-----------|------------|-------|
| Dominant (60%) | `#0E0F11` (`colors.surface.base`) | `#F5F6F8` | Screen backgrounds |
| Secondary (30%) | `#1D2027` (`colors.surface.card`) | `#FFFFFF` | Cards, skeleton base fill, chat bubbles (others) |
| Accent (10%) | `#B9FF3B` / `#4D7C00` (`colors.interactive.accent`) | see dark note | Reserved elements only — see list below |
| Fading pulse | `#F59E0B` (amber-500) | `#D97706` (amber-600) | FADING heartbeat ring on radar bubbles ONLY |
| Destructive | `#F87171` / `#DC2626` (`colors.interactive.destructive`) | see dark note | Failure state border/tint on optimistic messages; Delete action in context menu |
| Skeleton shimmer | `colors.surface.overlay` (rgba overlay) | same token | Gradient band inside SkeletonPulse |

**Accent reserved for:**
- Own message bubbles (`ownBubble` background in MessageBubble)
- "Add a friend" CTA button in zero-friends empty state card
- Reaction badge border when user has reacted (`reactionBadgeOwn`)
- ALIVE PulseRing border color (existing — do not change)

**Fading pulse color rationale (Claude's discretion — D-04, D-06):**
Use `#F59E0B` (Tailwind amber-500) in dark mode, `#D97706` (amber-600) in light mode. This is warm yellow-orange — reads as "caution" against both dark (`#0E0F11`) and light (`#F5F6F8`) backgrounds, distinct from the green/red/yellow status colors. The existing `COLORS.status.maybe` is `#FACC15` (yellow-400) — amber is visually distinct and reads warmer/more urgent. Do NOT reuse `status.maybe` for this — they represent different signals.

**Optimistic failure state:**
Red border on bubble: `borderWidth: 2, borderColor: colors.interactive.destructive`. No background tint change — border only to preserve bubble readability. Own bubble background (`colors.interactive.accent`) must remain intact.

## Animation Contract

Source: `src/theme/animation.ts` (locked tokens — no raw numbers permitted)

| Element | Animation | Duration | Easing / Config |
|---------|-----------|----------|----------------|
| SkeletonPulse shimmer | `Animated.loop` translateX | `ANIMATION.duration.verySlow` (1200ms) | `ANIMATION.easing.decelerate()` |
| ALIVE PulseRing | `Animated.loop` scale 1.0→1.7 + opacity 0.7→0 | 1200ms (hardcoded in existing code) | `Easing.out(Easing.ease)` — do not change |
| FADING PulseRing | `Animated.loop` scale 1.0→1.5 + opacity 0.6→0 | 2000ms | `Easing.out(Easing.ease)`, `isInteraction: false` — slower than ALIVE to feel "languid" |
| Home card press (HOME-04) | `Animated.spring` scale 1.0→0.96 on press, 0.96→1.0 on release | spring (no duration) | `ANIMATION.easing.spring` (`damping: 15, stiffness: 120`) |
| Message bubble long-press (CHAT-04) | `Animated.spring` scale 1.0→0.96 on gesture fire, hold while menu open, 0.96→1.0 on close | spring (no duration) | `ANIMATION.easing.spring`, `useNativeDriver: true` |
| Optimistic send confirmation | opacity snap 0.7→1.0 | instant (no animation) | `Animated.timing` 150ms or direct `setValue(1)` |
| Optimistic send failure | no animation — state change only | — | Apply `failed: true` flag; styles respond synchronously |

## Zero-Friends Empty State (HOME-02)

Source: decisions D-07, D-08, D-09, D-10

**Layout:** Inline card within the scroll area. Not a full-screen replacement. Positioned where the friend content section would be (below status card and widgets).

**Card structure:**
```
[Icon: campfire/flame or users icon from Ionicons]
[Heading: "No friends yet"]
[Body: one line of copy — see Copywriting section]
[CTA button: "Add a friend"]
```

**Card styling:**
- Background: `colors.surface.card`
- Border radius: `RADII.lg` (12px)
- Padding: `SPACING.xl` (24px) all sides
- Alignment: center (icon + text + button stacked, `alignItems: 'center'`)
- Icon: `Ionicons` "flame" or "people-outline", size 40, color `colors.text.secondary`
- Heading: `FONT_SIZE.xl` (20px), `FONT_WEIGHT.semibold`, `BricolageGrotesque`, `colors.text.primary`
- Body: `FONT_SIZE.sm` (13px), `FONT_WEIGHT.regular`, PlusJakartaSans, `colors.text.secondary`
- CTA button: styled using the project's `PrimaryButton` component variant (or equivalent), accent background, full-width or auto-width

**CTA behavior:** Switch to Squad tab (bottom nav index 1). Use `router.replace` or tab navigation switch — do not navigate to add-friend flow directly (D-09).

**Trigger condition:** `!loading && friends.length === 0`

## Optimistic Message States (CHAT-03)

Source: decisions D-16, D-17, D-18, D-19

### Pending state (message not yet confirmed by server)
- Bubble opacity: 0.7 (`pendingOpacity` style — already exists in `MessageBubble.tsx`)
- Clock icon: `Ionicons` "time-outline", size 12, color `rgba(245,245,245,0.5)` (dark) / `rgba(26,29,35,0.4)` (light), positioned inline next to timestamp text
- No layout shift: clock icon is inline in the timestamp row, not a separate positioned element

### Confirmed state (server acknowledges)
- Opacity: snap to 1.0 (no animation — instant)
- Clock icon: removed (conditional render on `pending` flag)

### Failed state (send error)
- Border: `borderWidth: 2, borderColor: colors.interactive.destructive`
- Bubble background: unchanged (own bubble stays `colors.interactive.accent`)
- "Tap to retry" label: appears below the bubble, `FONT_SIZE.xs` (12px), `colors.interactive.destructive`, `FONT_WEIGHT.regular`
- Tap target: the entire bubble wrapper re-sends on press

### State machine (in `useChatRoom.ts`)
```
SENT → pending: true, failed: false  (optimistic insert)
SUCCESS → pending: false, failed: false  (real message from realtime replaces, or flag cleared)
FAILURE → pending: false, failed: true
RETRY → pending: true, failed: false  (same as SENT)
```

## Interaction Contracts

### HOME-04: Press Scale Feedback

Apply to every `Pressable` or `TouchableOpacity` that wraps a tappable home card:

1. `HomeFriendCard` — replace `opacity: 0.7` pressed style
2. `OwnStatusCard` — add spring scale if tappable
3. `EventCard` in `UpcomingEventsSection` — add spring scale
4. IOU and birthday widget rows in `HomeWidgetRow` — add spring scale to each Pressable

Pattern:
```
const scaleAnim = useRef(new Animated.Value(1)).current;

function handlePressIn() {
  Animated.spring(scaleAnim, {
    toValue: 0.96,
    ...ANIMATION.easing.spring,
    useNativeDriver: true,
  }).start();
}

function handlePressOut() {
  Animated.spring(scaleAnim, {
    toValue: 1.0,
    ...ANIMATION.easing.spring,
    useNativeDriver: true,
  }).start();
}

<Animated.View style={{ transform: [{ scale: scaleAnim }] }}>
  <Pressable onPressIn={handlePressIn} onPressOut={handlePressOut} ...>
```

Do NOT change routing or tap behavior — only the visual feedback changes.

### CHAT-04: Message Bubble Long-Press Scale

The bubble compresses to 0.96 when the long-press gesture fires and holds compressed while the context menu Modal is visible. Spring back to 1.0 when menu closes (onRequestClose or backdrop tap).

Pattern:
```
const bubbleScaleAnim = useRef(new Animated.Value(1)).current;

function handleLongPress(event) {
  Animated.spring(bubbleScaleAnim, {
    toValue: 0.96,
    ...ANIMATION.easing.spring,
    useNativeDriver: true,
  }).start();
  // existing menu logic...
  setMenuVisible(true);
}

function closeMenu() {
  Animated.spring(bubbleScaleAnim, {
    toValue: 1.0,
    ...ANIMATION.easing.spring,
    useNativeDriver: true,
  }).start();
  setMenuVisible(false);
}
```

Wrap the existing `TouchableOpacity` bubble in `Animated.View` with scale transform. Use `useNativeDriver: true`.
