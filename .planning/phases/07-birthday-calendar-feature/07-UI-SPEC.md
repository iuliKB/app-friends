## Typography

All sizes from `src/theme/typography.ts`. Two weights used in this phase.

| Role | Token | Size | Weight | Line Height | Usage |
|------|-------|------|--------|-------------|-------|
| Card title | FONT_SIZE.lg | 16px | FONT_WEIGHT.semibold (600) | 1.4 | "Birthdays 🎂" card header, screen nav title |
| Body / row name | FONT_SIZE.lg | 16px | FONT_WEIGHT.regular (400) | 1.5 | Friend display_name in birthday rows |
| Secondary info | FONT_SIZE.md | 14px | FONT_WEIGHT.regular (400) | 1.5 | Date label ("Jan 15"), days-until label ("In 3 days"), card count line |
| Caption / muted | FONT_SIZE.sm | 13px | FONT_WEIGHT.regular (400) | 1.4 | "Today" badge text, skeleton placeholder labels |

Rules:
- Exactly 3 effective sizes: 16px (primary), 14px (secondary), 13px (caption/badge).
- Exactly 2 weights: regular (400) for all body/info, semibold (600) for card title only.
- No bold (700) used in this phase — no hardcoded weights.

**Source:** `src/theme/typography.ts` [verified], RESEARCH.md Pattern 6, StreakCard.tsx label/subline styles.

## Component Inventory

Components to build and their visual contracts:

### BirthdayCard (new — `src/components/squad/BirthdayCard.tsx`)

Match StreakCard exactly for card chrome:
- Container: `backgroundColor: COLORS.surface.card`, `borderRadius: RADII.lg` (12px), `paddingVertical: SPACING.xxl` (32px), `paddingHorizontal: SPACING.xl` (24px), `marginHorizontal: SPACING.lg` (16px), `marginTop: SPACING.xl` (24px)
- Pressable with `opacity: 0.85` on pressed state
- Card title: "Birthdays 🎂" — FONT_SIZE.lg (16px), FONT_WEIGHT.semibold, COLORS.text.primary
- Count line: "{N} birthdays in the next 30 days" — FONT_SIZE.md (14px), FONT_WEIGHT.regular, COLORS.text.secondary — count is `entries.filter(e => e.days_until <= 30).length`
- Divider: height 1, COLORS.border, `marginVertical: SPACING.lg`
- Nearest friend row: `flexDirection: 'row'`, `alignItems: 'center'` — AvatarCircle size={32} + display_name (FONT_SIZE.lg, regular, text.primary, flex:1, marginLeft:SPACING.sm) + days label (FONT_SIZE.md, regular, text.secondary)
- Tap target: entire card navigates to `/squad/birthdays`
- Loading state: skeleton (see skeleton spec below)
- Empty state (D-12): "No upcoming birthdays" — FONT_SIZE.md, regular, COLORS.text.secondary, centered

**Source:** CONTEXT.md D-06, D-07, D-08, D-09, D-10, D-12; StreakCard.tsx styles [verified].

### BirthdayCardSkeleton (inline within BirthdayCard)

Match StreakCardSkeleton pattern:
- Wrapper: same card chrome as BirthdayCard, `opacity: 0.5`
- Skeleton title bar: width ~120px, height SPACING.lg (16px), RADII.md (8px), COLORS.border fill
- Skeleton count bar: width ~180px, height SPACING.md (12px), RADII.md, COLORS.border fill, marginTop SPACING.md
- Divider: same as live divider
- Skeleton avatar row: 32px circle (COLORS.border fill) + two bars (name bar ~100px, days bar ~60px)

**Source:** StreakCard.tsx StreakCardSkeleton [verified].

### BirthdayRow (new — inline within BirthdaysScreen or extracted as `src/components/squad/BirthdayRow.tsx`)

Row layout matches FriendCard pattern:
- Container: `flexDirection: 'row'`, `alignItems: 'center'`, `minHeight: 64` (eslint-disable), `paddingHorizontal: SPACING.lg` (16px), `paddingVertical: SPACING.md` (12px)
- Background: COLORS.surface.base normally; `TODAY_BG` (`rgba(249, 115, 22, 0.12)`) when `days_until === 0` (eslint-disable)
- Left: AvatarCircle size={40}, imageUri from avatar_url, displayName from display_name
- Middle: `flex: 1`, `marginLeft: SPACING.lg` (16px)
  - Line 1: display_name — FONT_SIZE.lg (16px), FONT_WEIGHT.regular, COLORS.text.primary
  - Line 2: date string ("Jan 15") — FONT_SIZE.md (14px), FONT_WEIGHT.regular, COLORS.text.secondary
- Right: days label — FONT_SIZE.md (14px), FONT_WEIGHT.regular, COLORS.text.secondary normally; COLORS.interactive.accent when `days_until === 0`
- "Today" badge (when `days_until === 0`): small pill beside the days label or inline — FONT_SIZE.sm (13px), FONT_WEIGHT.regular, COLORS.interactive.accent

Combined date+days format: `"Jan 15 · In 3 days"` / `"Jan 15 · Today"` / `"Jan 15 · Tomorrow"` — date first, then label (D-01, D-02; RESEARCH.md Pattern 3 confirms "date first").

**Source:** CONTEXT.md D-01, D-02, D-03; RESEARCH.md Pattern 6, Code Examples section; FriendCard.tsx row layout [verified].

### BirthdaysScreen (new — `src/app/squad/birthdays.tsx`)

- Root: plain `View` with `flex: 1`, `backgroundColor: COLORS.surface.base` — NOT ScrollView (FlatList inside ScrollView breaks Android)
- FlatList with `flex: 1`, data = birthday entries from `useUpcomingBirthdays()`, `keyExtractor` = `friend_id`
- `ItemSeparatorComponent`: 1px height, COLORS.border — matches UpcomingEventsSection pattern
- `refreshControl`: `RefreshControl` with `onRefresh` calling hook's `refetch`, `tintColor: COLORS.interactive.accent`
- Empty state: `EmptyState` component with heading "No birthdays yet" and body "Ask your friends to add theirs!" (D-11)
- Nav header title: "Birthdays" (D-05) — set via `export const options = { title: 'Birthdays' }`

**Source:** CONTEXT.md D-04, D-05, D-11, D-13; RESEARCH.md Pattern 5, Anti-Patterns section.

### Squad Sub-Route Layout (new — `src/app/squad/_layout.tsx`)

Match `src/app/friends/_layout.tsx` exactly:

## Interaction Contract

| Interaction | Component | Behavior |
|-------------|-----------|----------|
| Tap BirthdayCard | BirthdayCard (Pressable) | opacity: 0.85 pressed, navigates to `/squad/birthdays` |
| Pull to refresh | BirthdaysScreen FlatList | RefreshControl calls `useUpcomingBirthdays().refetch()` |
| Back from BirthdaysScreen | Nav header back button | Standard Expo Router Stack pop — returns to Squad goals tab |
| Loading state | BirthdayCard | BirthdayCardSkeleton replaces card content |
| Error state | BirthdayCard + BirthdaysScreen | Silent fallback to empty state (no error toast) — matches StreakCard D-17 precedent |

No destructive actions in this phase — read-only birthday display only.

## Utility Functions Contract

Two pure utility functions — no React dependency, fully unit-testable.

```typescript
// formatDaysUntil(days: number): string
// 0 → "Today", 1 → "Tomorrow", N → "In N days"

// formatBirthdayDate(month: number, day: number): string
// Uses Intl.DateTimeFormat('en-US', { month: 'short', day: 'numeric' })
// Built into Hermes SDK 55 — no package needed
// Example: formatBirthdayDate(1, 15) → "Jan 15"

// Combined display: `${formatBirthdayDate(m, d)} · ${formatDaysUntil(days)}`
// → "Jan 15 · Today" / "Jan 15 · Tomorrow" / "Jan 15 · In 3 days"
```

30-day filter for card count: `entries.filter(e => e.days_until <= 30).length` — client-side, not in RPC.
Nearest friend: `entries[0]` — RPC returns sorted by `days_until ASC`.
