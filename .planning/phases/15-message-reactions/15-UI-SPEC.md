## Spacing Scale

Project tokens from `src/theme/spacing.ts`:

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| `SPACING.xs` | 4px | Emoji strip item gap; badge pill vertical padding; strip-to-pill gap |
| `SPACING.sm` | 8px | Badge pill horizontal padding; emoji strip row horizontal padding; gap between pill and emoji strip |
| `SPACING.md` | 12px | Not used in this phase |
| `SPACING.lg` | 16px | Not used in this phase |
| `SPACING.xl` | 24px | Not used in this phase |
| `SPACING.xxl` | 32px | Not used in this phase |

Exceptions:
- Emoji tapback button tap target: minimum 44×44pt (React Native touch target rule). Achieved via `minWidth: 44, minHeight: 44` with `alignItems: 'center', justifyContent: 'center'` on the `TouchableOpacity` wrapper. The visible emoji glyph is 24pt; padding brings it to 44pt.
- Badge pill `paddingVertical`: `SPACING.xs` (4px) — tighter than standard, intentional for inline density below bubble.

## Color

Project tokens from `src/theme/colors.ts`:

| Role | Token | Value | Usage |
|------|-------|-------|-------|
| Dominant (60%) | `COLORS.surface.base` | `#1a1a1a` | App background — not modified this phase |
| Secondary (30%) | `COLORS.surface.card` | `#2a2a2a` | Context menu pill background (existing); reaction badge pill background (inactive state) |
| Accent (10%) | `COLORS.interactive.accent` | `#f97316` | Own-reaction badge background tint; already-reacted emoji strip highlight background tint |
| Destructive | `COLORS.interactive.destructive` | `#ef4444` | Not used in this phase |
| Border | `COLORS.border` | `#3f3f46` | Own-reaction badge pill border |
| Text primary | `COLORS.text.primary` | `#f5f5f5` | Badge count text (inactive), emoji strip non-reacted state |
| Text secondary | `COLORS.text.secondary` | `#9ca3af` | Badge count text in inactive pill (contextually appropriate) |
| Overlay | `COLORS.surface.overlay` | `#ffffff14` | Emoji strip row container background (floating row above pill) |

### Accent reserved for (explicit list):

1. **Emoji strip — already-reacted emoji background:** `rgba(249, 115, 22, 0.20)` tint fill behind the emoji the current user has reacted with. Achieved via `backgroundColor: 'rgba(249, 115, 22, 0.20)'` (a 20% opacity of the accent hex — no raw hex in code; use the string literal form as it is a derived alpha, not a new token).
2. **Own-reaction badge pill background:** same `rgba(249, 115, 22, 0.20)` tint.
3. **Own-reaction badge pill border:** `COLORS.interactive.accent` at full opacity (1px border).

All other interactive elements (tapping emoji, tapping badge) use opacity feedback via `activeOpacity={0.7}` on `TouchableOpacity`.

Note on `surface.elevated`: The CONTEXT.md mentions `COLORS.surface.elevated` but this token does not exist in `src/theme/colors.ts`. Use `COLORS.surface.card` (`#2a2a2a`) for all elevated surface backgrounds. Do NOT introduce new color tokens.

## Component Inventory

New components/sub-components required for this phase:

### 1. EmojiStripRow (inline JSX variable or sub-component in MessageBubble.tsx)

Purpose: 6-emoji horizontal tapback row rendered above the contextPill inside the existing Modal.

| Property | Value |
|----------|-------|
| Container layout | `flexDirection: 'row'`, `alignSelf: 'center'`, `gap: SPACING.sm` |
| Container background | `COLORS.surface.overlay` with `borderRadius: RADII.lg`, `paddingHorizontal: SPACING.sm`, `paddingVertical: SPACING.xs` |
| Position | Absolute-positioned above the existing `contextPill` — same Modal, new sibling View |
| Gap between strip and pill | `SPACING.sm` (8px) vertical gap |
| Shadow | Match existing `contextPill` shadow (`shadowColor: '#000'`, `shadowOffset: {width: 0, height: 2}`, `shadowOpacity: 0.3`, `shadowRadius: 8`, `elevation: 8`) |

Per emoji button:

| Property | Value |
|----------|-------|
| Touch target | `minWidth: 44, minHeight: 44` — `alignItems: 'center'`, `justifyContent: 'center'` |
| Emoji glyph size | 24pt (system emoji Text node, no fontSize token — exempt per typography section) |
| Default state | No background, transparent |
| Highlighted state (user reacted with this emoji) | `backgroundColor: 'rgba(249, 115, 22, 0.20)'`, `borderRadius: RADII.full` |
| Accessibility | `accessibilityLabel` per emoji button — e.g. `"React with heart"`, `"React with laughing"` etc. |
| On press | Call `onReact(message.id, emoji)` → close menu |

Emoji set (fixed, in order): `❤️` `😂` `😮` `😢` `👍` `🔥`

### 2. ReactionBadgeRow (inline JSX variable or sub-component in MessageBubble.tsx)

Purpose: Horizontal row of reaction count pills rendered below the message bubble body.

| Property | Value |
|----------|-------|
| Render condition | Only rendered when `message.reactions` array has at least one entry with `count > 0` |
| Container layout | `flexDirection: 'row'`, `flexWrap: 'wrap'`, `gap: SPACING.xs` |
| Alignment | `alignSelf: 'flex-end'` for own messages; `alignSelf: 'flex-start'` for others' messages |
| Top margin | `SPACING.xs` (4px) — tight gap below bubble body |

Per badge pill:

| Property | Value |
|----------|-------|
| Layout | `flexDirection: 'row'`, `alignItems: 'center'`, `gap: SPACING.xs` |
| Padding | `paddingHorizontal: SPACING.sm` (8px), `paddingVertical: SPACING.xs` (4px) |
| Border radius | `RADII.full` (9999) — fully pill-shaped |
| Default (not own reaction) | `backgroundColor: COLORS.surface.card`, no border |
| Own reaction state | `backgroundColor: 'rgba(249, 115, 22, 0.20)'`, `borderWidth: 1`, `borderColor: COLORS.interactive.accent` |

| Count text | `FONT_SIZE.sm` (13px), `FONT_WEIGHT.regular`, `color: COLORS.text.primary` |
| Accessibility | `accessibilityLabel` — e.g. `"Heart reaction, 2 people"` |
| On press | Call `onReact(message.id, emoji)` — toggles own reaction off if `reacted_by_me: true`, otherwise adds |

## Interaction Contracts

### Long-press → context menu with emoji strip

1. User long-presses a message bubble (existing Phase 14 `handleLongPress` behavior).
2. Modal opens. Two visual elements render inside the Modal:
   - **EmojiStripRow** — positioned above the `contextPill`, same `top: pillY` anchor with vertical offset to clear the strip height (~52pt strip + 8pt gap).
   - **contextPill** — existing Phase 14 pill, unchanged structure.
3. Already-reacted emoji (if any) shows highlighted background in the strip.
4. Tap outside (TouchableWithoutFeedback backdrop) dismisses both — existing `closeMenu` behavior.

### Tap emoji in strip

1. `onReact(messageId, emoji)` fires.
2. Optimistic update: immediately update `message.reactions` in local state.
   - If user had a previous reaction: remove it from the array (or set count - 1, `reacted_by_me: false`).
   - Add new emoji (or set count + 1, `reacted_by_me: true`).
3. Close menu.
4. DB write: delete existing reaction for user on this message (if any), insert new reaction.
5. On failure: silently revert local state to pre-tap state.

### Tap emoji in strip (same emoji = toggle off)

1. `onReact(messageId, sameEmoji)` fires.
2. Optimistic update: set count - 1, `reacted_by_me: false`. Remove pill if count reaches 0.
3. DB write: delete reaction row.
4. On failure: silently revert.

### Tap reaction badge pill

Same behavior as tapping the same emoji in the strip: toggles own reaction off if `reacted_by_me: true`.

### Realtime reaction update

1. New `message_reactions` INSERT event → find affected message in local state, increment count for emoji. If `user_id === currentUserId`, set `reacted_by_me: true`.
2. New `message_reactions` DELETE event → find affected message, decrement count. Remove pill if count reaches 0.
3. No full reload. No visible loading state for realtime updates.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Emoji strip accessibility prefix | "React with [emoji name]" — e.g. "React with heart", "React with laughing face", "React with surprised", "React with crying", "React with thumbs up", "React with fire" |
| Badge pill accessibility | "[Emoji name] reaction, [N] people" — e.g. "Heart reaction, 2 people" |
| Own reaction badge accessibility | "[Emoji name] reaction, [N] people. You reacted. Tap to remove." |
| Optimistic revert (silent) | No user-visible copy — failure is silent per D-07; no toast needed |
| Primary CTA label | No explicit CTA — reactions are tap-to-react (icon affordance only) |
| Empty state | Not applicable — badge row is hidden when no reactions exist |
| Error state | Silent revert only (no error toast for failed reaction write) |
| Destructive confirmation | Not applicable — no destructive action in this phase |

## States Reference

For every interactive element, these states must be handled:

### EmojiStripRow emoji button

| State | Visual |
|-------|--------|
| Default (no reaction from user) | Transparent background, emoji at full opacity |
| Highlighted (user has reacted with this emoji) | `backgroundColor: rgba(249,115,22,0.20)`, `borderRadius: RADII.full` |
| Pressed (active) | `activeOpacity: 0.7` on TouchableOpacity |

### ReactionBadgeRow pill

| State | Visual |
|-------|--------|
| Other users' reaction, not own | `backgroundColor: COLORS.surface.card`, no border |
| Own reaction | `backgroundColor: rgba(249,115,22,0.20)`, `borderWidth: 1`, `borderColor: COLORS.interactive.accent` |
| Pressed (active) | `activeOpacity: 0.7` |
| Optimistic pending (just tapped) | No distinct visual — update appears instant |

### Badge row visibility

| Condition | Behavior |
|-----------|----------|
| `message.reactions` is undefined, null, or empty | Badge row not rendered |
| All reactions have count 0 (post-remove) | Badge row not rendered |
| At least one reaction with count ≥ 1 | Badge row rendered |
