Create `src/components/common/ErrorDisplay.tsx` with two modes: inline and screen-level.

**Props interface (export as `ErrorDisplayProps`):**
```typescript
interface ErrorDisplayProps {
  message: string;                // User-friendly message shown to the user
  technicalDetails?: string;      // Logged to console.error, NOT displayed
  mode?: 'inline' | 'screen';    // Default: 'inline'
  onRetry?: () => void;          // Optional retry callback — shows retry button when provided
}
```

**Inline mode (`mode='inline'`, default):**
- Simple `<Text>` with error styling
- `color: COLORS.interactive.destructive`
- `fontSize: FONT_SIZE.md` (14)
- `fontWeight: FONT_WEIGHT.regular`
- No icon, no padding — the parent controls layout

**Screen-level mode (`mode='screen'`):**
- Centered layout (like EmptyState): `flex: 1, alignItems: 'center', justifyContent: 'center', paddingHorizontal: SPACING.xxl`
- Ionicons `alert-circle-outline` icon, size 48, color `COLORS.interactive.destructive`
- Message text below icon: `FONT_SIZE.lg`, `FONT_WEIGHT.regular`, `COLORS.text.secondary`, `textAlign: 'center'`, `marginTop: SPACING.lg`
- When `onRetry` is provided: render a `TouchableOpacity` retry button below the message
  - `marginTop: SPACING.xl`
  - `paddingHorizontal: SPACING.xl`, `paddingVertical: SPACING.md`
  - `borderRadius: RADII.md` (import RADII)
  - `backgroundColor: COLORS.interactive.accent`
  - Text: "Try Again", `color: COLORS.surface.base`, `fontSize: FONT_SIZE.lg`, `fontWeight: FONT_WEIGHT.semibold`

**Technical details logging:**
- In a `useEffect` (or at render time before return), if `technicalDetails` is provided: `console.error('[ErrorDisplay]', technicalDetails)`

Use `StyleSheet.create()` at module level. Named export for both `ErrorDisplay` and `ErrorDisplayProps`.
