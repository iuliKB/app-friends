<objective>
Add a session-ready `useEffect` to `src/app/(tabs)/_layout.tsx` that calls `registerForPushNotifications(userId)` once when the user becomes authenticated AND re-registers on every `AppState` 'active' transition. Store the result in component state so the layout can render `<PrePromptModal>` when the result is `'needs_pre_prompt'`. This closes the silent plan-invite gap on fresh installs (PUSH-05 → satisfied transitively by PUSH-01) AND completes the runtime wiring for PUSH-08 (pre-prompt → native prompt chain).

Purpose: The current bug is that registration only fires when the user toggles the Profile switch. Moving it to a session-ready effect means every authenticated launch registers a token automatically. The `AppState` foreground listener catches APNs/FCM token rotation that happens silently while the app is backgrounded. The pre-prompt render here is what makes PUSH-08 actually work at runtime — without it, the PrePromptModal component (Plan 06) is dead code.

Output: One updated file (`src/app/(tabs)/_layout.tsx`) with a session-ready effect, AppState listener, registration-result state, and a `<PrePromptModal>` JSX render with Accept / Decline handlers.
</objective>

**2. Inside `TabsLayout()` function**, after the existing `const insets = useSafeAreaInsets();` line, add the session-ready effect AND the registration-result state:

```typescript
  // PUSH-01 + PUSH-02 + PUSH-08 (D-16, D-01..D-03): register token once per authenticated
  // session, re-register on every AppState 'active' transition to catch OS token rotation,
  // and surface the value-led PrePromptModal when the hook reports 'needs_pre_prompt'.
  // Failures are intentionally swallowed — registration must not crash the tabs layout.
  const userId = useAuthStore((s) => s.session?.user?.id);
  const appState = useRef<AppStateStatus>(AppState.currentState);
  const [registerState, setRegisterState] = useState<RegisterResult>('skipped');

  useEffect(() => {
    if (!userId) return;

    // Initial register on session-ready
    registerForPushNotifications(userId)
      .then((result) => setRegisterState(result))
      .catch(() => {});

    // Foreground re-register
    const sub = AppState.addEventListener('change', (next) => {
      if (appState.current.match(/inactive|background/) && next === 'active') {
        registerForPushNotifications(userId)
          .then((result) => setRegisterState(result))
          .catch(() => {});
      }
      appState.current = next;
    });

    return () => sub.remove();
  }, [userId]);

  // PrePromptModal handlers (D-02, D-03).
  async function handlePrePromptAccept() {
    if (!userId) return;
    // Mark seen BEFORE re-calling register, so the hook takes the skipEligibilityCheck
    // branch and fires the native iOS prompt exactly once.
    await markPushPrePromptSeen();
    const result = await registerForPushNotifications(userId, {
      skipEligibilityCheck: true,
    }).catch(() => 'skipped' as RegisterResult);
    setRegisterState(result);
  }

  async function handlePrePromptDecline() {
    // D-03: "Not now" does NOT call requestPermissionsAsync. We also set the
    // opt-out flag so the pre-prompt will not re-surface on every foreground.
    await markPushPrePromptSeen();
    await setNotificationsEnabledFlag(false);
    setRegisterState('skipped');
  }
```

**3. Render `<PrePromptModal>` inside the returned JSX.** The existing return statement wraps `<Tabs>...</Tabs>` — wrap the whole return in a Fragment so the modal is a sibling:
