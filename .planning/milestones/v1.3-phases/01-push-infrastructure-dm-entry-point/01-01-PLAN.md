**Why an EAS dev build is required (v1.3)**
- iOS notification action buttons (Free / Busy / Maybe morning prompt) DO NOT render in Expo Go — they require a custom dev client.
- Differentiated Android channels (`plan_invites` MAX, `friend_free` HIGH, `morning_prompt` DEFAULT, `system` LOW) are not introspectable from Expo Go.
- `Notifications.setNotificationCategoryAsync` for remote pushes is unreliable in Expo Go on iOS.
- Per D-11, this build is the FIRST deliverable — every subsequent client-side smoke test depends on it.

**Prerequisites**
- `eas-cli` installed: `npm install -g eas-cli` (or use `npx eas-cli@latest`)
- Logged in: `eas login` (Expo account with access to the campfire project)
- `EAS_PROJECT_ID` env var set OR `eas init` run once (writes `extra.eas.projectId`)
- Apple Developer account (iOS) — EAS handles provisioning automatically with `--auto-submit` off
- Android keystore — EAS generates one on first build

**Commands**
```bash
# 1. From repo root, ensure deps clean
npm install

# 2. Initialize EAS if not already done (writes eas.json if missing — review before commit)
eas init   # SKIP if eas.json already exists

# 3. iOS dev build
eas build --profile development --platform ios

# 4. Android dev build
eas build --profile development --platform android

# 5. Wait for both builds (5-15 min each). Output is a downloadable .ipa / .apk URL.
```

**Installing on test devices**
- iOS: open the build URL on the device, install via Safari (TestFlight not needed for dev builds — direct install with provisioning profile)
- Android: download the .apk, enable "Install from unknown sources," install

**Smoke check (after install)**
- App launches and shows the splash screen
- Logging in works
- Push token registers (verify in Supabase: `select * from push_tokens where user_id = '<your-id>'` — should return at least 1 row after Plan 04 ships; for now just confirm app launches)

**Troubleshooting**
- "Project ID not found" → run `eas init` to write `extra.eas.projectId` into `app.config.ts` (or set `EAS_PROJECT_ID` env var)
- "expo-notifications plugin missing icon" → expected; Plan 03 of this phase converts the plugin to tuple form
- iOS provisioning errors → run `eas credentials` to inspect/regenerate
- Android keystore errors → run `eas credentials --platform android`
